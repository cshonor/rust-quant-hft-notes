## 2.3.1 时间尺度与排查走查

> **口诀：** 先圈 **战场量级** → 再选术语 + [命令](./section-2.2-术语与命令速查.md) → **别跳档优化**。

← [2.2 命令速查](./section-2.2-术语与命令速查.md) · → [2.3.2 性能权衡](./section-2.3.2-性能权衡.md)

**两层用法：**

```
① 时间尺度  →  订单延迟落在哪个数量级？先圈定排查的「战场范围」
② 专业术语  →  在该量级上用 Throughput / RTT / Saturation 等量化、拆到具体组件
```

### Gregg「时间放大」类比

```
基准：1 个 CPU 周期 ≈ 0.3 ns  →  在类比里 = 1 秒
换算：真实延迟 ÷ 0.3 ns  ≈  放大后的「秒数」
```

| 事件 | 真实延迟 | 放大后感受 |
|------|----------|------------|
| L1 命中 | ~1 ns | **~3 秒** |
| L2 / L3 | ~10～40 ns | **数分钟** |
| 主存访问 | ~100 ns | **~5 分钟** |
| 网络 RTT | ms 级 | **数年～世纪级** |
| 跨核 / 跨 NUMA | miss 上再翻倍 | **~10 分钟** |
| 内核态 / syscall | μs～ms | **世纪级** |

```
L1 不命中 → 主存  =  从「几秒」→「几分钟」
再跨网络        =  直接「几年」档 — HFT 热路径上的世纪级浪费
```

**HFT 推论：** 订单簿、持仓、风控计数器 **锁在 L1/内存**；tick→撮合→信号 上每多一次主存 miss，都是在抢「几秒 vs 几分钟」。

### HFT · L1 命中是什么

**大白话：** 数据 **刚好在 L1**，不用去 L2/L3/主存。

| 概念 | 说明 |
|------|------|
| **L1 未命中** | 延迟翻几倍～几十倍 — 撮合从百 ns 飙到 μs |
| **HFT 热数据** | 订单簿档位、最优价、持仓、风控计数器 — 布局紧凑、绑核 |

**和时间尺度的关系：** 只有锚到 **路径 B**（~500ns 撮合）才抠 L1；端到端 **2ms 网络**（路径 A）先别纠结 cache。

→ [Ch 6](../../chapter-06-cpus/) · [Ch 4 PMC](../../chapter-04-observability-tools/notes/section-4.3-核心观测数据源.md)

### 延迟落在哪一档，就查哪一档

| 端到端延迟 | 战场 | 优先查 | **别死磕** |
|------------|------|--------|------------|
| **μs 级** tick→signal | CPU / cache | `perf`、绑核、false sharing | 交换机 ms 抖动 |
| **百 μs～ms 级** ACK | 网卡、TCP 队列 | `ss -s`、`ethtool -S`、RTT | 纳秒 cache 微调 |
| **ms+ 级** | 网络路径、拥塞 | RTT、重传、路径 | 单机指令级优化 |

**反例：** 只抠 cache，没解决 ms 级网络抖动 — **时间尺度锚错了**。

### 排查顺序 vs 优化优先级

Gregg **「网络是替罪羊」** = **排查顺序**，不是「网络不重要」。

| | 规则 |
|--|------|
| **排查顺序** | 先 `perf`/`mpstat`/`vmstat` 排除本地 — **再碰网络** |
| **优化优先级** | `tcpdump`/`ss` 实锤后 — 网络 **最高优先级**（年级收益） |

**一句话：** 排查时网络是「最后怀疑对象」；实锤后是「第一个优化对象」。

#### 路径 A · 发单→ACK 2ms（网络战场）

```
0. 本地排查：mpstat / vmstat / 分段计时 — 别一上来怪交易所
1. 锚量级：tcpdump / ACK 时间戳 → 2ms 在网络段
2. ss -s / ethtool -S / sar -n DEV / RTT
3. 队列满、丢包 → 迁 IRQ、换路径 — 不是改 if-else
```

#### 路径 B · 撮合 500ns（CPU/cache 战场）

```
1. rdtsc / histogram → 锚定 ns～μs CPU 热路径
2. perf top / 火焰图 / cache-misses
3. 改数据结构、绑核 — 不是先怪交换机
```

| 量到什么 | 锚在哪 | 工具 | 别误闯 |
|----------|--------|------|--------|
| ACK **2ms** | ms·网络 | `ss`、`ethtool` | `perf` 抠 cache |
| 撮合 **500ns** | ns～μs·CPU | `perf`、火焰图 | `tcpdump` |

| 步骤 | 做什么 |
|------|--------|
| 1 | P99/Max 落在 ns/μs/ms → **定战场** |
| 2 | 选 Utilization / Saturation / Throughput / Latency |
| 3 | 用 [2.2 命令表](./section-2.2-术语与命令速查.md) → **定工具** |

### 优化优先级：先砍「分钟、年级」，再挤「毫秒」

| 你在砍什么 | 类比里 | HFT 动作 |
|------------|--------|----------|
| **主存 miss** | ~5 分钟 | 紧凑 order book、SoA |
| **跨核 / NUMA** | ~10 分钟 | 绑核、减 false sharing |
| **内核 / syscall** | 世纪级 | DPDK、用户态栈 |
| **JVM / GC** | 毫秒档挤水分 | **大尺度清完后再碰** |

```
错误：P99 飙了 → 先改 G1
正确：miss + 跨核 + 内核 → 算法/绑核 → 最后 JVM
```
