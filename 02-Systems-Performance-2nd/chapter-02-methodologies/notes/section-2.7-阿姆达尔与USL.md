## 2.7 · 可扩展性模型 — Amdahl 与 USL

> 排队论管 **单资源饱和**；可扩展性模型管 **加核有没有用**。

### 阿姆达尔定律（Amdahl's Law）

```
S(N) = 1 / ( f + (1-f)/N )
```

- **N** = 并行度（核数）
- **f** = **必须串行** 比例

| f | 8 核上限 | HFT |
|---|----------|-----|
| 5% | ~4.7× | 一点全局锁就封顶 |
| 10% | ~4.3× | 常见策略引擎 |
| 20% | ~3.3× | 堆 32 核可能只值 3× |

**用法：** tick→**串行风控**→send — 先量 **f**，再算加核值不值；不如 **砍串行**。

### 通用可扩展性模型（USL）

```
X(N) = N / ( 1 + α(N-1) + βN(N-1) )
```

| 项 | 含义 | HFT |
|----|------|-----|
| **α(N-1)** | 争用 | 锁、mutex、单连接发单 |
| **βN(N-1)** | 一致性 | false sharing、NUMA 订单簿 |

加核过峰值后吞吐 **反降** — 须 **分片 / 无锁 / 多实例**。

**tick 2× 反推资源：**

```
1. 基线：N 核、tick_rate R、ρ_cpu、P99
2. 目标：2R
3. Amdahl：f>10%？→ 先减 f
4. USL：拟合 α、β — N' 核能否 ≥ 2R？
5. M/M/c：ρ 仍 < 安全水位？网卡 pps、RSS WSS？
```

**例：** 50 万→100 万 tick/s

| 步 | 动作 |
|----|------|
| M/M/c | 4 核 ρ=0.55 → 2× 需至少 c=8 |
| USL | α 大 → 先 shard 订单簿 |
| 网卡 | 查 **PPS** 非仅 Gbps |
| 内存 | 2× tick 压 WSS |
