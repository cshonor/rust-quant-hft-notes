## 大白话 · 本章就三件事

> 先记住下面顺序，后面学工具时不迷路。

**① 目标就一句：让程序跑更快、等的时间更少。**

- 书里叫降低延迟、提高吞吐量；量化里就是：行情到了 → 算完 → 发出去，**整条链路上少等一点**。
- 下单少等 1ms，有时就是更好的成交价 — 这就是性能优化的业务价值。
- **热路径** = 上面这条链路上 **每个 tick 必跑** 的那几段代码 → [1.4 定义](./notes/section-1.4-热路径Resource与双视角.md)

**② 不同人看性能的角度不一样。**

| 谁 | 主要看什么 |
|----|-----------|
| 开发 | 代码逻辑、程序在干什么 |
| 运维 | CPU、内存、磁盘、网络够不够用 |
| **你做量化** | **策略计算 + 发单这两步各花了多久**（够用就行，不必照搬运维全套） |

**③ 找问题有固定顺序 — 别一上来钻底层代码。**

```
先看：程序要做什么工作（业务流程 / 热路径）
再看：用了多少 CPU、内存、网络（资源消耗）
```

对应原书两个词：**工作负载分析** → **资源分析**。  
你现在的抓手：**先看业务流程，再查资源消耗。**

**量化专用：** [行情 tick 全链路 Checklist](./notes/section-1.1-1.3-系统性能角色与活动.md#hft--行情-tick-全链路-checklist) · [HFT 10 秒检查清单](./notes/section-1.9-1.11-云计算方法论与案例.md#hft--10-秒检查清单方法论适配实盘)（1.10 节）
