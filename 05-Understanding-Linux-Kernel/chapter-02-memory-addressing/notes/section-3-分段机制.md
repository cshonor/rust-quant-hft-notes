## 3. 硬件分段与 Linux 的分段策略

---

### 一、硬件分段怎么工作

逻辑地址 = **段选择符（Segment Selector）** + **偏移量（Offset）**

1. 用段选择符在 **GDT**（全局描述符表）或 **LDT**（局部描述符表）里找 **段描述符**
2. 段描述符 + 偏移 → **线性地址**

---

### 二、Linux 为什么「弱化」分段

| 倾向 | 原因 |
|------|------|
| **主要靠分页** | 内存管理更简单 |
| **便于移植** | RISC 等架构往往**没有** x86 式分段 |

Linux 2.6 对分段**极大简化**：

- 用户态、内核态进程使用**相同的寻址范围**
- 内核主要用 **四个段**：
  - 用户代码段、用户数据段
  - 内核代码段、内核数据段
- 这些段的线性地址均从 **0** 起，跨越整个 **4 GB** —— 等于「分段平坦化」，**真正管内存的是分页**

→ 分页 [section-4](./section-4-硬件分页.md) · Linux 四级模型 [section-5](./section-5-Linux四级分页.md)

---

### 三、Modern 对照

- x86-64 长模式下分段作用更弱，几乎**平坦模型 + 分页**
- 读 2.6 时抓「**Linux 用分段满足硬件要求，用分页做真正隔离**」即可
