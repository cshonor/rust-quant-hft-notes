## 3. 设备驱动程序模型 (Device Driver Model)

> Linux 2.6 **统一驱动模型** + **`sysfs`**

---

### 一、sysfs 与硬件拓扑

**`sysfs`**（挂载于 `/sys`）将 **硬件拓扑与设备状态** 映射为 **目录树** — 用户态可直接读写在驱动模型中注册的对象属性。

→ VFS 特殊 FS：[Ch 12 section-5](../chapter-12-VFS/notes/section-5-挂载与路径查找.md)

---

### 二、底层：kobject / kset / subsystem

| 对象 | 作用 |
|------|------|
| **`kobject`** | 驱动模型 **基础** — 对应 sysfs 中一个 **目录**；引用计数、层次链接 |
| **`kset`** | 同类型 kobject 的 **集合** |
| **`subsystem`** | 多个 kset 的 **上层分组** |

生命周期：注册 → sysfs 出现 → 引用归零 → 释放。

> **深潜可选：** `kobject_init`、`kobject_put` — 见 `lib/kobject.c`。

---

### 三、四大高层组件

内核用统一模型抽象 **整个硬件世界**：

| 组件 | 含义 |
|------|------|
| **Device（设备）** | 物理或逻辑硬件实例 |
| **Driver（驱动）** | 与该设备匹配的 **驱动程序** |
| **Bus（总线）** | 挂载与 **匹配** device ↔ driver（如 PCI、USB） |
| **Class（类）** | **逻辑功能** 分组（如「网卡」「块设备」）— 与用户看到的 `/dev` 类型相关 |

**匹配流程（概念）：** 设备注册到总线 → 内核尝试绑定已有驱动 → probe → 创建设备节点。
