## 2. I/O 体系结构 (I/O Architecture)

> 80x86 视角的 **硬件 I/O 基础**

---

### 一、总线与设备

计算机通过 **系统总线** 连接外设：

| 总线示例 | 典型设备 |
|----------|----------|
| **PCI / PCIe** | 网卡、显卡、NVMe |
| **ISA**（遗留） | 老 DMA 限制（→ [Ch 8 ZONE_DMA](../chapter-08-memory-management/notes/section-2-页框管理.md)） |
| **USB** | 外设 |

CPU 不直接「摸」设备芯片，经 **I/O 接口** 与 **设备控制器** 通信。

---

### 二、CPU 如何寻址设备

| 方式 | 说明 |
|------|------|
| **I/O 端口 (Port I/O)** | 独立端口地址空间；x86 用 **`in` / `out`** 汇编读写控制/状态/数据寄存器 |
| **内存映射 I/O (MMIO)** | 设备寄存器映射到 **物理内存地址** — 像访存一样读写 |

现代 PCI 设备 **多为 MMIO**；legacy 芯片仍可能用端口 I/O。

---

### 三、接口 vs 控制器

| 组件 | 角色 |
|------|------|
| **I/O 接口** | 硬件电路 — 在 CPU 侧端口/MMIO 与设备侧信号之间 **翻译** 命令与数据 |
| **设备控制器** | 设备内部微芯片 — **实际执行** I/O（如磁盘寻道、NIC 发包） |

驱动程序通过读写 **寄存器** 向控制器下命令，再等待完成。
