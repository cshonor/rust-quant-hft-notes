## 3. 扇区、块与段 (Sectors, Blocks, Segments)

> 统一 **硬件 / FS / DMA** 各层对「一块数据」的叫法

---

### 一、扇区 (Sector)

| 属性 | 说明 |
|------|------|
| **层级** | 磁盘 **硬件** 最小传输单位 |
| **典型大小** | **512 字节**（现代也有 4K 扇区） |
| **视角** | 控制器把盘看成 **扇区数组** |

---

### 二、块 (Block)

| 属性 | 说明 |
|------|------|
| **层级** | **VFS / 文件系统** 逻辑单位 |
| **大小** | 扇区的 **整数倍**；须能放进 **一个物理页框**（如 1KB、4KB） |
| **关系** | FS 按 **块** 组织；块层再映射到 **扇区** |

→ 页框：[Ch 8](../chapter-08-memory-management/notes/section-2-页框管理.md)

---

### 三、段 (Segment)

为 **scatter-gather DMA** 引入：

| 含义 | 单次 DMA 中 **内存连续**、且对应磁盘 **相邻扇区** 的一段 |
|------|--------------------------------------------------------|
| 目的 | 一次 DMA 传 **多段** 不连续物理页 — 减少 CPU 拷贝 |

→ DMA 概念：[Ch 13 section-5](../chapter-13-io-architecture/notes/section-5-驱动通用特性.md)

---

### 四、三者关系简图

```
文件 read()
    ↓ 以 block 为单位（FS / 页缓存）
bio 描述若干 sector
    ↓ bio_vec[] 每段 = 连续物理页 + 扇区范围
DMA scatter-gather 到磁盘
```
