## 5. IPC 消息队列 (System V Messages)

> 带 **类型标签** 的消息链表 — 选择性接收

---

### 一、模型

| 操作 | 说明 |
|------|------|
| **发送** | 进程将 **短数据块** + **message type** 入队 |
| **接收** | 可 **只读特定类型** — 不必 FIFO 顺序 |

适合 **多类型事件** 共用一个队列。

---

### 二、消息结构

| 部分 | 说明 |
|------|------|
| **`msg_msg` 头** | 固定大小元数据（类型、长度等） |
| **文本体** | 可变长度 payload |

**超长消息：** 超过 **一页** 时 — 分配 **额外页**，用 **`msg_msgseg`** **链式** 串联。

→ 页框：[Ch 8](../chapter-08-memory-management/notes/section-2-页框管理.md)

---

### 三、与管道的对比

| | 管道 | SysV 消息队列 |
|--|------|---------------|
| 边界 | 字节流 | **离散消息** + **类型** |
| 命名 | 匿名 / FIFO 路径 | **IPC key** |
| 拷贝 | 经内核缓冲 | 经内核 **msg** 结构 |
