# Ch 8 §1 IPv6 地址架构 · IPv6 Addresses

> **Linux Kernel Networking** · Rami Rosen · **跳过 ⚪**

### 1. IPv6 地址架构 (IPv6 Addresses)

IPv6 地址 **128 位** — 书写为 **8 组 16 位十六进制**（`2001:db8::1`）。按 **通信语义** 分为 **单播、任播、多播** — **无广播**。

---

## 三种地址类型

| 类型 | 语义 | 示例场景 |
|------|------|----------|
| **单播 (Unicast)** | 唯一接口 | 主机/global 地址、link-local |
| **任播 (Anycast)** | **一组接口中路由最近的一个** | DNS Anycast、部分 **VIP** |
| **多播 (Multicast)** | 一组订阅者 | MLD 组、**ff02::1** 所有节点 |

**取消广播：** IPv4 **255.255.255.255 / 子网广播** → IPv6 用 **多播** 替代 — 减 **L2 泛洪**。

```
IPv4 ARP 广播问 MAC  →  IPv6 NS 发往 solicited-node 多播（Ch 7）
IPv4 子网广播       →  ff02::1 等链路范围多播
```

---

## 特殊地址

| 地址 | 用途 |
|------|------|
| **`::` / `::0`** | **未指定** — DAD 期间源地址、bind 占位 |
| **`::1`** | **环回** — 等同 127.0.0.1 |
| **`fe80::/10`（常用 /64）** | **链路本地 (Link-local)** — **仅同一 L2**，ND/RS/RA、本地服务 |
| **`ff02::1`** | 链路本地 **所有节点** 多播 |
| **`ff02::2`** | 链路本地 **所有路由器** 多播 |

**Link-local：** 每接口 **自动** `fe80::/64`（EUI-64 或 stable privacy）— **路由器不转发** fe80。

---

## 与 IPv4 对照

| | IPv4 | IPv6 |
|---|------|------|
| 长度 | 32 bit | **128 bit** |
| 广播 | 有 | **无** |
| 本地自动 | APIPA 169.254/16 | **fe80 + SLAAC**（§4） |
| 组播 | 224/4 + IGMP | **ff00::/8 + MLD**（§6） |

**HFT：** 共置 **IPv4 私网 + 固定 MAC** 为主；若见 **fe80** 流量 — 多为 **ND/MLD 控制面**，非行情 payload。
