### M/D/1 — 固定耗时纯内存报单 {#md1}

> **M/D/1** = 随机到达 + **固定服务** + 1 节点。拐点 **~80%**，比 M/M/1 **高约 10%**。

| | M/M/1 | M/D/1 |
|--|-------|-------|
| 服务时间 | 随机 | **固定** |
| 排队来源 | 到达 + 服务随机 | **仅到达随机** |
| 拐点 | **~70%** | **~80%** |

```
W / W₀ = (2 - ρ) / (2(1 - ρ))
L      = ρ + ρ² / (2(1 - ρ))
```

| ρ | M/M/1 W/W₀ | M/D/1 W/W₀ |
|---|------------|------------|
| 70% | 3.3× | 2.2× |
| **80%** | 5.0× | **3.0×** ← M/D/1 拐点区 |

**HFT：** 固定长度序列化 + 内存订单簿、无 DB/复杂分支 → **ρ ≲ 0.8**（μ=10k → 限 **~8k/s** vs M/M/1 **~7k**）。

**不能套 M/D/1：** 慢查风控、随机分支、cache miss 主导 → **M/M/1** 或 **M/G/1**。

---

### M/M/c — 多核撮合集群 {#mm/c}

> **M/M/c** = 随机到达 + 随机服务 + **c 节点**。

```
ρ = λ / (c × μ)     （系统级利用率）
```

| 对比 M/M/1 | M/M/c |
|------------|-------|
| 单核 ρ=0.7 已陡 | c=4 分摊 λ，理想近 4× 吞吐 |
| 单队列 | 共享队列 + 锁 → 有效 c 可能 < 物理 c（→ [2.7.1 USL](./section-2.7.1-阿姆达尔与USL.md)） |

**拐点随 c 上移：**

| 模型 | 拐点（参考） | 告警（−10~15%） |
|------|--------------|-----------------|
| M/M/1 | ~70% | ~60% |
| M/D/1 | ~80% | ~65~70% |
| **M/M/8** | **~75%~80%** | **~65~70%** |

**HFT 映射：** 多核撮合（c=核数）、RSS 多队列、线程池报单序列化。

**实操：** 扫 c=1,2,4,8… 至 ρ 低于模型安全水位且 W 达标；加核不成比例 → 争用，查 USL。

---

### M/G/1 — 快慢单混合

> **G** = 任意服务时间分布 — 只需 **E[S]** 与 **Var(S)** / P99。

- Var(S) 越大 → 同样 ρ 排队 **越长**
- 90% 快单 + 10% 慢查 → **M/M/1 低估队列**，用 M/G/1

**输入：** [2.5 histogram](./section-2.5-性能分析方法论.md) 或压测服务时间样本 → 喂计算器。
