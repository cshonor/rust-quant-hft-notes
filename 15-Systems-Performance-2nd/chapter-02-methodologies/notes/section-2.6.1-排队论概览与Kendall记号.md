### 大白话：排队论是什么？

排队论研究的是 **「系统接请求的排队规律」**。

**餐厅类比：**

| 餐厅 | 排队论 | HFT |
|------|--------|-----|
| 进店的 **客人** | **请求（到达）** | **报单请求**、行情 tick |
| **服务员 + 后厨**（接待上限） | **服务节点** | **报单引擎**、撮合 loop |
| 等位多久、队排多长 | **排队延迟 L、逗留时间 W** | submit→ACK 变慢、网关对象积压 |
| 饭点会不会 **爆单** | 利用率 ρ 过高 → 系统失稳 | 开盘 burst → **timeout / reject** |

1. **客人多久能吃上** → **W**
2. **队排多长** → **L**
3. **高峰期会不会炸** → λ 接近 μ 时延迟 **陡增**

**落到 HFT：** 安全水位取决于模型 — **M/M/1 ≈ 70%** vs **M/D/1 ≈ 80%**（详见各子节）。

---

### 三参数标记法：A / B / c

```
  A        /    B      /   c
  到达间隔      服务时间     并行节点数
```

#### 第一位 · A（到达间隔）

| **M**（Markov） | 负指数间隔 → Poisson 到达 | 行情 tick、零散报单、开盘 burst |

#### 第二位 · B（服务时间）

| **M** | 服务时间随机 | 分支、cache miss |
| **D** | **Deterministic** 固定 | 纯内存查订单簿、无分支撮合 |
| **G** | **General** 任意 | 快单 + 慢查混合 |

#### 第三位 · c（节点数）

| **1** | 单线程报单引擎 |
| **c** | 多核撮合、线程池、RSS 多队列 |

**三个 HFT 常碰组合：**

| 模型 | 场景 | 拐点参考 |
|------|------|----------|
| **M/M/1** | 随机服务 + 单线程 | **ρ ≲ 70%** → [详](./section-2.6.2-M-M-1-拐点与预警线.md) |
| **M/D/1** | 固定服务 + 单线程 | **ρ ≲ 80%** → [详](./section-2.6.3-M-D-1-M-M-c-M-G-1.md#md1) |
| **M/M/c** | 随机服务 + 多核 | **~75%~80%**（c=8）→ [详](./section-2.6.3-M-D-1-M-M-c-M-G-1.md#mm/c) |

**选型速查：**

```
服务耗时有随机波动     →  M/M/1   ~70%
纯内存、无分支热路径   →  M/D/1   ~80%
多核 / 线程池         →  M/M/c
快慢单混合            →  M/G/1
```

---

### 符号与 Little 定律

资源有队列时，**利用率未到 100%，响应时间也会恶化** — 「CPU 才 70% 怎么 P99 爆了？」常对应 **M/M/1**。

| 符号 | 含义 | HFT 例子 |
|------|------|----------|
| **λ** | 到达率 | 报单/s、tick/s |
| **μ** | 服务率（单节点上限） | 单线程 **orders/s** |
| **ρ = λ/μ**（M/M/1） | 利用率，须 **< 1** | λ=7000, μ=10000 → ρ=0.7 |
| **L** | 平均任务数（含排队） | 网关积压对象 |
| **W** | 平均逗留时间 | submit→ACK |

**Little 定律：** `L = λ × W`
