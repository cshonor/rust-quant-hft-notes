## 4. sti 与 hlt 冻结 Bug

---

### 一、现象

首次运行 **`hlt` 循环 .bin`：**

| 表现 | 说明 |
|------|------|
| **终端无响应** | 看似 **应用占住** |
| **整个 OS 假死** | 鼠标 · 定时器 · 其他任务 **全停** |

---

### 二、根因

**调用应用前 CPU 处于 `cli`（中断关闭）状态**（GUI/终端消息处理路径常见）。

```
cli 状态下执行 hlt
    → 等待中断唤醒
    → 定时器/键盘中断进不来
    → 永久睡眠
```

| 组件 | 依赖 |
|------|------|
| **Ch11 APIC 定时器** | **中断** 驱动 **Preempt / Wakeup** |
| **Ch13 多任务** | 无 tick → **无法切换** |
| **hlt** | 需 **IF=1** 才有意义 |

→ [Ch7 中断](../chapter-07-interrupt-fifo/) · [Ch13 抢占](../chapter-13-multitask1/)

---

### 三、修复

**跳转应用前：**

```cpp
__asm__("sti");
auto entry = reinterpret_cast<int(*)()>(load_addr);
int exit_code = entry();
// 或 sti 后 call
```

| 原则 | 说明 |
|------|------|
| **应用运行期 IF=1** | 定时器可 **抢占** · 系统 **仍活** |
| **返回后** | 可按需 **cli** 再进临界区 |

**书中教训：** **跑外来代码** 必须考虑 **全局中断状态** — 与 **驱动 ISR** 同类问题。
