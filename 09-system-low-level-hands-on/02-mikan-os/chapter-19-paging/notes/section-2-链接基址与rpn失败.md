## 2. 链接基址与 rpn 失败

---

### 一、现象

```
> rpn 2 3 +
0          ← 期望 5
```

**应用已加载、已执行、未崩溃** — 但 **逻辑/数据错误**。

---

### 二、链接器与加载地址

**编译 C++ 应用时：**

```makefile
LDFLAGS += --image-base 0    # 早期配置
```

| 链接器假设 | 实际 OS 行为 |
|------------|--------------|
| **代码/全局变量在 VA 0 附近** | 读 ELF 后拷到 **0x100000** 等 **固定物理区** |
| **`e_entry`、符号地址基于 0** | CPU 按 **错误 VA** 取指令/读写 `.data` |

**结果：** 栈/堆/全局 **错位** — 计算器读到 **0** 或垃圾。

---

### 三、为何不选「软件重定位」

| 方案 | 本书选择 |
|------|----------|
| **加载时修 ELF 重定位表** | 可行但 **实现量大** |
| **虚拟地址 + 页表** | **硬件 MMU** 做映射 — 与 **现代 OS** 一致 |

**思路：** 让 **链接基址 = 应用使用的 VA** — OS 建页表把 **VA 映到实际物理页**。

→ [Ch18 ELF 加载](../chapter-18-apps/notes/section-5-ELF格式与命令行参数.md)

---

### 四、Ch18 遗留

| 项 | Ch18 | Ch19 改 |
|----|------|---------|
| 加载 | 物理 buffer **直接 jump** | **换 CR3** 后按 **VA entry** 调用 |
| 隔离 | 无 | **独立 PML4** |
| 清理 | 无 | **CleanPageMaps** |
