## 3. 虚拟地址与地址转换

---

### 一、虚拟地址（VA）是什么

**应用视角：** 每条 `mov`/`load`/`store` 使用 **虚拟地址**。

**CPU（MMU）视角：** 查 **当前 CR3 页表** → 得到 **物理地址（PA）** 再访问 RAM。

```
应用:  mov rax, [0xffff800000001000]   ; VA
MMU:   VA → PA 0x0034_5000
RAM:   读物理 0x0034_5000
```

---

### 二、解决的问题

| 问题 | 分页如何解决 |
|------|--------------|
| **链接地址 ≠ 加载物理地址** | **VA 固定**（链接基址）· **PA 任意**（OS 分配） |
| **多应用冲突** | 各自 **CR3/页表** — 可 **同 VA 不同 PA** |
| **保护** | 页表项 **U/S · NX · RW** — Ch20 **用户态** 基础 |

---

### 三、MikanOS 地址空间布局（本章）

```
低地址 ───────────────────────────── 高地址
[ 内核 · 设备 · 身份映射区 ]  |  [ 应用 LOAD 段 — 高半 0xffff8000… ]
         OS 侧                          应用侧
```

| 设计 | 说明 |
|------|------|
| **内核低半** | Ch8 **identity map** 延续 |
| **应用高半** | 与内核 **VA 区间分离** — 减少 **误踩** |

---

### 四、执行流程（概念）

```
1. 读 ELF LOAD 段 → 分配物理页
2. SetupPageMaps(app_cr3, vaddr, size, phys_pages)
3. mov cr3, app_cr3
4. call app_entry (VA)
5. mov cr3, kernel_cr3
6. CleanPageMaps(app_cr3)
```

**切换 CR3** 必须在 **内核映射仍有效** 的上下文中 — 通常 **内核高半/identity** 与 **应用表** 并存于 **不同 PML4 项**。
