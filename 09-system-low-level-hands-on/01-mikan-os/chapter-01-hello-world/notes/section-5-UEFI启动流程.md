## 5. UEFI BIOS 启动流程

---

### 一、从通电到程序运行

```
通电
  ↓
CPU 复位 → 执行固化在 ROM 中的 **UEFI 固件**（BIOS 的现代继任）
  ↓
固件初始化：CPU、芯片组、内存、基本 I/O
  ↓
枚举启动设备（硬盘、U 盘、网络…）
  ↓
在 FAT 等分区查找 **/EFI/BOOT/BOOTX64.EFI**（或可配置路径）
  ↓
将 EFI 镜像 **加载到主内存**
  ↓
移交执行权 → 进入程序入口（手写版或 **EfiMain**）
  ↓
Hello, world! 显示在屏幕上
```

---

### 二、关键概念

| 术语 | 含义 |
|------|------|
| **UEFI** | Unified Extensible Firmware Interface — **可扩展固件接口** |
| **UEFI 应用** | 在 UEFI 固件 **服务支持下** 运行的 `.efi` 程序 — **不是** 完整 OS |
| **BOOTX64.EFI** | x64 架构 **默认引导应用** 文件名 |
| **固件服务** | 内存分配、文本输出、磁盘读写等 — 后续章节通过 **EDK II / Boot Services** 调用 |

**与 01 BIOS 引导对比：**

| | **BIOS + 软盘** | **UEFI + EFI** |
|---|-----------------|----------------|
| 入口 | **512B 引导扇区**，实模式 | **PE/COFF 格式 .efi**，已为 **64 位长模式** 环境 |
| 体积 | 极小 IPL | 含头表、节区的 **可执行文件** |
| 后续 | OS 自建 GDT/分页 | 固件已提供 **EFI System Table** |

### 三、本章程序在链中的位置

```
[ 硬件 ]
[ UEFI 固件 ]  ← 非你编写
[ BOOTX64.EFI / Hello World ]  ← 本章 — 第一个「你的代码」
[ 未来：MikanOS 内核 / 加载器 ]  ← Ch 2+
```
