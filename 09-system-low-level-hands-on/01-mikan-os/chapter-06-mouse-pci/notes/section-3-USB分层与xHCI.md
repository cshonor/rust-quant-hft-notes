## 3. USB 分层与 xHCI

---

### 一、为何不用 PS/2 教程路线

| 传统 OS 教程 | **MikanOS** |
|--------------|-------------|
| BIOS 提供 **PS/2 鼠标** 或简化端口 | 直接 **USB 协议栈** |
| 适合老机器 / 模拟器 | **现代 PC 标配 USB** |

**PS/2：** 老键盘鼠标专用接口 — 新硬件常通过 **USB 转接** 或内置 **xHCI**。

---

### 二、USB 驱动分层（概念）

```
┌─────────────────────────────────┐
│  类驱动（Class Driver）          │  HID 鼠标 — 解析报告描述符
├─────────────────────────────────┤
│  USB 总线驱动（Bus Driver）      │  设备枚举、端点、传输调度
├─────────────────────────────────┤
│  主机控制器驱动（HCD）           │  xHCI — 寄存器、环、事件
└─────────────────────────────────┘
         │
    PCI 设备（xHC 硬件）
```

| 层 | 本章触及 |
|----|----------|
| **HCD（xHCI）** | **重点** — MMIO 寄存器、事件环 |
| **Bus** | 简化枚举 — 假定已知鼠标已插 |
| **Class / HID** | 解析 **位移报告**（Δx, Δy） |

→ 对照 Linux **`drivers/usb/host/xhci*.c`**（概念，非代码级）

---

### 三、xHCI 是什么

**xHCI（eXtended Host Controller Interface）** — **USB 3.x** 主机控制器标准接口。

| 特点 | 说明 |
|------|------|
| **统一** | USB 2.0/3.0 设备可挂同一控制器 |
| **MMIO** | 控制寄存器映射到 **物理内存地址**（非 x86 `in/out` 端口） |
| **事件驱动（硬件）** | 硬件写 **Event Ring** — 本章暂用 **轮询** 读 |

**规范角色：** OS 通过 xHCI 寄存器提交 **Transfer Ring**，设备数据经 **DMA** 写入内存。
