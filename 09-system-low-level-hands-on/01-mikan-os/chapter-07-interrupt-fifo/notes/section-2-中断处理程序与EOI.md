## 2. 中断处理程序与 EOI

---

### 一、轮询 vs 中断

| | **轮询（Polling）** | **中断（Interrupt）** |
|---|---------------------|------------------------|
| 谁主动 | **OS 反复问** 硬件 | **硬件** 有事件 **通知** CPU |
| CPU 占用 | 高（空转） | 低（可休眠） |
| 响应 | 取决于循环频率 | 事件到达 **尽快** 进 ISR |

```
轮询：  while(1) if (event) handle();
中断：  idle → [硬件打断] → ISR → resume
```

---

### 二、编写中断处理程序（ISR）

书中使用编译器属性声明 **中断例程**：

```cpp
__attribute__((interrupt))
void IntHandler(/* frame */) {
    // 极简：读状态、Push Message、EOI
}
```

| 要求 | 说明 |
|------|------|
| **尽量短** | 不画光标、不 printk 大段 — 防 **丢中断** |
| **保存/恢复上下文** | 编译器/汇编入口桩处理寄存器 — 与 Ch3 `hlt` 内核协同 |
| **可重入意识** | 同一向量未 EOI 前可能屏蔽 — 视 PIC/APIC/MSI 配置 |

---

### 三、EOI — End Of Interrupt

处理完成后必须告知中断控制器：**本次中断已处理完毕**，CPU 才能可靠接收 **下一次**。

书中 **`NotifyEndOfInterrupt()`** — 向 **特定 MMIO/端口地址写入约定值**（具体依 **APIC/MSI 配置** 与平台）。

| 若忘记 EOI | 后果 |
|------------|------|
| 控制器认为仍处理中 | **后续中断丢失或嵌套异常** |

**原则：** ISR 末尾 **必定 EOI**（除非规范要求特殊 EOImode）。
