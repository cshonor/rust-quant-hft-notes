## ①.1 IPL、bootpack 与镜像布局

> **本节第一次出现三个名字** — 先分清，后面代码才看得懂。

### IPL 和 bootpack 是两回事

| | **IPL** | **bootpack** |
|---|---------|--------------|
| **是什么** | **初始程序加载器**（Initial Program Loader） | **操作系统本体**（后面进 32 位、跑 C 的那一大块） |
| **在软盘上** | **第 1 扇区**（C0-H0-S1），固定 **512 B** | **从第 2 扇区起** 连续很多扇区（本书读 **10 柱面**） |
| **进内存后** | BIOS 放到 **`0x7C00`** 并执行 | IPL 把它放到 **`0x8200` 起**（约 **180 KB**） |
| **Day 1–2** | 只会打印文字，**不读盘** | 还没有 |
| **Day 3 起** | **真 IPL**：读盘，最后 **`JMP 0x8200`** | **被 IPL 搬进内存** 的 OS 数据 |

记两句话：

1. **IPL = 512 字节的「搬运工」** — 自己先被 BIOS 跑起来。  
2. **bootpack = 「货」** — 不是 512 B，是后面那 ~180 KB 的 OS 本体。

### `INT 0x13` 是什么？

它**既不是 IPL，也不是 bootpack**。

- **`INT 0x13`** = **BIOS 磁盘中断**，固件提供的「读软盘扇区」服务。  
- OS 还没加载、还在 **16 位实模式** 时，代码**不能直接操作磁盘硬件**，只能 **`INT 0x13`** 让 BIOS 代读。  
- Day 3 里 **`AH = 0x02`** 表示读扇区；IPL 里的 **`readloop` 反复调用它**，把 bootpack **一块块（每块 512 B）** 从软盘读进 **`0x8200` 起** 的内存。  
- 读失败时用 **`AH = 0x00`** 复位软驱，**单扇区最多重试 5 次**（详见 [§3.1.3](./section-3.1.3-INT0x13与ipl代码拆解.md)）。

```text
BIOS 读 512B IPL → 0x7C00 执行（搬运工上岗）
       ↓
IPL 循环 INT 0x13（AH=0x02）→ 软盘 S2 起读入 bootpack → 内存 0x8200
       ↓
JMP 0x8200 → bootpack / nasmhead 继续（见 [§3.2](./section-3.2-纸娃娃操作系统.md)）
```

**别和硬盘 MBR 搞混：** 网上常把「磁盘最前 512 B + 分区表」也叫引导 — 那是 **PC 硬盘** 说法。本书 Day 3 用的是 **1.44 MB 软盘**：512 B 启动扇区就是 **IPL**，**没有**「先读 MBR 分区表再找系统盘」这一步，直接从 **A 软驱、S2 起** 读 bootpack。

### 软盘镜像里：`0x00` 填充 ≠ bootpack

Day 2 拼 **1.44 MB** 软盘时（[§2.3](../../day-02-asm-makefile/notes/section-2.3-先制作启动区.md)），典型做法是：

1. 先造 **1,474,560 B** 的空白底稿（内容全是 **`0x00`**）  
2. 把 **`ipl.bin`（512 B）** 贴到**文件最开头**（偏移 0）

```text
helloos.img / haribote.img（1.44 MB 文件布局）

偏移 0        512 B          约 180 KB              剩余 …
┌──────────┬─────────────────────────────┬──────────────┐
│  IPL     │  Day2 时全是 0x00（占位）    │  仍是 0x00   │
│  55 AA   │  ← IPL 要从这里开始读        │              │
└──────────┴─────────────────────────────┴──────────────┘
  S1         S2 起 …                      未用到
```

**三个容易混的点：**

| 误解 | 正解 |
|------|------|
| **`0x55AA` 后面就是 bootpack** | **`0x55AA` 只在第 1 扇区里**（第 510、511 字节）。bootpack 在**下一个扇区**（文件偏移 **512** 起） |
| **镜像里那些 `0x00` 就是 bootpack** | **不是。** Day 2 的 `0x00` 只是**占位/空白**，表示「这里还没写 OS 本体」 |
| **读 10 个扇区** | 本书读的是 **10 个柱面**（C0~C9），不是 10 扇区 — 约 **180 KB**（见 [§3.1.2](./section-3.1.2-软盘CHS结构与读盘范围.md)） |

**QEMU 启动时实际发生什么：**

1. BIOS 只把**第 1 扇区 512 B** 读进内存 **`0x7C00`**，检查末尾 **`55 AA`**，然后执行 IPL。  
2. IPL 再 **`INT 0x13`**，从软盘 **S2、S3、S4 …** **一块块读进内存 `0x8200` 起**。  
3. 读的是**磁盘上真实存着的字节** — Day 2 若仍是 **`0x00`**，读进内存的也是 **`0x00`**（读盘可以成功，但 **`JMP 0x8200` 没有可执行代码**）。  
4. Day 3 往后：**把 bootpack 二进制写进镜像 S2 起**，用真实 OS 数据**替换掉原来的 `0x00` 占位**。

```text
Day 2（只有 IPL）          Day 3+（写入 bootpack 后）
磁盘 S2 起：0x00 0x00 …    磁盘 S2 起：bootpack 机器码 …
              ↓ INT 0x13 读                    ↓ INT 0x13 读
内存 0x8200：0x00 0x00 …    内存 0x8200：bootpack 机器码 …  → 能继续跑
```

> **`AH = 0x00`（INT 0x13 复位软驱）** 和 **磁盘里的字节 `0x00`** 是两码事 — 前者是 BIOS 命令号，后者是「空白占位」。
