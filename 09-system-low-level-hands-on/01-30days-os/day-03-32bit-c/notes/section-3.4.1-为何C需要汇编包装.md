## ④.1 为何 C 需要汇编包装？

C 是 **可移植的高级语言** — 编译器 **不会** 为每一种 CPU 特殊指令都提供语法。

| C 擅长 | C 搞不定（必须用汇编） |
|--------|------------------------|
| 算法、数据结构、业务逻辑 | **`HLT`** — CPU 休眠 |
| 函数调用、循环、条件 | **写 CR0 / CR3 / MSR** — 开保护模式、分页 |
| 大部分内存读写 | **切 16→32→64 位** 的关键指令序列 |
| | **`CLI/STI`** 等精确控制（裸机/OS 早期） |

**做法：** 汇编写 **独立函数** → 链接器与 C **拼成同一个 bootpack** → C 里 **`io_hlt();`** 像调普通函数。

```
bootpack.c              asmfunc.asm
    │                       │
    │  void io_hlt(void);   │  io_hlt:
    │  … io_hlt(); …        │      HLT
    └───────────┬───────────┘      RET
                ▼
           链接 → bootpack 二进制
```

| 步骤 | 谁干 |
|------|------|
| **汇编** | `nasm` → `.o`，含 `HLT`、`RET` 等 |
| **编译 C** | `gcc -m32 -c` → `.o`，含 `HariMain` |
| **链接** | `ld` 把 **`call io_hlt`** 的地址 **填成 asm 函数入口** |
