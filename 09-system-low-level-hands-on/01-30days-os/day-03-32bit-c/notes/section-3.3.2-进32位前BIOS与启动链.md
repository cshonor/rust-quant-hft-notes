## ③.2 进 32 位前：BIOS 与 Day 3 启动链

### 为什么要进 32 位？

| 16 位实模式 | 32 位保护模式 |
|-------------|----------------|
| 可用内存 **≈ 1MB 级** 限制 | **远大于 1MB** 的寻址空间 |
| 可直接 **`INT` 调 BIOS** | **不能再调 16 位 BIOS** |
| 汇编为主 | 适合 **C 写复杂逻辑** |

**关键约束：** 进 32 位 **之前**，必须在 **16 位实模式** 做完所有 **BIOS 依赖** — 详见 [§3.1.4](./section-3.1.4-实模式读盘与保护模式切换.md)。

---

### 为什么必须先在 16 位做完 BIOS 操作？

BIOS 中断服务全是 **8086 时代 16 位代码**。一旦切到 **32 位保护模式**：

- **不能再直接 `INT 0x10` / `INT 0x13`** 调 BIOS  
- 读盘、改显示、查键盘若还要做，只能 **自己写驱动** — **512 B 的 IPL 根本装不下**

所以：**能靠 BIOS 的，全在 16 位实模式一次性做完** → 再切换 → 之后用 **OS 自己的代码**（C + 少量汇编）。

---

### Day 3 启动链：四阶段

| 阶段 | 谁在做 | 模式 | BIOS 操作 | 你仓库里已有？ |
|------|--------|------|-----------|----------------|
| **① IPL** | [ipl.asm](../code/sec-3.1-ipl-int13-disk-load/ipl.asm) | **16 位实模式** | **`INT 0x13` 读 bootpack**（10 柱面 → `0x8200`） | **是** |
| **② nasmhead** | bootpack 开头汇编 | **仍 16 位** | **`INT 0x10` 切 VGA**、开 A20、建 GDT… | 见 [§3.2](./section-3.2-纸娃娃操作系统.md) |
| **③ 切保护模式** | nasmhead | 16→32 切换 | **此后不再调 BIOS** | bootpack 里做 |
| **④ HariMain** | `bootpack.c` | **32 位保护模式** | 无 — 用 C + 自写驱动 | [§3.4](./section-3.4-汇编与-C-的结合.md) |

**IPL 已在 16 位完成读 bootpack：**

```nasm
    MOV     AH, 0x02
    MOV     AL, 1
    INT     0x13
    …
    JMP     0x8200
```

**还没在 IPL 里做的**（也不该塞进 512 B）：**切图形模式**（`INT 0x10`）、**进 32 位** — 在 **nasmhead** 完成，然后才跑 **`HariMain`（C，32 位）**。

**进 32 位前 BIOS 清单：**

- **读盘载入 OS 本体** — **`INT 0x13`**（IPL 已做）  
- **设定画面模式** — **`INT 0x10`**（nasmhead，见 §3.2）  
- **键盘状态等** — 若需要，同样 **切 32 位前** 用 BIOS 做完
