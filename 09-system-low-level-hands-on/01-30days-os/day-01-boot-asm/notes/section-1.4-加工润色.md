## ④ 加工润色 · `$`、`TIMES` 与启动区签名

**自动填充：** NASM 里用 **`TIMES 510-($-$$) DB 0`** 让汇编器 **自动填 `0x00`**，把 **第 510–511 字节** 留给启动魔数 **`55 AA`**（x86 **引导扇区有效标志**）。原书 nask 用 **`$`** 表达同一思想；NASM 用 **`$`（当前地址）** 和 **`$$`（段起始）** 做算术。

```
软盘 / 映像布局（引导扇区，共 512 字节）
┌──────────────────────────────────────┬──────┐
│  IPL 代码 + 数据（TIMES 自动填 0）     │55 AA │  ← 字节 510–511
└──────────────────────────────────────┴──────┘
  字节 0                              511
```

改输出文字时，只动源码里的字符串 — **不必手算** 后面要填多少 `0x00`，也不必在 HxD 里一格一格补 hex。这是 **NASM 相对昨天手工编辑器的第一次显身手**。

编译：

```bash
nasm -f bin helloos.nasm -o helloos.img
```
