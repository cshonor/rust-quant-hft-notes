## ③ .hrb 文件头 · 装载数据段

#### 现象

C app 调 API **显示字符串仍失败** — 代码段进了内存，**`.rodata`/字符串还在文件里没进数据段**。

#### `.hrb` 格式（原书）

| 区域 | 说明 |
|------|------|
| **文件头 36 字节** | 魔数 **「Hari」** 等标识 |
| 头内字段 | **数据段大小** 等装载参数 |
| 后续 | 代码 + 数据 blob |

**OS `file_loadfile` 升级：**

1. 读头 → 解析 **datasize**
2. **MEMMAN 分配** 正确大小 **数据段**
3. **复制数据** 到 app **1004 段** 对应内存
4. 更新 **`0xfe8` 等基址**（Day 21）

```
.hrb 文件
┌──────────┬─────────┬──────────┐
│ 36B 头   │  code   │  data    │
│ Hari+size│         │ 字符串…  │
└──────────┴─────────┴──────────┘
         load → 1003 CS + 1004 DS 各就各位
```

**里程碑：** **C 编译产物（含数据）** 与 **OS loader** 契约固定 — 类似 **ELF PT_LOAD** 的迷你版。
