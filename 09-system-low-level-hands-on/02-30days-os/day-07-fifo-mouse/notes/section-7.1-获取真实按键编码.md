## ① 获取真实按键编码 · 中断必须快

Day 6 能 **响应中断**，还不知道 **按了哪键**。

#### 读键盘数据端口

| 端口 | 作用 |
|------|------|
| **`0x0060`** | 键盘控制器 **数据端口** — 读 **扫描码（scancode）** |

| 动作 | 示例码（原书） |
|------|----------------|
| 按下 **A** | **`0x1E`** |
| 松开 **A** | **`0x9E`** |

#### 底层铁律：ISR 越短越好

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| 在中断里 **putstr / 画屏** | ISR：**读端口 → 写入缓冲 → 立刻 `IRETD`** |
| 中断占用长 → **卡顿、丢中断** | **HariMain** 主循环空闲时 **慢慢显示/处理** |

```
键按下 ──► IRQ ──► ISR: 读 0x60 → 丢进 FIFO ──► 返回
                      │
HariMain ◄────────────┘ 轮询/取 FIFO → sprintf → 屏幕
```

**HFT 对照：** 网卡 **NAPI / 收包 ISR 只 enqueue**、策略线程 ** dequeue 算信号** — 同一 **「中断短、业务慢」** 分工。
