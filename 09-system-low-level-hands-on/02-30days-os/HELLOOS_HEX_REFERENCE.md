# Tolset · helloos 十六进制对照资料

> **原著配套：** 川合秀实《30 天自制操作系统》· tolset / `helloos` 工程  
> **本仓库：** HxD 替代 BZ.exe 时的对照表 + **可复制的 512 字节引导扇区**  
> **二进制文件：** [./day-01-boot-asm/code/](./day-01-boot-asm/code/)

## 一、背景

tolset 是原书配套工具包；`helloos` 是第一个软盘镜像工程。书中用 **BZ.exe** 二进制编辑器手工输入十六进制，本仓库用 **HxD** 完成同一任务。

两份核心对照：

1. **基础进制转换表**（0~F）
2. **`helloos.img` 原始字节表**（512 B 引导扇区 + 1.44 MB 全镜像布局）

### 关键偏移速查

| 偏移 | 字节 | 含义 |
|------|------|------|
| `0x000` | `EB 4E 90` | 短跳转 + NOP，跳过 BPB |
| `0x003` | `HELLOIPL` | OEM 名（8 字节） |
| `0x050` | `B8 00 00 …` | 程序入口（机器码） |
| `0x076` | `hello, world` | 显示字符串 |
| **`0x1FE`** | **`55 AA`** | **引导签名（必检）** |

## 四、完整 1.44 MB 镜像其余区段

| 区段 | 偏移（约） | 内容 |
|------|------------|------|
| 引导扇区 | `0x000`–`0x1FF` | 上表 512 字节 |
| FAT12 头 | `0x200` 起 | `F0 FF FF 00 00 00 00 00` + 填充至 9 扇区 FAT |
| 根目录 | 随后 | 全 `00`，224 项 |
| 数据区 | 其余 | 默认全 `00` |
| **总大小** | — | **1,474,560 B** = 2880 × 512 B |

本仓库 [helloos.img](./day-01-boot-asm/code/helloos.img) 已按 tolset 布局生成，**不必手填全盘**；手工实验只需保证 **引导扇区正确 + 全文件 1.44 MB**。

## 五、工具对照

### HxD（替代 BZ.exe）

| BZ | HxD |
|----|-----|
| 左栏地址 | **偏移 (h)** |
| 中间十六进制 | 中间 `00`–`0F` 列 |
| 右侧 ASCII | **对应文本** |
| 输入到指定地址 | `Ctrl+G` 跳转偏移 |

### NASM 源码 ↔ 十六进制

```nasm
DB 0xeb, 0x4e, 0x90     ; EB 4E 90
DB "HELLOIPL"           ; 48 45 4C 4C 4F 49 50 4C
; …
DB 0x55, 0xaa           ; 偏移 0x1FE
```

**汇编 ↔ 机器码详解**（含 `MOV AX,0x0123` → `B8 23 01` 与 `0x050` 起逐条对照）→ [section 1.3](./day-01-boot-asm/notes/section-1.3-初次体验汇编程序.md#汇编--机器码昨天那些-hex-从哪来)。

### 易错点

→ 完整条目见 [section 1.1.6 错误报告](./day-01-boot-asm/notes/section-1.1.6-启动链路与排错.md#day-1-错误报告条目规范)

| # | 要点 |
|---|------|
| 1 | 全镜像 **1,474,560 B**，不能只留 512 B |
| 2 | **`55 AA`** 在 **`0x1FE`**，不在文件最末尾 |
| 3 | 改 **hex 区** 第一字节 **`EB 4E 90`**，偏移列是标尺 |

- **`0x1FE` 必须是 `55 AA`**，否则 BIOS 不认启动盘
- FAT 媒体描述符 **`0xF0`**；总扇区 **`0x0B40`**（2880）
- 空白区填 **`00`**，不能留「空」
