## ②.1 VGA 模式 0x13 详解（320×200×256 色）

> **别和 `INT 0x13` 搞混：** `INT 0x13` = **磁盘中断**（读软盘）；**模式 0x13** = **显示模式号**（320×200 图形）。数字都是 0x13，功能完全无关 → [Day 3 README 易混点表](../README.md#易混点int-0x13-和模式-0x13-不是一回事)。

**`0x13`** = **BIOS VGA 图形模式号**（常写作 **Mode 13h**）。在 **16 位实模式** 下用 **`INT 0x10`** 切换；切完后向 **`0xA0000`** 帧缓冲 **写字节** 即可画点 — 纸娃娃 OS 在 [nasmhead.asm](../code/sec-3.4-bootpack-asm-and-c/nasmhead.asm) 里完成切换，Day 4 在 C 里写显存。

---

### 1. 基础定义

| 属性 | 说明 |
|------|------|
| **模式号** | **`AL = 0x13`**，`AH = 0x00` → **`INT 0x10`** |
| **分辨率** | **320 × 200** 像素 |
| **色彩** | **256 色调色板模式**（每像素 **8 bit**，存的是 **色号 0～255**，不是 RGB 三元组） |
| **显存基址** | 线性 **`0xA0000`**；实模式段地址 **`0xA000`**（即 **`0xA000:0000`**） |
| **像素存储** | **1 字节 = 1 个调色板索引**；硬件查调色板 DAC 输出屏幕颜色 |

笔记里写的 **「320×200×8」** = **320×200 分辨率、每像素 8 位** = **256 色索引模式**，与 **「256 色」** 是同一模式的不同说法。

---

### 2. 0～255 = 256 个「颜色编号」（不是黑→白刻度）

VRAM 里 **1 字节** 存的不是 RGB，也不是亮度，而是 **调色板索引（色号）**：

```text
你写：  vram[i] = 7
意思：  这个像素用「7 号颜色」
不是：  亮度 = 7/255，也不是从黑到白的第 7 格
```

| 概念 | 说明 |
|------|------|
| **0～255** | 共 **256 个编号**，每个编号是调色板表里 **一项的门牌号** |
| **查表** | 硬件用编号去 **调色板（Palette）** 查 RGB，再送到屏幕 |
| **「256 色」** | 最多可同时定义 **256 种不同颜色**（由调色板内容决定） |

**常见误解：** 编号越大越白、0=黑、255=白 — **错。** 默认 BIOS 调色板里各号颜色是 **固定搭配**（红、绿、蓝、黄…），不是按数字大小渐变。

| 色号（默认调色板示意） | 常见效果 |
|------------------------|----------|
| **0** | 黑（Day 3 `vram[i]=0` → 全黑屏） |
| **1** | 往往是 **深蓝**，不是白 |
| **15** | 偏 **白 / 亮灰**（原书常用作「白」） |
| **255** | 具体颜色 **看调色板**，不一定是白 |

若要 **256 级灰度** 或自定义红绿蓝，要在 Day 4 [§4.3 调色板](../../day-04-c-graphics/notes/section-4.3-调色板Palette与色号.md) 用端口 **`0x03c8` / `0x03c9`** 改写调色板表 — 那时 **0～255 才可以** 被你定义成想要的任意颜色。

```text
VRAM 字节（0~255 编号）  ──索引──►  调色板[编号]  ──RGB──►  屏幕像素
```

---

### 3. 显存寻址规则

**总像素：** 320 × 200 = **64000** 字节 — 刚好落在 **64 KB** VGA 段（`0xA0000`～`0xAFFFF`）内。

像素坐标 **`(x, y)`**（`x`: 0～319，`y`: 0～199）：

```text
offset   = y × 320 + x
线性地址 = 0xA0000 + offset
实模式   = 0xA000:offset   （段基址 0xA0000 = 0xA000 << 4）
```

向该地址 **写 1 字节色号**，屏幕 **`(x, y)`** 显示调色板里对应颜色。

**C（32 位平坦地址，haribote）：**

```c
volatile unsigned char *vram = (volatile unsigned char *)0xA0000;
vram[y * 320 + x] = color;   /* color: 0~255 色号 */
```

**汇编（实模式段寄存器写法，示意）：**

```nasm
mov     ax, 0xa000
mov     es, ax
; es:bx → 0xA0000 + bx
mov     byte [es:bx], 15
```

---

### 4. 为什么《30 天自制操作系统》用 0x13

| 原因 | 说明 |
|------|------|
| **实模式原生** | 上电 / IPL 阶段 CPU 在 **16 位实模式**，BIOS **自带** VGA 模式表，**无需自写显卡驱动** |
| **寻址简单** | 帧缓冲 **固定映射 `0xA0000`**，无现代 EDID / 多分辨率配置 |
| **门槛极低** | **写字节 = 画点**；适合手写引导 + 最早用 C 画图形 |
| **原书验收** | 切模式后 **`HariMain` 填 0** → **全黑屏** = bootpack 已执行（[§3.2](./section-3.2-纸娃娃操作系统.md)） |

**注意：** 切 **0x13** 必须在 **还能调 BIOS 的 16 位阶段** 完成（nasmhead 前半）；进 **32 位保护模式** 后 **不能再 `INT 0x10`** — 见 [§3.3.2](./section-3.3.2-进32位前BIOS与启动链.md)。

---

### 5. 图形模式 0x13 vs 文本模式 0x03

两套 **完全不同的显存布局** — 写错地址会「有输出但不对」或花屏。

| | **模式 0x03（文本）** | **模式 0x13（图形）** |
|--|----------------------|----------------------|
| **画面** | **80×25** 字符 | **320×200** 像素 |
| **显存起点** | **`0xB8000`** | **`0xA0000`** |
| **单元格式** | **2 字节/字符**（ASCII + 属性） | **1 字节/像素**（调色板色号） |
| **典型用途** | Day 1 **`INT 0x10` 打印 `load done`** | Day 3+ **图形 GUI** |
| **切换** | `mov ax, 0x0003` + `int 0x10` | `mov ax, 0x0013` + `int 0x10` |

```text
Day 1 IPL 消息     →  文本 0xB8000  （BIOS 仍在文本模式）
nasmhead 切 0x13  →  图形 0xA0000  （之后 HariMain 画点）
```

[Day 3 minimal 示例](../code/sec-3.4-minimal-16-to-32-call-c/kernel_main.c) 写 **`0xB8000`** 是 **教学用文本 demo**，**不是** 原书 0x13 图形路径。

### 6. 切换模式：汇编示例

**进入 VGA 0x13（本仓库 [nasmhead.asm](../code/sec-3.4-bootpack-asm-and-c/nasmhead.asm)）：**

```nasm
        mov     ax, 0x0013
        int     0x10
        ; 此后可操作 0xA0000 帧缓冲
```

**恢复文本 0x03（调试 / 退出时用，原书后期少用）：**

```nasm
        mov     ax, 0x0003
        int     0x10
```

| 寄存器 | 进入 0x13 | 恢复 0x03 |
|--------|-----------|-----------|
| **AX** | **`0x0013`**（`AH=0`, `AL=13`） | **`0x0003`** |
| **中断** | **`INT 0x10`** | **`INT 0x10`** |
