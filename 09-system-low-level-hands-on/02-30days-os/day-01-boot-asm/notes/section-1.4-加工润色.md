## ④ 加工润色 · NASM 替你填字节

原书「加工润色」= 不再跟 **二进制格子** 较劲，把 **凑 512 字节、写启动魔数、算偏移** 交给汇编器。

昨天 [1.1 HxD](./section-1.1-先动手操作.md)：自己数「还差多少个 `00`」、**`Ctrl+G` 跳到 `0x1FE`** 填 **`55 AA`**。  
今天 [1.3 NASM](./section-1.3-初次体验汇编程序.md)：同样效果写在源码里，**编译出的机器码与手工版一致**。

---

### 三个核心技巧

#### ① `TIMES 510-($-$$) DB 0` — 自动填零到第 510 字节

| 符号 | 含义 |
|------|------|
| **`$$`** | 本段（引导扇区）的 **起始地址** — 配合 `ORG 0x7C00` 时用于算 **段内偏移** |
| **`$`** | **当前** 已生成到的地址（「写指针」停在哪） |
| **`$-$$`** | 从段起点到现在，已经写了 **多少字节**（代码 + 数据） |
| **`510 - ($-$$)`** | 距离第 510 字节还差多少 → **用 `0x00` 填满** |

BIOS 规定引导扇区 **共 512 字节**：**前 510 字节** 放代码/数据（不足用 `0` 垫），**最后 2 字节** 必须是启动签名。

```
偏移:  0x000 ─────────────── 0x1FD (509) │ 0x1FE  0x1FF
       │  代码 + 字符串 + TIMES 填 0      │ 55     AA
       └──────── 共 510 字节 ─────────────┴── 启动魔数 ─┘
```

改 `hello, world` 字符串长短时，**不必重算**「后面还要敲几个 `00`」— `TIMES` 每次编译自动对齐。

#### ② `$` / `$$` — 自动算偏移，不用 HxD 一格一格数

| 昨天 HxD | 今天 NASM |
|----------|-----------|
| 看左侧偏移列，`0x076` 是字符串 | 写 `msg:` 标签，`MOV SI, msg` |
| 手算 `BE 74 7C`（`0x7C74` = `0x7C00+0x74`） | `ORG 0x7C00` 后标签地址由汇编器算 |
| 改一行代码，后面全盘 hex 可能错位 | 改源码，**重新 `nasm` 一次** |

**重点：** 你写的是 **逻辑（标签、指令）**；**段内偏移、跳转相对距离、立即数** 由 NASM 编码 — 这就是「加工润色」省下来的脑力。

#### ③ 启动魔数 `0xAA55` — 自动写到第 511、512 字节

两种写法等价（小端序：低字节在前）：

```nasm
        DB      0x55, 0xAA      ; 显式：偏移 0x1FE=55, 0x1FF=AA
; 或
        DW      0xAA55          ; 一个 16 位字 → 内存里仍是 55 AA
```

**对，不用再去 HxD 里 `Ctrl+G` 定位 `1FE`。** 编译后 **最后两个字节一定是 `55 AA`**，BIOS 才认「这是可启动扇区」。

→ 与 [1.1.4 启动签名](./section-1.1.4-启动签名与自检.md) 同一规则；昨天手工填的 **`0x1FE`** 位置，今天由源码最后两行保证。

### 示例：带「润色」的 `helloos.asm` 骨架

```nasm
        ORG     0x7C00

        MOV     AX, 0
        MOV     SS, AX
        MOV     SP, 0x7C00
        ; … 打印 hello, world 的循环 …

        TIMES   510-($-$$) DB 0
        DW      0xAA55
```

编译：

```cmd
nasm -f bin helloos.asm -o ipl.bin -l helloos.lst
```

用 **`helloos.lst`** 对照 [HELLOOS_HEX_REFERENCE](../../HELLOOS_HEX_REFERENCE.md) 或昨天 HxD 内容 — **应逐字节相同**。

### 512 字节 vs 1.44 MB：NASM 只生成哪一部分？

这是和昨天最容易混的一点：

| | 大小 | 是什么 |
|---|------|--------|
| **引导扇区** | **512 字节** | 软盘 **第一个扇区**；BIOS 只检查这里的代码 + **`55 AA`** |
| **完整软盘映像** | **1,474,560 字节** = **1.44 MB** | 标准 3.5 寸软盘；[1.1.2](./section-1.1.2-HxD界面与新建映像.md) 在 HxD 里建的就是这个 |

```
1.44 MB 软盘 helloos.img（1,474,560 B）
┌────────────────────────────────┬──────────────────────────────────┐
│ 扇区 0：引导扇区 512 B          │ 其余扇区（FAT / 根目录 / 数据区） │
│ NASM -f bin 直接产出这一块       │ 默认填 0；后面 Day 放 OS 本体     │
│ 偏移 0x000 – 0x1FF              │ 偏移 0x200 – 文件末尾            │
└────────────────────────────────┴──────────────────────────────────┘
```

- **`nasm -f bin helloos.asm -o ipl.bin`** → 通常得到 **正好 512 B** 的 **`ipl.bin`**（纯二进制启动区，**不是** 1.44 MB）。
- **`ipl.bin` 最后两字节必须是 `55 AA`** — BIOS 靠这对魔数判断「有效启动区」；源码里用 `DB 0x55,0xAA` 或 `DW 0xAA55`（见上文 §③）。
- 昨天你在 HxD 里做的是：**先建 1.44 MB 全零文件**，再在 **最开头 512 字节** 里填代码和 `55 AA` — 其余保持 `00`。
- **今天：** 先用 NASM 得到 **`ipl.bin`（512 B）**，再 **拼到 1.44 MB 软盘映像开头** → 就是 BIOS / QEMU 能认的 **可启动镜像**。

**做法 A（继续用 HxD，最快对照）**  
1. 仍保留 [code/helloos.img](../code/helloos.img) 那种 **1,474,560 B** 模板（或 HxD 新建 1440 KB）。  
2. 用 NASM 生成 `ipl.bin`（512 B），把 **512 字节覆盖到映像偏移 0**（HxD 打开 `ipl.bin` 全选复制，贴到 `helloos.img` 开头）。

**做法 B（命令行拼盘，Day 2 Makefile 会用）**  
在 **MSYS2 / Git Bash** 等环境：

```bash
# 生成 1.44MB 全零底稿，再把 512B 引导扇区写到开头
dd if=/dev/zero of=helloos.img bs=512 count=2880
dd if=ipl.bin of=helloos.img conv=notrunc
```

（`2880 × 512 = 1,474,560`）

**QEMU 启动：**

```cmd
qemu-system-i386 -fda helloos.img -boot a
```

需要 **完整 1.44 MB 映像**（或至少 BIOS 能识别的软盘格式）；**不能**只有 512 B 空文件就指望当整盘用 — 除非仅做 hex 对照实验。

### 昨天 vs 今天 · 一张表

| 步骤 | 昨天 HxD | 今天 NASM「加工润色」 |
|------|----------|----------------------|
| 写指令 | 手敲 `B8 00 00` … | 写 `MOV AX, 0` |
| 凑 510 字节 | 自己数还要多少 `00` | `TIMES 510-($-$$) DB 0` |
| 启动魔数 | `Ctrl+G` → `1FE` → `55 AA` | `DW 0xAA55` 或 `DB 0x55,0xAA` |
| 算字符串地址 | 看偏移列 `0x076` | 标签 `msg:` + `ORG` |
| 产出 | 直接 1.44 MB `helloos.img` | 先 **512 B** `ipl.bin`，再 **嵌入** 1.44 MB |
| 验证 | QEMU 出字 | `helloos.lst` 与昨天 hex **逐字节对比** + QEMU |

**你要记住的一句：** NASM 自动处理的是 **最开头这 512 字节引导扇区**；**1.44 MB** 是 **载体大小**，后面空间预留给 OS，现在可以全是 `0`。

### 编译后自检

- [ ] `ipl.bin` 大小 **512 字节**（或与你源码 `ORG`/布局一致）
- [ ] HxD 打开 `ipl.bin`，偏移 **`0x1FE`** 为 **`55 AA`**
- [ ] `helloos.lst` 中 **`0050` 起** 与 [1.1.3](./section-1.1.3-写入引导扇区机器码.md) 机器码一致
