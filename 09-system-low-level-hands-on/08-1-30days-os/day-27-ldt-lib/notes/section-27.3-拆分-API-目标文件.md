## ③ 拆分 API 目标文件 · 应用程序瘦身

#### 问题

**`a_nask.nas`** 整包链进 **每个 .hrb** → **`hello3` 极简** 也 **~520B**（未用 API 汇编仍在）。

#### 解法

**每个 API 函数 → 独立 `.obj`** — 链接器 **只拉被 CALL 的符号**。

| 之前 | 之后 |
|------|------|
| 全量 `a_nask` | **按需 link** |
| hello3 **520B** | **显著减小**（原书对比） |

**死代码消除** 在 **链接期** — 与 **`-ffunction-sections --gc-sections`** 思想一致。

→ [Day 20 `_api_*`](../day-20-api/)
