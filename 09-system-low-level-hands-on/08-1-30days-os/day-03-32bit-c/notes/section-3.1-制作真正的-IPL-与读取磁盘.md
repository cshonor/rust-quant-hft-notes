## ① 制作真正的 IPL 与读取磁盘

Day 1–2 的 IPL **只显示文字**，**没有装载 OS 本体**。从 Day 3 起，IPL **名副其实**。

#### 软盘物理结构（原书）

| 概念 | 含义 |
|------|------|
| **柱面（Cylinder）** | 盘片上一圈同心磁道 |
| **磁头（Head）** | 上下盘面各一读头 |
| **扇区（Sector）** | 磁道上最小读写单位（常 **512 B**） |

读盘靠 **BIOS `INT 0x13`** — 在 **16 位实模式** 下调用固件例程。

#### 可靠性 · 重试 5 次

软盘 **易读错** → 代码里 **失败重试最多 5 次**，再报错/放弃。

#### 循环读入

用 **循环** 按柱面/扇区读，最终把软盘上 **10 个柱面（约 180KB）** 的数据 **完整读入内存** — 这就是 **bootpack（OS 本体）** 的装载过程。

```
ipl.bin (512B @ 0x7c00)
    │
    │  INT 0x13 循环读盘
    ▼
内存中的 bootpack（~180KB）
```

**HFT 对照：** 像 **从磁盘/NVMe  mmap 策略库** — 启动链上 **「谁读、读多少、读失败怎么办」** 必须显式写清；Linux 里由 bootloader + 内核完成。
