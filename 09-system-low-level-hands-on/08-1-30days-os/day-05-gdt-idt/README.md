# Day 5 · 结构体、文字显示与 GDT/IDT


> **原书第五章** · 界面丰满 + **底层基石** — `BOOTINFO`、自绘文字/鼠标、**GDT / IDT** 初始化。

---

### 本节四段结构

| 段 | 做什么 | 带走什么 |
|----|--------|----------|
| **① 结构体** | **`BOOTINFO`** + **`->`** | 启动参数 **不写死** 分辨率/显存 |
| **② 自绘文字** | 8×16 点阵 + 字体 + **`sprintf`** | **告别 BIOS**，屏幕 **printf 调试** |
| **③ 鼠标指针** | 16×16 位图 + 透明 | 箭头光标上屏 |
| **④ GDT / IDT** | 分段 + 中断向量表 | **内存保护** 与 **硬件事件** 入口 |

## 本日小结

| 问题 | 答案 |
|------|------|
| 启动信息怎么灵活？ | **`BOOTINFO` struct** + 固定内存传参 |
| 文字谁画？ | **8×16 点阵** 写 VRAM，**不用 BIOS** |
| 怎么 debug？ | **`sprintf` + 屏幕输出** `%d` / `%x` |
| 鼠标图标？ | **16×16** bitmap + **透明** |
| GDT 干什么？ | **段描述符表** — 内存 **界限与权限** |
| IDT 干什么？ | **0~255 中断 → 处理函数** — **硬件并发** |

**承前启后：** 外观（字、鼠标图）+ **GDT/IDT 环境就绪** → Day 6 **鼠标真正动起来**。
