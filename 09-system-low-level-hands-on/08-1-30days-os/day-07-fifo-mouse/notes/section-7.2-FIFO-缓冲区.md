## ② FIFO 缓冲区 · 从排队到环形

#### 问题：单变量不够

| 场景 | 后果 |
|------|------|
| 连按 / 键鼠同时 | 主程序还没读，**下一中断覆盖** 上一字节 |
| 只用一个 **`unsigned char buf`** | **丢键、丢鼠标包** |

#### 第一版 FIFO：移位（慢）

每次 **读走队头**，后面数据 **全体前移一格** — **O(n) 搬运**，CPU 浪费。

#### 改良：环形缓冲区（Ring buffer）

| 指针 | 作用 |
|------|------|
| **写指针（wp）** | 中断 **写入** 位置 |
| **读指针（rp）** | 主程序 **读出** 位置 |

写满末尾 → **指针绕回开头**（覆盖 **已读、可复用** 的空间）— **不移动数据**，**O(1)**。

```
      rp          wp
       ▼           ▼
buf [·|已读|数据数据|空|·]
       └─写绕回──► 环形
```

封装为通用结构 **`FIFO8`**（8 位数据、固定容量）— **键盘一条、鼠标一条**，随时 **`Fifo8_Put` / `Fifo8_Get`**。

**HFT：** **SPSC/MPSC ring buffer**（行情、订单、日志）— 本 OS 课是 **最简 8 位单生产者单消费者** 原型。
