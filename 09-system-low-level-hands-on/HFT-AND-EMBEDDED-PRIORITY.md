# HFT / 嵌入式主线 · 学什么、先搁置什么

> **结论（定死）：** **C → MikanOS（02）→ Linux 内核 / DPDK → HFT**。《30 天自制 OS》（01）**彻底后置**，可当休闲拓展，**不必系统敲码**。  
> 与 [LEARNING-CHAIN](../../LEARNING-CHAIN.md) 对齐；09 模块内 **HFT 走 02，不走 01 先行**。

---

## 〇、为什么前后说法会矛盾？

仓库里曾写 **「01 启蒙 → 02 MikanOS」** 或 **「先 haribote Day 3 再 MikanOS」** — 那条路 **只适用于**：

| 适用人群 | 特征 |
|----------|------|
| **纯零基础** | 完全不懂计算机底层 |
| **目标** | 只想入门 OS 原理，建立笼统认知 |
| **职业** | **不做** 量化 / 服务器底层 / 嵌入式 Linux 开发 |

**为什么那条路对他们是合理的：**

- 《30 天自制 OS》门槛低，16 位 BIOS 逻辑直观，上手快；
- 能快速建立「操作系统怎么启动」的 **故事线**。

**缺点（对你无效）：**

- 软盘、`INT 0x13`、16 位实模式 — **与现代 x86-64、UEFI、Linux 体系割裂**；
- 对 HFT / 嵌入式主线 **复用极低**，还容易 **混淆** 64 位寄存器与调用约定。

---

## 一、你的专属路线：跳过 30 天，直接 MikanOS

**目标：** HFT 低延迟 + 嵌入式 Linux 飞控 — 需求完全不同。

| 维度 | **02 MikanOS** | **01 30 天 haribote** |
|------|----------------|------------------------|
| 架构 | **UEFI + x86-64 长模式** — 现代 PC 标准 | BIOS 软盘 + 16 位实模式 → 32 位保护模式 |
| 语言 | **C/C++ 为主**，少量 64 位 asm | 大量 16 位 asm + 32 位 nask |
| 核心知识 | **64 位分页、特权级、syscall、内存模型** | GDT 分段、BIOS 中断、软盘 CHS |
| 工具链 | 交叉编译、Makefile、**LLD** — 工业通用 | 专用 nask、软盘映像 — **淘汰路径** |
| 与主线打通 | **CSAPP · Linux 内核 · DPDK** 直接对照 | 需「翻译」才能接到 64 位 Linux |

**MikanOS 对你当下的四点价值：**

1. 全程现代架构，知识点与 CSAPP、LKD、DPDK **同一条线**；
2. 主体 C/C++，与你 **正在主攻的 C** 无缝衔接；
3. 分页、特权级、syscall — **高频优化刚需底层**；
4. 工具链以后写 C++/Rust 底层、FFI **直接复用**。

**30 天自制 OS 的定位（对你）：**

- **彻底后置** — 休闲拓展，翻翻笔记即可；
- **不必** 系统敲代码、不必跟完 Day 1–30；
- 若已写了一点 Day 3 IPL — **可以停**，不亏；宏观记住「上电 → boot → C 内核」就够。

## 二、最终固定学习顺序（HFT 主线）

```text
1. 当前首要：吃透 C 语言
   语法、指针、内存、结构体
   ↔ 01-CSAPP · 08-TLPI（可与下步并行）

2. C 基础达标后：直接学 MikanOS（02）
   UEFI、长模式、页表、syscall、APIC
   ↔ 02-mikan-os/ · 与 CSAPP Ch9、LKD 对照

3. MikanOS 推进中 / 完成后：
   Linux 内核基础 · 性能调优 · DPDK
   ↔ 05-LKD · 03-SysPerf · 14-DPDK → 15-HFT

4. 《30 天自制 OS》（01）：彻底后置
   可选休闲阅读；HFT 路线不要求敲 haribote 代码

5. 嵌入式 Linux（另线，延后）
   ARM64、设备树、U-Boot — 飞控阶段再开
```

**与通用零基础路线对比：**

```text
通用零基础：  01 30天（启蒙） → 02 MikanOS → Linux …
HFT 你的路线： C 扎实 → 02 MikanOS → Linux/DPDK/HFT → 01 可选后置
```

## 三、现阶段核心：优先吃透 C

| 理由 | 说明 |
|------|------|
| **工业栈都以 C 为根** | Linux 内核、DPDK、低延迟网络、嵌入式驱动 |
| **MikanOS 入口就是 C/C++** | 不必先写 512 B 引导扇区 |
| **HFT 离不开 C 底子** | 反汇编、FFI、缓存与内存模型 |

**暂时搁置：** 所有引导扇区、16 位实模式汇编、`INT 0x13` 软盘读写。

**在本仓库：**

- **主投入：** [01-CSAPP-3rd](../../01-CSAPP-3rd/) · [08-TLPI](../../08-The-Linux-Programming-Interface/) · [02-mikan-os](./02-mikan-os/)
- **01 haribote：** 已有笔记可当参考；**新精力不要往里投**

## 四、汇编只学两类（对你有用）

| 类型 | 何时学 | 用途 |
|------|--------|------|
| **x86-64（CSAPP）** | **紧接 C** | 反汇编、缓存、流水线、延迟毛刺 — HFT 刚需 |
| **ARM64** | 嵌入式飞控阶段 | 设备树、驱动里少量 asm |

| | haribote 16 位 | MikanOS 64 位 | CSAPP x86-64 |
|--|----------------|---------------|--------------|
| HFT 路线 | **不学 / 后置** | **主攻** | **主攻** |
| 典型场景 | 已淘汰的 BIOS 软盘 | 现代 UEFI 启动链 | perf、热路径分析 |

## 五、若你已经在写 Day 3 haribote

| 情况 | 建议 |
|------|------|
| 刚跑通 IPL、黑屏 | **够了** — 知道「汇编搭台、C 唱戏」即可 **收工** |
| 纠结 GDT / `INT 0x13` | **停** — MikanOS 用 UEFI，不走这套 |
| Day 4 指针写 VRAM | 概念有用（mmap 思维），但 **TLPI + CSAPP 更高效** |
| 舍不得沉没成本 | Day 3 笔记留着；**下一小时投给 C 或 MikanOS Ch0–1** |
