## 2. Ext2 磁盘数据结构 (Disk Data Structures)

> 分区划分为 **块组 (Block Groups)** — 局部性、缩短寻道

---

### 一、为何分块组

| 目的 | 说明 |
|------|------|
| **局部性** | 相关 inode / 数据块 **同组** → 磁头移动少 |
| **备份** | 超级块、组描述符 **多组冗余** |

除 **块组 0 的引导扇区** 外，各组 **内部布局规律相同**。

---

### 二、块组内布局（概念顺序）

| 区域 | 内容 |
|------|------|
| **超级块 (Superblock)** | FS 全局：块大小、inode 总数、空闲块数 … |
| **组描述符 (Group Descriptors)** | 各块组统计与位置 |
| **数据块位图** | 0/1 标记块是否空闲 |
| **inode 位图** | 0/1 标记 inode 是否空闲 |
| **inode 表** | 连续块存 **全部 inode**（Ext2 磁盘 inode **128 字节**） |
| **数据块** | 文件/目录内容 |

**备份：** 超级块与组描述符在 **其他块组有副本** — 防损坏；内核正常用 **块组 0 主副本**。

→ VFS superblock/inode：[Ch 12 section-3](../chapter-12-VFS/notes/section-3-四大核心对象.md)

---

### 三、磁盘 inode 内容（概要）

每个 inode：**类型、权限、大小、时间戳、数据块指针 `i_block[]`** 等。

---

### 四、扩展属性与 ACL

128 字节 inode **装不下** 复杂元数据时：

- **扩展属性** 存于 **额外磁盘块**  
- Linux 2.6 借此支持 **POSIX ACL** — 比「属主/组/其他」更细粒度权限  

→ 用户态：[08 TLPI](../../../08-The-Linux-Programming-Interface/)
