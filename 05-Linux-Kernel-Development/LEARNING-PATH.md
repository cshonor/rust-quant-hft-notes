## 一、为什么三门课放在同一总文件夹

| 子目录 | 角色 |
|--------|------|
| `01_Course_LFS` | 建立「整机构建」上下文 |
| `02_Course_Kernel_7Lectures` | 建立「写内核代码」手感 |
| `03_Course_Kernel_Architecture` | 建立「五大子系统」理论地图 |
| `00_Book_3rd_Notes` | **官方体系化主线** — Love 第三版 |

视频 = 前置预习；书本 = 核心规范与源码细节。二者互补，**不是看完课就可以不读书**。

## 二、三门课分别扫清哪些阅读障碍

### （1）LFS → 编译、系统上下文、用户态/内核态

| 视频建立的能力 | 对应书本章节 |
|----------------|-------------|
| 完整编译内核、根文件系统、glibc 配合 | Ch 2 内核入门、`menuconfig`、镜像与模块安装 |
| 用户程序 vs 内核代码边界 | Ch 5 系统调用 |
| 「操作系统如何跑起来」的整体图景 | Ch 1 简介 |

### （2）内核编程视频 · 6 集内核编程 → 模块、进程、中断前半段

| 视频建立的能力 | 对应书本章节 |
|----------------|-------------|
| 编写/加载/卸载 LKM、模块生命周期 | Ch 17 设备与模块 |
| 中断处理、Oops、KGDB | Ch 7–8 中断与下半部 · Ch 18 调试 |
| 编译报错、模块签名等踩坑经验 | Ch 2、Ch 17 注意事项 |

### （3）内核架构视频 · Complete Kernel Guide → 五大子系统理论

| 视频建立的能力 | 对应书本章节 | HFT 关联 |
|----------------|-------------|----------|
| 虚拟内存、HugePage、NUMA、页表、缺页 | Ch 12、Ch 15 | 衔接 [03-Gorman](../06-Linux-Virtual-Memory-Manager/) |
| 网络栈分层、UDP 收发包、NAPI/软中断 | 书本网络概述 + [06-Rosen](../10-Linux-Kernel-Networking/) | 行情/订单路径 |
| 调度器、RT 进程、spinlock/mutex/RCU | Ch 4、Ch 9–10 | 绑核、无锁、抖动 |

## 三、阅读体验对比

| 路径 | 体验 |
|------|------|
| 直接啃书 | 名词/API/流程陌生，每页查资料，易放弃 |
| **先课后书** | 书本变成系统化梳理；视频零散点被串联；晦涩章节省力 |

## 四、推荐三步闭环

1. **三门课完整学习** — 在 `01`/`02`/`03` 写完配套笔记
2. **通读 LKD 第三版** — 在 `00_Book_3rd_Notes` 按 [OUTLINE](./00_Book_3rd_Notes/OUTLINE.md) 补充书本独有知识点
3. **合并回主线笔记** — 视频里没有的规范细节、边界场景写入 `00_Book_3rd_Notes`

## 五、HFT 额外收益

三门课 + 书本重叠的 **虚拟内存、内核网络、调度、中断**，正是低延迟调优最关键的内核机制。先课后书，能更快筛出 HFT 相关章节：

| HFT 主题 | 书本 | 视频铺垫 |
|----------|------|----------|
| 绑核 / RT 调度 | LKD Ch 4 | 03 架构课 · 调度 |
| 中断 / softirq 抖动 | LKD Ch 7–8 | 02 实操 · 03 架构 · NAPI |
| NUMA / 大页 | LKD Ch 12 + 03 Gorman | 03 架构 · VM |
| 无锁 / RCU | LKD Ch 9–10 | 03 架构 · 并发 |

→ 工程师落地：[12-HFT-Low-Latency-Practice](../12-HFT-Low-Latency-Practice/)
