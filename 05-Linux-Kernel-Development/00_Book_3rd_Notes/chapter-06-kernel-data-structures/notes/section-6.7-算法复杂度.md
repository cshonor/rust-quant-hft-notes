## ⑥ 算法复杂度 · Big-O

内核开发者需 **定量** 想扩展性 — **大 O 记法** 描述输入规模 **n** 增长时性能趋势。

| 记号 | 名称 | 内核中常见于 |
|------|------|--------------|
| **O(1)** | 常数 | `list_add`、`list_del` |
| **O(log n)** | 对数 | `rbtree` 查找/插入 |
| **O(n)** | 线性 | `list_for_each` 扫全表 |
| **O(n²)** 及以上 | 应极力避免 | 大 **n** 时不可接受 |

| 实践 | 说明 |
|------|------|
| 看 **典型 n** | 链表 O(n) 扫 **10 项** 可接受；**百万项** 不行 |
| 热路径 | 中断、调度器 — 倾向 **O(1) / O(log n)** |
| 冷路径 | 初始化、罕见 ioctl — 可略宽松 |

**HFT 对照：** 用户态订单簿用 **红黑树/跳表 O(log n)**；内核 CFS 同理 — **算法层与业务层同一套复杂度语言**。
