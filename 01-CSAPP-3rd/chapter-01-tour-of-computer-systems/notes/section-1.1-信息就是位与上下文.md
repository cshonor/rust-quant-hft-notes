## 1.1 信息就是位 + 上下文

### 核心概念

- **一切信息在机器里都是比特（0/1）** — 不同「上下文」决定这些比特被解释成指令、整数、浮点、字符还是指针。
- **源文件即文本字节序列** — `hello.c` 在磁盘上就是 ASCII/UTF-8 字节；编译器读入后按 C 语法解析。

| 上下文 | 同一串比特可能被读成 |
|--------|---------------------|
| 编译器 | C 源码 token |
| CPU | 机器指令 |
| 内存 | `int` / `char*` / 指令流 |
| 网络 | TCP 载荷 |

### 程序 = 比特 + 上下文

```
磁盘上的 hello.c（文本）
  → 编译后变成机器码比特
  → 加载到内存后 CPU 当指令执行
  → printf 输出又是另一段字符比特
```

**HFT：** 行情二进制协议、SBE/Fix 字段 — 本质也是 **比特 + 约定好的 layout（上下文）**。解析错 offset = 上下文错，和 C 里把 `float*` 当 `int*` 一样危险。
