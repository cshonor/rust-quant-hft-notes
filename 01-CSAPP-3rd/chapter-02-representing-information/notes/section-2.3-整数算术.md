## 2.3 整数算术（2.3.1–2.3.8）

### 2.3.1 无符号加法

- 结果 **模 2^w** — 溢出 = 丢弃进位
- 检测：`sum < a`（无 unsigned wraparound 时）

### 2.3.2 补码加法

- 位级与无符号加法 **相同电路**；溢出当 **有符号** 解释时才重要
- **溢出条件（同号相加得异号）：**  
  `(s > 0 && t > 0 && u < 0) || (s < 0 && t < 0 && u > 0)`

**HFT：** 累计成交量、序号用 **uint64** + 显式检查 wrap；有符号 tick 差分注意 INT_MIN 边界。

### 2.3.3 补码的非

- **−t** = **~t + 1**（w 位）
- INT_MIN 取负 **仍是自己**（8 位 −128）

### 2.3.4–2.3.5 无符号与补码乘法

- 硬件 **截断到 w 位** — 等价模 2^w（无符号）或补码语义（有符号）
- 编译器优化：常数乘法 → 移位 + 加减（→ 2.3.6）

### 2.3.6 乘以常数

- `x * (2^k ± c)` 可优化为 shift/add — 理解 [Ch 3](../../chapter-03-machine-level-programs/) / [Ch 5](../../chapter-05-优化程序性能.md) 里的 strength reduction

### 2.3.7 除以 2 的幂

- **无符号 >> k** — 逻辑右移，等价 `/ 2^k`
- **有符号 >> k** — 应实现 **算术右移**（向 −∞ 舍入 toward −∞ 的除法语义在 C 中由实现保证算术右移时等价 floor division for negative）

**坑：** `-1 >> 1` 期望 −1 还是 0？在主流 gcc/clang x86 上为 −1。

### 2.3.8 关于整数算术的最终思考

| 原则 | 说明 |
|------|------|
| 计算机算术 ≠ 数学算术 | 固定宽度 + 溢出 |
| 无符号环绕有时 **故意使用** | 哈希、checksum、环形序号 |
| 安全关键路径 | 用更大宽度检查、Rust 的 `checked_*`、Sanitizer |

**HFT 环形 buffer 序号：** 常 `uint32_t` 序号模 2³²，比较时用 **序列空间 half-range** 规则（与 TCP 序号类似），不能简单 `(a < b)` 若跨 wrap 边界。

→ 安全整型实践：策略层 Rust；C++ 用 `-fsanitize=undefined` 测 overflow
