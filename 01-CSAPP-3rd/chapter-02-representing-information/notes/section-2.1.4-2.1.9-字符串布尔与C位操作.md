## 2.1.4–2.1.9 字符串、代码、布尔与 C 位/逻辑/移位

### 2.1.4 字符串表示

- C 字符串 = **字节序列 + 末尾 `'\0'` (0x00)**
- **ASCII** — 英文子集；**UTF-8** — 变长编码，兼容 ASCII 前缀
- 二进制协议里 **不是 C 字符串** — 常为定长字段或 `length + payload`，无 `\0`

**HFT：** 交易所 symbol、account 可能是 **定长 8/16 字节** 空格填充；用 `strcmp` 前确认是否以 `\0` 结尾。

### 2.1.5 代码表示

- **机器码也是字节序列** — 与数据无本质区别，靠 **上下文（PC 指向、页权限 X/D）** 区分
- 安全：数据段不可执行（NX/DEP）、W^X — 防 shellcode

### 2.1.6 布尔代数简介

- 与 / 或 / 非 / 异或 — 对应 `& | ~ ^`
- 德摩根定律等 — 读电路与优化位掩码时有用（HFT 可略读）

### 2.1.7 C 语言中的位级操作

| 运算符 | 含义 |
|--------|------|
| `&` | 按位与 — **掩码** 取字段 |
| `\|` | 按位或 — **合并** 标志位 |
| `^` | 按位异或 — 切换位、简单 checksum |
| `~` | 按位取反 |
| `<<` `>>` | 左/右移 |

```c
// 从 word 中取出第 k 位
int get_bit(unsigned x, int k) { return (x >> k) & 1; }

// 设置第 k 位为 1
unsigned set_bit(unsigned x, int k) { return x | (1u << k); }
```

**HFT：** 解码 SBE/自定义 header — **shift + mask** 是热路径基本动作；Rust 里等价 `>> &` 常由编译器优化为单条指令。

### 2.1.8 C 语言中的逻辑操作

- `&&` `||` `!` — **短路求值**，返回 0/1，**不**按位作用于全体 bit
- 别混淆：`x & y` vs `x && y`

### 2.1.9 C 语言中的移位操作

- **左移 `<<`：** 低位补 0；无符号等价 **× 2^k**（溢出则截断）
- **右移 `>>`：**
  - 无符号：**逻辑右移**，高位补 0
  - 有符号：实现定义 **算术右移**（补符号位）— C 标准对负数右移留坑，关键代码用无符号或显式分支

**HFT：** 价格 **tick 整数化** 常用 `price_raw / tick_size` 或定点缩放，优先整数运算避免 2.4 浮点坑。
