## 2.6 实例分析：ARM Cortex-A53 与 Intel Core i7-6700

两个 **设计目标截然不同** 的真实处理器，展示同一套层次思想的不同取舍。

---

### ARM Cortex-A53 — 移动 / 嵌入式 (PMD)

| 维度 | 设计选择 |
|------|----------|
| **定位** | 能效优先，非峰值性能 |
| **缓存** | 相对 **简单的两级** cache |
| **TLB** | **微型 TLB**，面积与功耗约束 |
| **预取** | 保守 — 省电、可预测 |

**启示：** 在功耗预算内，用 **足够但不过度** 的层次结构服务 PMD workload。

| HFT 视角 |
|----------|
| 一般不用于低延迟实盘；理解 **「简单层次 + 小 TLB」** 在边缘/网关设备上的行为 |

---

### Intel Core i7-6700 — 桌面 / 服务器

| 维度 | 设计选择 |
|------|----------|
| **定位** | 高性能，复杂 **乱序** 流水线 |
| **缓存** | **三级** — L1/L2 per core，**L3 共享** |
| **预取** | **极其激进** 的硬件预取 |
| **非阻塞** | 深度配合乱序执行 |

**启示：** 为常见桌面/服务器负载（顺序扫描、规则 stride）优化 AMAT；非常规访问可能 **预取适得其反**。

| HFT 视角 |
|----------|
| **colo 服务器常见类似层次** — L3 共享意味着多策略/多线程 **争用 LLC** |
| 绑核 + 减少跨核共享数据 → 降低 L3 争用与 false sharing |
| 压测时换 **同代不同 SKU**（L3 大小不同）可能显著改变 replay 性能 |
| 与 [14-Systems-Performance Ch7](../../../15-Systems-Performance-2nd/chapter-07-memory/) 的 NUMA/TLB 观测对照 |

---

### 对照总表

| | Cortex-A53 | Core i7-6700 |
|---|------------|--------------|
| 目标 | 功耗 | 性能 |
| Cache 级数 | ~2 级 | 3 级 + 共享 L3 |
| TLB | 小 | 大 |
| 预取 | 保守 | 激进 |
| 典型场景 | 手机、IoT | 桌面、交易服务器 |
