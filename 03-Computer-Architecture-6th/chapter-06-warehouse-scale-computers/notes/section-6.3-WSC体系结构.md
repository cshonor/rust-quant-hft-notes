## 6.3 仓储级计算机的体系结构

### 网络层次

```
服务器 → ToR（机架顶交换机）→ 阵列级交换机 → L3 边界路由器 → 互联网
```

| 层级 | 角色 |
|------|------|
| **Rack** | 数十台服务器共享 ToR |
| **Array/Pod** | 多机架聚合 |
| **Cluster/WSC** | 全设施 |

**Clos / Fat-tree** 等拓扑提供 **高对分带宽**（→ 6.7 Jupiter）。

---

### WSC 的「内存层次」

并非只有 DRAM — **数据放置** 决定延迟数量级：

| 访问 | 相对延迟（直觉） |
|------|------------------|
| 本机 DRAM | 1× |
| 同机架 | 略高 |
| **跨机架 / 跨阵列** | 可达本地 DRAM 的 **~1/300** 带宽/延迟恶化 |

**软件必须：** 分片、本地化、减少跨 rack 聊天。

| HFT 视角 |
|----------|
| colo：**策略、行情、发单同机** — 避免跨机 TCP |
| 多机部署时：**按品种/通道分片**，而非任意 RPC |
| 与 [Ch5 NUMA](../chapter-05-thread-level-parallelism/notes/section-5.1-引言与多处理器挑战.md) 同构 — **「近」永远赢** |
| 行情组播、**同 rack 订阅** — 网络拓扑即性能（→ [13-DPDK](../../../14-DPDK-Low-Latency-Network/)） |
