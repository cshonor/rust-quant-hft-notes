## 6.7 综合运用：谷歌的仓储级计算机

### 能效设计

| 措施 | 目的 |
|------|------|
| **蒸发冷却** | 利用湿球温度，减 CRAC 依赖 |
| **更高运行温度** | 减冷却能耗 |
| **机架级分布式 DC UPS** | 替代巨型集中 UPS，提高效率 |

**结果：** 业界领先 **PUE** 水平（随代际演进）。

---

### Jupiter 网络

| 特点 | 说明 |
|------|------|
| **Clos 拓扑** | 多级交换 |
| **高对分带宽** | 机架间海量东西向流量 |
| **容错** | 路径冗余 |

支撑 Google 内部 **MapReduce、Bigtable、Spanner** 等全设施流量。

| HFT 视角 |
|----------|
| Jupiter 优化 **TB 级批流** 与 **全局服务** — 不是 **μs 级单播撮合** |
| 可借鉴：**拓扑规划、无单点**；不可照搬：**分布式存储做订单簿** |
| 定制网络芯片思路与 **SmartNIC/DPDK** 同属「网络不能成瓶颈」（不同尺度） |
