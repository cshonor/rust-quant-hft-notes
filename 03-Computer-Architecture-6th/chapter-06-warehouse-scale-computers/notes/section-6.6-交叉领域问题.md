## 6.6 交叉领域问题

### 防止网络成为瓶颈

| 趋势 | 对策 |
|------|------|
| 流量指数增长 | **商用芯片** 定制交换、Clos 拓扑 |
| 成本控制 | **集中式 SDN** 式路由控制 |
| 对分带宽 | 多级交换避免 oversubscription 过高 |

WSC 网络是 **第一等公民** — 与计算、存储并列。

### 能量比例 (Energy Proportionality)

**理想：** 负载 10% → 功耗 ~10%。

| 现实 | 优化 |
|------|------|
| 空载仍耗电 | CPU **DVFS**、核心关断 |
| PSU 效率曲线 | 选用高效电源、合理负载点 |

**与 WSC：** 低利用率时仍付 PUE 税 — 驱动 **混部批处理** 填谷。

| HFT 视角 |
|----------|
| 实盘机常 **近满载** 热核 — 能效曲线在 **高负载区** 选型 |
| **空闲关核**（非热核）可省电；**热核** 宁可满载换延迟稳定 |
| 批处理任务调度到 **夜间/低峰** — 类比 AWS Spot |
