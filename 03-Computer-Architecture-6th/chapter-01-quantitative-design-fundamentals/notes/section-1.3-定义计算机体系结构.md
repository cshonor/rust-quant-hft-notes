## 1.3 定义计算机体系结构

架构师的工作 **不限于 ISA**，而是三层叠加：

```
指令集架构 (ISA)     ← 程序员可见的指令、寄存器、内存模型
      ↓
计算机组织 (微架构)   ← 流水线、缓存、分支预测、乱序执行
      ↓
硬件实现             ← 工艺、版图、时钟、物理设计
```

### 设计约束

在满足功能的前提下，在多维约束间 **权衡**：

| 约束 | 问题 |
|------|------|
| **价格** | 芯片面积、产量、学习曲线 |
| **功耗 / 能量** | 散热、机房电费、电池（PMD） |
| **性能** | 响应时间、吞吐 |
| **可用性 / 可靠性** | MTTF、冗余、修复时间 |

**要点：** 同一 ISA 可有多种微架构实现（如 x86 的不同代 CPU）；换 CPU 可能 **ISA 不变、微架构大变** — 程序行为（尤其是时序、cache）会变。

| HFT 视角 |
|----------|
| 你写的 C++/Go **针对 ISA + ABI**；延迟调优针对 **微架构 + OS**（cache 行、NUMA、prefetch） |
| 选型时问：**这颗 CPU 的微架构对我的 hot loop 友好吗？**（L3 大小、内存通道、AVX 宽度） |
| 「换同 ISA 新 CPU」≠ 性能线性提升 — 要 **重新 profile** |
