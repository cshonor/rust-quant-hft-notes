## 1.10–1.12 综合运用与谬误陷阱

### 1.10 综合运用 (Putting It All Together)

通过 **真实服务器配置** 与基准（如 **SPECpower**）对比，展示 **性能 × 价格 × 功耗** 三维权衡。

| 维度 | 要问的问题 |
|------|------------|
| 性能 | 在我的 workload 下 wall-clock 多少？ |
| 价格 | $/性能 或 $/瓦特是否可接受？ |
| 功耗 | 机房散热、电费、能否 sustained turbo？ |

**案例直觉（Dell PowerEdge 类服务器）：** 更高配置不一定线性更快 — 要看瓶颈在 CPU、内存带宽还是 I/O。

| HFT 视角 |
|----------|
| colo 机器选型：**replay 基准 + 功耗墙** 同看 |
| 「加核」对单线程热路径可能 **零收益** — Amdahl + 绑核策略 |

### 1.11–1.12 谬误与陷阱 (Fallacies and Pitfalls)

| 谬误 | 真相 |
|------|------|
| **峰值性能 ≈ 实际性能** | 差距极大；峰值 marketing 数字少用于工程决策 |
| **硬件增强一定同时提高能效** | 并不绝对；更高 IPC/频率可能增功耗 |
| **只看单一指标** | MIPS、核数、GHz 都不能代替端到端时间 |
| **忽略常见情况** | 优化冷门路径浪费面积与复杂度 |
| **假设内存与 CPU 同速增长** | 带宽-延迟鸿沟持续存在 |

| HFT 视角 |
|----------|
| 网卡 **线速 Gbps** ≠ 你的策略 **tick-to-trade** |
| 「新 CPU 代数」要 **同 trace 重测**；勿外推 SPEC 比值 |
| 低延迟优化常 **增功耗**（busy poll、全核满载）— 要算 **$/纳秒** 与散热 |
