## 1.7–1.8 可靠性与性能量化

### 1.7 可靠性 (Dependability)

| 指标 | 含义 |
|------|------|
| **MTTF** | Mean Time To Failure — 平均故障间隔时间 |
| **FIT** | Failures In Time — 每 **十亿小时** 的故障数（MTTF 的倒数量级） |
| **MTTR** | Mean Time To Repair — 平均修复时间 |
| **可用性** | 系统可提供服务的时间比例；与 MTTF、MTTR 相关 |

**关系直觉：** 可用性 ↑ ← MTTF ↑ 或 MTTR ↓（冗余、热备、快速 failover）。

| HFT 视角 |
|----------|
| 交易所对接：**双机热备、链路冗余** — 可用性直接等于「能否下单」 |
| 单点故障（网卡、交换机、进程）要有 **检测 + 切换时间** 预算 |
| 「软件 bug」不算 FIT 里的硬件故障，但 **同等影响可用性** — 要靠测试与风控 |

### 1.8 性能评估

**黄金法则：**

> **执行时间（响应时间）** 是唯一一致、可靠的性能衡量标准。

| 误区 | 正确做法 |
|------|----------|
| 只看峰值 MIPS / FLOPS | 测 **真实 workload** 的 wall-clock |
| 只看平均延迟 | HFT 必看 **尾延迟 (P99/P999)** |
| 算术平均 SPECRatio | 多基准汇总用 **几何平均数 (Geometric Mean)** |

**常见基准家族：**

| 系列 | 用途 |
|------|------|
| **SPEC CPU** | 通用计算性能（整型/浮点） |
| **SPECpower** | 性能-功耗权衡（服务器能效） |
| **TPC** | 事务处理（数据库类负载） |

**几何平均为何重要：** 算术平均会被「某一基准暴涨」拉高，掩盖短板；几何平均更反映 **乘法意义上的整体比**。

| HFT 视角 |
|----------|
| 自有 benchmark：**端到端 tick-to-trade**、**replay 回放** — 不用 SPEC 代替 |
| 对比两台机器：固定 **相同输入 trace、相同内核/网卡配置** |
| 报告 **中位数 + P99 + max**；几何平均可用于 **多场景 replay 套件** 汇总 |
