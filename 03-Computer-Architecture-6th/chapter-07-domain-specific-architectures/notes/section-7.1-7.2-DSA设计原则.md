## 7.1–7.2 DSA 的设计指导原则

### 背景

| 趋势 | 后果 |
|------|------|
| **摩尔定律放缓** | 晶体管增长变慢 |
| **Dennard 终结** | 不能靠提频白嫖性能 |
| **通用 CPU 单线程停滞** | 转向 **领域专用** |

**DSA 定义：** 为 **某一计算领域** 高度优化；其他任务仍跑在 **通用处理器** 上。

---

### 五大设计原则

#### 1. 专用内存，最小化数据移动

| 通用 CPU | DSA |
|----------|-----|
| 复杂 cache 层次 | **Scratchpad**（软件管理暂存器内存） |
| 不可预测 miss | **可预测** 数据放置 |

**能耗事实：** 数据移动往往比 **算术** 更费电。

#### 2. 资源投入算术与片上存储

摒弃 **乱序、SMT** 等大面积微架构 → 换成 **成百上千简单 MAC/ALU**。

#### 3. 用最简单的并行形式

若 **2D SIMD** 或 **VLIW** 够用的领域，不用复杂多线程。

#### 4. 缩减数据大小与类型

| 手段 | 效果 |
|------|------|
| INT8/INT16 代替 FP32/64 | 带宽、算力密度 ↑ |
| 领域足够精度即可 | **内存受限** 领域收益巨大 |

#### 5. 领域语言 (DSL)

TensorFlow、Halide 等 — 编译/映射到 DSA 硬件。

| HFT 视角 |
|----------|
| **固定格式行情解码** = 窄类型 + 规则并行 — 类似 DSA 思维，即使用 CPU 也宜 **表驱动 + SIMD** |
| **Scratchpad 直觉**：mlock 热数据、hugepage、**无 page fault** — 追求 **确定性延迟**（与 TPU 单线程确定性同源） |
| FPGA 行情 filter：**专用逻辑** 换通用分支（→ 7.5 Catapult） |
| 别在 tick 路径盲目 **INT8 量化** — 金融精度与合规先验证 |
