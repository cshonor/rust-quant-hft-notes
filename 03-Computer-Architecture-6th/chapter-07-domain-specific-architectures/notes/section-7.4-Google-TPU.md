## 7.4 Google 张量处理单元 (TPU)

### 定位

| 属性 | 说明 |
|------|------|
| **形态** | 数据中心 **DNN 推理** ASIC |
| **非目标** | 通用 CPU 替代、灵活训练 |

---

### 微架构要点

| 组件 | 规模/特点 |
|------|-----------|
| **脉动阵列 (Systolic array)** | **65,536 个 8-bit MAC** |
| **片上统一缓冲区** | 最高 **24 MiB** scratchpad 风格 |
| **指令风格** | **CISC**、**单线程**、**确定性** 执行 |
| **数据类型** | **INT8** 推理为主 |

**脉动阵列直觉：** 数据在 PE 网格中 **节律流动**，乘加在本地完成，减少访存。

---

### 为何适合云端推理

| 需求 | TPU 匹配 |
|------|----------|
| 高吞吐推理 | 巨大 MAC 阵列 |
| **99% 尾延迟** | **无乱序/无 SMT** — 行为可预测 |
| 能效 | 砍掉通用机制 |

| HFT 视角 |
|----------|
| TPU 优化 **批量推理 SLA** — 与 HFT **单 tick 超低延迟** 不同问题域 |
| 可借鉴：**确定性执行模型**（少推测、少共享）对 **尾延迟** 的价值 — 实盘 CPU 也关 HT/省电扰动 |
| INT8 量化推理 ≠ 交易价格精度；**风控模型** 若上生产需单独验证 |
