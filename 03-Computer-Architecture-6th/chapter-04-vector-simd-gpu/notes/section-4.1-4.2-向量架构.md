## 4.1–4.2 向量架构

### DLP 与向量指令

**数据级并行（DLP）**：一条指令对 **多个数据元素** 施加相同操作。

**向量架构** 典型流程：

```
内存 → 加载向量寄存器 → 向量 ALU 运算 → 写回内存
```

通过 **流水线化向量单元** 与 **大寄存器文件** 隐藏内存延迟、吃满带宽。

---

### RISC-V 向量扩展 (RV64V) 关键机制

#### 多泳道 (Multiple Lanes)

| 概念 | 说明 |
|------|------|
| **Lane** | 并行流水线，每周期完成多个元素 |
| **峰值吞吐** | lanes × 每 lane 每周期元素数 |

#### 向量长度与条带挖掘 (Strip mining)

| 问题 | 解法 |
|------|------|
| 程序向量长度 ≠ 硬件最大向量长度 (mvl) | **向量长度寄存器 `vl`** 指定本次有效长度 |
| **条带挖掘** | 长循环分块：每轮处理 ≤ mvl 个元素，直至完成 |

#### 向量掩码 (Vector-Mask / Predicate)

| 用途 | 说明 |
|------|------|
| **条件循环向量化** | 谓词寄存器控制 **哪些元素** 真正执行/写回 |
| 避免分支 | `if` 在向量中变成 **掩码运算** |

#### 步幅、收集-散布

| 访问模式 | 指令/机制 |
|----------|-----------|
| **单位步幅** | 连续内存 — 最优 |
| **非单位步幅 (Stride)** | 多维数组列/行访问 |
| **收集 (Gather) / 散布 (Scatter)** | 索引向量间接寻址 — 稀疏矩阵 |

**硬件需求：** 多 **独立 memory bank** 支撑并行访存带宽。

| HFT 视角 |
|----------|
| 热路径宜 **连续、固定 stride** 批量处理（如一次解 N 档 bid/ask）— 类似条带挖掘 |
| **Gather/Scatter** 延迟高、带宽差 — 订单簿指针追逐 **不宜** SIMD 化 |
| RVV 思路可类比理解 **AVX-512 长度与掩码**（→ 4.3） |
