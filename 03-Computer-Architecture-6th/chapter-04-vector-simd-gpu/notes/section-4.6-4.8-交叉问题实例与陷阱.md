## 4.6–4.8 交叉领域问题、实例与陷阱

### 4.6 能耗与 DLP

| 策略 | 特点 |
|------|------|
| **快而窄** | 高频率乱序 CPU — 高单线程性能，功耗大 |
| **慢而宽** | 多泳道向量、降频降压 — **同吞吐下能耗可低一个数量级** |

**启示：** DLP 架构适合 **吞吐型** workload；与 HFT「低延迟单线程」目标常 **错位**。

---

### 4.7 实例：Tegra vs Tesla、Core i7 vs GTX 280

| 对比 | 要点 |
|------|------|
| **移动 GPU (Tegra)** vs **服务器 GPU (Tesla)** | 规模、带宽、双精度、功耗墙 |
| **Core i7** vs **GTX 280** | GPU 在 **计算密集 + 大工作集** 上峰值极高 |
| **Gather-Scatter** | 无硬件支持时 SIMD 潜力 **大打折扣** — 稀疏/间接访问是 GPU/AVX 共同软肋 |

| HFT 视角 |
|----------|
| 选型：**实盘 CPU 调延迟**，**研究集群 GPU 调吞吐** — 勿混为一谈 |
| i7 单线程低延迟 + 复杂分支仍常胜 GPU **小 batch** |

---

### 4.8 谬误与陷阱

| 谬误 | 真相 |
|------|------|
| **只看向量峰值性能** | **启动开销 (Start-up overhead)** — 短向量/短循环得不偿失 |
| **GPU 线程越多越好** | **散乱访存** 再多线程也救不了带宽；合并访问才是关键 |
| **SIMD = 总是更快** | 分支、依赖、对齐、gather 成本可抵消收益 |
| **忽略数据布局** | SoA vs AoS 决定能否向量化（→ Ch2） |

```

```
