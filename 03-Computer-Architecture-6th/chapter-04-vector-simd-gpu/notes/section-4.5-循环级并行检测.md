## 4.5 循环级并行的检测与增强

编译器向量化/SIMD 化的前提是找到 **可并行的循环迭代**。

### 循环携带依赖 (Loop-carried dependence)

| 概念 | 说明 |
|------|------|
| **迭代间依赖** | 第 `i` 次迭代的结果被第 `i+k` 次使用 |
| **无携带依赖** | 各迭代独立 → **可向量化** |

**例子：**

```c
// 可向量：每次迭代写不同 a[i]
for (i = 0; i < n; i++) a[i] = b[i] + c[i];

// 不可直接向量：归约
for (i = 0; i < n; i++) sum += a[i];
```

---

### 依赖检测：GCD 测试

对仿射访问下标 \(a \cdot i + b\) 与 \(c \cdot i + d\) 的依赖，用 **最大公约数 (GCD)** 等测试判断是否存在跨迭代冲突。

**编译器流水线：** 依赖分析 → 若安全则向量化 / 展开 / 软件流水线。

---

### 增强并行：标量扩展等

| 技术 | 作用 |
|------|------|
| **归约 (Reduction) 识别** | 特殊并行归约（树形加和） |
| **标量扩展** | 将部分依赖循环 transform 为可并行形式 |
| **循环交换 / 分块** | 改善访问模式（衔接 Ch2、4.2 条带挖掘） |

| HFT 视角 |
|----------|
| 手写热循环前自问：**迭代间有没有 RAW？** — 有则 SIMD 需掩码或改算法 |
| `#pragma omp simd` / 编译器报告 `-fopt-info-vec` — 看 **为何未向量化** |
| 归约（如全市场成交量累加）适合 **批量离线**，不适合单 tick 关键路径 |
