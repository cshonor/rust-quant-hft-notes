## 3.2 揭示 ILP 的基本编译器技术

### 循环展开 (Loop Unrolling)

| 做法 | 效果 |
|------|------|
| **复制循环体** 多次 | 减少 **分支次数**（每 N 次迭代才判一次） |
| 调整循环终止与归纳变量 | 暴露更多 **独立指令** 供调度 |
| 配合寄存器分配 | 减少循环内 RAW 链对流水线的束缚 |

**直觉：** 分支 = 控制冒险 + 预测压力；展开后编译器/CPU 有更大 **指令窗口** 做调度。

| 展开度 | 权衡 |
|--------|------|
| 过小 | 分支开销仍显著 |
| 过大 | **ICache 压力**、寄存器溢出到栈（反而变慢） |

| HFT 视角 |
|----------|
| 固定 tick 处理 **批量 N 档行情** 时，手动或编译器展开内层循环 — 常见于 checksum/解码 |
| `-O3 -funroll-loops` 对 **极热小循环** 可试；务必 **benchmark + 看 ICache miss** |
| 展开不能替代 **数据布局**（Ch2）— miss 时 ILP 无米下锅 |
