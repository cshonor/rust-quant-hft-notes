## 3.4–3.5 动态调度与 Tomasulo 算法

### 顺序 vs 乱序

| 模式 | 行为 |
|------|------|
| **顺序发射、顺序执行** | 遇 RAW 且无法旁路 → **停顿** |
| **动态调度** | **乱序执行、乱序完成** — 操作数就绪即可执行，不必等程序顺序上更早的慢指令 |

**限制：** 仍须保证 **数据流正确性**；WAR/WAW 需额外处理。

---

### Tomasulo 算法核心

```
指令 → 保留站 (Reservation Station) 等待操作数
     → 操作数到齐 → 执行
     → 结果经 CDB (Common Data Bus) 广播给所有等待该结果的保留站
```

| 机制 | 作用 |
|------|------|
| **保留站** | 缓冲未就绪指令，解耦发射与执行 |
| **寄存器重命名** | 动态消除 **WAR/WAW** — 逻辑寄存器映射到更多物理寄存器 |
| **CDB 广播** | 结果一产生即转发，减少 RAW 停顿 |

| HFT 视角 |
|----------|
| 你 **看不见** Tomasulo，但感受到：**独立指令可并行**、**长延迟 load** 阻塞后续依赖链 |
| **load 使用点** 尽量靠后排列（编译器调度）— 给硬件隐藏 miss 的窗口 |
| 重命名 **不解决** cache miss — Ch2 仍是第一战场 |
