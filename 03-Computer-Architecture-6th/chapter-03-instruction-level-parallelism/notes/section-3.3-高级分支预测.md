## 3.3 降低分支成本的高级分支预测

流水线加深 + **多发射** → 一次分支预测错误可浪费 **十余条甚至更多** 在飞指令 → **分支惩罚极高**。

### 动态分支预测演进

| 预测器 | 思路 |
|--------|------|
| **相关预测器 (Correlating)** | 利用 **其他分支历史** 预测当前分支 —「分支行为相关」 |
| **锦标赛预测器 (Tournament)** | **局部** vs **全局** 预测器 **自适应选择** 谁更准 |
| **标记混合 / TAGE** | 多表、不同历史长度；当前高性能 CPU 常用方案 |

**要点：** 预测的是 **方向**（跳/不跳）；取指还需 **目标地址**（→ 3.7 BTB）。

| 可预测 | 难预测 |
|--------|--------|
| 计数器 `% 2 == 0` | 数据依赖分支 `if (price > threshold)` |
| 极少走的 error 路径 | 哈希表、红黑树查找 |
| `likely`/`unlikely` 标注的冷路径 | 订单类型 switch 若分布均匀 |

| HFT 视角 |
|----------|
| `perf stat -e branches,branch-misses` — branch-miss 率应进 **回归测试** |
| 热路径：**无分支**（cmov、位运算、查表）、**分支外提**、**separate hot/cold 函数** |
| 状态机：用 **表驱动** 替代长 if-else 链 — 利于预测与 ICache |
| 别在 tick 热路径里打 log（隐含分支 + syscall） |
