## 3.11 多线程技术：开发线程级并行

当 **ILP 无法隐藏 cache miss 的长延迟** 时，可让 **多线程共享** 同一处理器的功能单元 — 从 ILP 过渡到 **TLP**（衔接 [Ch5](../../../chapter-05-thread-level-parallelism/)）。

| 模型 | 行为 |
|------|------|
| **粗粒度多线程** | 一线程 stall（如 miss）才切换 — 切换开销大 |
| **细粒度多线程** | 每个周期轮换线程 — 隐藏停顿，单线程变慢 |
| **SMT (同步多线程)** | 同一周期 **混合发射** 多线程指令；建基于乱序超标量 + 重命名 |

**SMT 要点：** 利用已有 **重命名与动态调度**，让功能单元在一线程 stall 时被 **其他线程** 使用。

| HFT 视角 |
|----------|
| **Intel 超线程 (HT)**：逻辑核共享物理执行单元 — 两线程争用 **ROB、LSU、cache** |
| 低延迟实盘常见做法：**BIOS/OS 关 HT**，**一物理核绑一关键线程** — 换 **尾延迟确定性** |
| 非关键任务（日志、监控）可放 **非隔离核** 或 SMT  sibling |
| SMT ≠ 多核并行 — [Ch5](../../../chapter-05-thread-level-parallelism/) 讲 **真多核** 与一致性 |
