## 3.6 基于硬件的推测

### 克服控制相关

**硬件推测 (Hardware speculation)**：

1. **预测** 分支方向  
2. 在分支 **未解决前** 按预测方向 **取指、执行**  
3. 预测错误 → **撤销** 推测路径上的状态  

### 重排序缓冲区 (ROB)

动态推测 = **分支预测 + 动态调度 + ROB**。

| 原则 | 说明 |
|------|------|
| **乱序执行** | 保留站/CDB 层面可乱序 |
| **顺序提交 (In-order commit)** | 仅当指令 **非推测** 且位于 ROB 队首逻辑顺序时，才更新 **架构状态**（寄存器/内存） |
| **推测错误** | 清空 ROB 中错误路径，恢复预测前状态 |
| **精确异常** | 顺序提交保证异常点 **可精确定位** — 对 OS 重要 |

**内存推测更谨慎：** store 通常需等退休/commit 规则满足才对外可见。

| HFT 视角 |
|----------|
| 分支预测失败 = **流水线清空** — 数十周期级惩罚，远大于一条加法 |
| 热路径 **减少不可预测分支** 比启用更激进 `-O3` 更有效 |
| 侧信道（Spectre 类）与推测执行有关 — 交易代码一般 **不处理不可信指针**；了解即可 |
| **精确异常** 对你写用户态策略影响小；内核/驱动相关 |
