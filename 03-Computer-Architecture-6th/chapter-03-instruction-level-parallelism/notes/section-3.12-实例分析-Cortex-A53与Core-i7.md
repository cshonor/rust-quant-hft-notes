## 3.12 实例分析：ARM Cortex-A53 与 Intel Core i7-6700

与 [Ch2 §2.6](../../chapter-02-memory-hierarchy-design/notes/section-2.6-实例分析-Cortex-A53与Core-i7.md) **同一对 CPU**，本章从 **ILP 视角** 对照。

---

### ARM Cortex-A53

| 维度 | 设计 |
|------|------|
| **定位** | 移动 / 嵌入式，**能效** |
| **执行** | **双发射**、**静态顺序** 执行（in-order） |
| **推测** | 无激进乱序推测 |
| **分支** | 相对简单预测 |

**ILP 策略：** 靠 **编译器** 在顺序核上排好指令；硬件简单、省电。

| HFT 视角 |
|----------|
| 非典型实盘 CPU；理解 **顺序核** — miss/依赖直接 stall，无 ROB 深推测 |

---

### Intel Core i7-6700

| 维度 | 设计 |
|------|------|
| **定位** | 桌面 / 服务器，**峰值性能** |
| **流水线** | **深**、乱序、推测 |
| **发射** | **多发射超标量** |
| **SMT** | 支持 **同步多线程**（每核 2 逻辑线程） |
| **分支** | **TAGE 类** 高级预测 + BTB |

**ILP 策略：** 硬件尽最大努力挖掘 ILP；功耗与面积换 IPC。

| HFT 视角 |
|----------|
| colo **Xeon/Core 同级** 思路类似 — 善用乱序但 **别触发 miss/分支风暴** |
| 压测对比：**关 SMT**、绑核、`taskset` — 观察 P99 变化 |
| 与 Ch2 合读：i7 = **激进 cache 预取 + 激进 ILP + SMT** |

---

### ILP 维度对照

| | Cortex-A53 | Core i7-6700 |
|---|------------|--------------|
| 发射 | 2-wide，顺序 | 4-wide，乱序 |
| 推测 | 弱 | 强（ROB） |
| SMT | 无 | 有 |
| 分支预测 | 简单 | TAGE + BTB |
| 适用 | PMD/IoT | 服务器/HFT 主机 |

---

### 本章小结

Ch3 链路：**相关性 → 冒险 → 编译器展开/调度 → 分支预测 → Tomasulo/重命名 → 推测+ROB → 多发射+BTB → SMT**。

**HFT 落地优先级：**

1. 减分支、提可预测性（3.1–3.3）  
2. 减 cache miss（Ch2）— 否则 ILP 空转  
3. 绑核、慎 SMT（3.11–3.12）
