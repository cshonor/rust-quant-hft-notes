## 3.7–3.9 多发射与高级指令交付

### 多发射 (Multiple Issue)

要使 **CPI < 1**，每周期须 **发射多条指令**。

| 类型 | 调度 | 代表 |
|------|------|------|
| **静态超标量** | 编译器决定发射包 | 部分嵌入式 |
| **VLIW / EPIC** | 编译器显式打包 | Itanium 路线 |
| **动态推测超标量** | 硬件运行时调度 | x86 Core 系列 |

**瓶颈：** 不仅执行单元要够，还要 **前端每周期供给足够指令**。

---

### BTB 与取指

| 组件 | 作用 |
|------|------|
| **BTB (Branch Target Buffer)** | 缓存分支 **目标地址** — 预测正确时 **零周期** 分支惩罚（与方向预测配合） |
| **集成取指单元** | 预取、对齐、处理 taken 分支后的取指流 |

---

### 实现权衡：ROB vs 物理寄存器重命名

现代高性能 CPU 常用 **大量物理寄存器 + 重命名**，ROB 管 **提交顺序**；细节因厂商而异。

**共同目标：** 在 **功耗/面积** 预算内最大化 **有效 ILP**。

| HFT 视角 |
|----------|
| **宽发射 + 深流水线** → 频率高但 **分支/miss 惩罚深** |
| i7 类：**4-wide 发射 + SMT** — 单线程仍可能吃不满发射宽度若分支/miss 多 |
| 调优目标：**让热基本块 IPC 接近 1+**，而非追逐峰值 GHz |
| ICache：热代码 **紧凑、内联克制** — 前端供给不及会 **stall fetch** |
