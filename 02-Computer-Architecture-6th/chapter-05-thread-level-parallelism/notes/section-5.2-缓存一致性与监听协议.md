## 5.2 缓存一致性与监听协议

### 缓存一致性问题 (Cache Coherence)

多核各自缓存 **同一物理地址** 的副本时：

- 一核 **写入** 后，他核若仍读旧缓存 → **错误**
- 需协议保证：对 **同一地址**， eventually 所有核看到 **一致** 的值

**注意：** Coherence 针对 **单地址**；多地址 **访问顺序** 见 5.6。

---

### 监听协议 (Snooping)

各核 **监听总线/互连** 上的一致性事务 — 典型用于 **SMP/UMA**。

**写失效 (Write Invalidate)** — 最常用：

1. 写前：使 **其他核** 中该 cache line 副本 **失效 (Invalidate)**
2. 写者获得 **独占 (Exclusive)**，再修改

读 miss 时从内存或其他核 **共享** 副本获取。

---

### 协议状态演进

| 协议 | 状态 | 优化点 |
|------|------|--------|
| **MSI** | Modified, Shared, Invalid | 基础三态 |
| **MESI** | + **Exclusive**（独占且干净） | 避免无谓写回 |
| **MOESI** | + **Owned**（所有者提供共享读） | 减少脏块写回内存 |

| 状态直觉 | 含义 |
|----------|------|
| **M** | 本核独占且已修改（脏） |
| **E** | 本核独占、未改（干净） |
| **S** | 多核只读共享 |
| **I** | 无效 |

| HFT 视角 |
|----------|
| 多核写 **不同变量** 若在同一 **cache line** → 反复失效（→ 5.3 伪共享） |
| **只读共享**（如策略配置表）MESI 的 S 态友好；**写多** 则 line 在核间 **乒乓** |
| 理解 MESI 后，无锁算法的 **cache line 所有权** 竞争更可解释 |
