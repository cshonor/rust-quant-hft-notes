## 4.3 多媒体 SIMD 指令集扩展

### 起源与实现

为加速 **图形、音频** 等 **窄数据类型**（8/16 位）：

- 在宽 ALU 上 **切分进位链**
- 一周期处理多个短数据 — **x86 MMX → SSE → AVX → AVX-512**

### 与「真」向量架构的三大差距（早期）

| 缺陷 | 后果 |
|------|------|
| **无向量长度寄存器** | 长度常编码在指令或固定 — 条带挖掘不灵活 |
| **无步幅 / 收集-散布** | 非连续访问难向量化 |
| **无掩码寄存器**（早期） | 条件分支难向量化；AVX-512 后改善 |

→ 编译器 **自动向量化困难**，长期依赖 **手写 intrinsics / 库**（如 Intel MKL、Eigen）。

### Roofline 性能模型

二维图：**性能屋顶** vs **算术强度 (Arithmetic Intensity)**

\[
\text{算术强度} = \frac{\text{浮点运算次数}}{\text{访问内存字节数}}
\]

| 区域 | 瓶颈 |
|------|------|
| 低算术强度 | **内存带宽**（斜线以下贴内存屋顶） |
| 高算术强度 | **峰值算力**（平顶） |

**用途：** 判断算法在特定 CPU/GPU 上 **算力 bound 还是带宽 bound**。

| HFT 视角 |
|----------|
| **批量回放**：对百万 tick 做统计/特征 — Roofline 判断该 **加核/AVX** 还是 **优化访存** |
| 固定格式行情解析：算术强度 **中等** — AVX 解多路 price/qty **可能划算** |
| 实盘单 tick：**强度极低**（几次运算/百字节）— 在 **内存延迟/分支** 侧，不在 SIMD 屋顶 |
