# Ch 27 场内交易与自动交易系统 · Floor Versus Automated Trading Systems

> **Trading and Exchanges** · Larry Harris · **精读** · Part VII

本章对比 **传统人工场内交易大厅 (Floor)** 与 **现代电子自动交易系统 (Automated)** — 技术进步虽使 **完全自动化** 可行，许多高流动交易所（如 **NYSE**）仍保留场内成分。分析两种结构的 **优劣势** 与 **为何共存**。

> **HFT 读者：** 你工作在 **自动系统极端**；本章解释 **为何 block / upstairs 仍要人声**、**audit trail 为何是监管资产**、以及 **电子化未消灭的信息交换需求**（→ dark、IOI、Ch 15）。与 [Ch 6 口头拍卖](./chapter-06-指令驱动市场.md)、[Ch 15](./chapter-15-大宗交易者.md)、[Ch 24](./chapter-24-专家做市商系统.md)、[Ch 26](./chapter-26-市场内与市场间的竞争.md) 衔接。

---

## 0. 对比总览

| 维度 | 自动 (Electronic) | 场内 (Floor) |
|------|-------------------|--------------|
| **公平性** | 规则程序化、一致 | 依赖交易员技能与诚实 |
| **审计** | 完美轨迹 | 手动记录、监管成本高 |
| **容量** | **可扩展** | 口头沟通 **有上限** |
| **访问** | **分布式**、桌面交易 | 须在场 |
| **速度** | 撮合 + 报送 **极快** | 喊价未必慢，但 **记录慢** |
| **信息交换** | 结构化、有限 | **微妙、声誉、试探** — 大宗优势 |
| **运营成本** | 建好后 **低** | **劳动力 + 房地产** 极高 |
| **主客群** | **小额零售、高活跃** | **大型机构、大宗** |

---

## 1. 公平性与审计轨迹 (Fairness and Audit Trails)

### 1.1 操作公平性 (Operational Fairness)

| 自动系统 | **严格、无例外** 执行程序规则 — 许多交易员认为 **最公平** |
|----------|----------------------------------------------------------|
| **场内口头拍卖** | 公平性依赖 **交易员技能与诚实** |
| **压力情境** | 市场活跃 / 剧烈波动时 — 难跟踪最优价；**违规** 可能发生 |

→ [Ch 6 open-outcry](./chapter-06-指令驱动市场.md) · [Ch 11 front-run 肢体语言](./chapter-11-指令预期者.md)

| HFT 视角 |
|----------|
| **Deterministic matching** — `orderbook.go` 的 **公平性价值** |
| **Latency fairness** 新争议 — co-lo vs 远程 **非同一类「公平」** |

### 1.2 审计轨迹 (Audit Trails)

| 自动 | **全部活动** 可生成 **完美审计轨迹** |
|------|--------------------------------------|
| **场内** | 须 **手动** 记录价、量、对手、时间 — **巨大监管监控成本** |

| HFT 视角 |
|----------|
| **CAT、OATS** — 监管依赖电子轨迹 |
| **Replay** 能力 — 纠纷与 **合规** 基础 |

---

## 2. 可扩展性与容量 (Capacity and Scalability)

| 电子撮合 | **可扩展 (Scalable)** — **同时** 处理海量指令交互 |
|----------|---------------------------------------------------|
| **口头拍卖** | 规模受限于交易员 **有效沟通** 能力 |
| **超载** | 过多交易员同池喊价 → **信息处理超载** → **混乱** |

| HFT 视角 |
|----------|
| **Message rate limits**、**matching engine 水平扩展** |
| **Flash crash** — 电子系统 **容量边界** 另一类问题（非口头混乱） |

---

## 3. 速度与分布式访问 (Speed and Distributed Access)

### 3.1 分布式访问

| 自动 | 办公桌前交易 — 电话、电脑、数据服务 |
|------|-------------------------------------|
| **优势** | 不必挤 **交易大厅**、携带笨重设备 |

→ **Co-location** = 分布式访问的 **极端优化**（仍非 floor）

### 3.2 执行与数据报送速度

| 声称 | 部分场内员称口头喊价 **快** |
|------|---------------------------|
| **事实** | 自动系统 **免去手动记录** → **成交闭环更快** |
| **报送** | 向公众报告 **报价/成交** — 自动 **更快、更准** |

| HFT 视角 |
|----------|
| **Microsecond matching + SIP/NBBO feed** — HFT 生态基础 |
| **Manual floor** 无法竞争 **零售小额高频** 段 |

---

## 4. 场内核心优势：信息交换 (Exchange of Information)

**场内未被完全淘汰的最关键原因** — 电子系统 **难以复制** 的 **微妙信息交换**。

### 4.1 大宗交易的信息搜集

| 需求 | 经纪人须交换 **买卖双方信息** 以安排成交 |
|------|----------------------------------------|
| **场内优势** | 观察 **其他经纪人行为**、建 **信任**、**试探** 隐藏意图 |
| | 评估对手是否 **危险知情者** |

→ [Ch 15 楼上 / latent demand](./chapter-15-大宗交易者.md) · [Ch 18 隐蔽](./chapter-18-买方交易者.md)

### 4.2 揭示真实规模

| 场内 | 通过 **声誉机制** **可信暗示** 未公开订单 **剩余规模** |
|------|------------------------------------------------------|
| **电子** | 多数系统 **无法** 提供同等灵活 **信任与暗示** |

| HFT 视角 |
|----------|
| **IOI、negotiated block、voice trading** — 电子时代的 **场内替代** |
| **Iceberg / hidden** — 规模隐藏的技术解，**非信任解** |
| **Upstairs broker relationship** — Ch 15 **电子化后仍要人声** |

## 5. 成本结构 (Cost Structures)

| | 自动系统 | 场内系统 |
|---|----------|----------|
| **初始建设** | 都很高 | 都很高 |
| **日常运营** | **极低** — 电信、备份为主 | **极高** — 房地产 + **大量人力** |
| **人力** | 工程师维护 | 经纪人、跑单员、文员、监控官员 |

| HFT 视角 |
|----------|
| **Fixed cost amortized over billions of messages** — 规模经济 |
| Floor **劳动力成本** 无法与 **零售电子化** 竞争 |

## 6. 市场分化与未来共存

| 细分 | 占优结构 |
|------|----------|
| **小额零售、活跃市场** | **自动** — 快、便宜 |
| **大额机构、需隐蔽寻对手** | **场内 / 类场内** — 信息交换 |

> 只要场内（或 **类场内服务**）仍为大型交易者提供电子 **无法替代** 的 **有价值信息交流**，大型场内市场（如 NYSE floor 残余角色）**不会轻易消亡**。

```
两种系统 → 不同客户群 → 并存竞争
```

| 现代现实 | NYSE **hybrid** — 大部分电子化；floor **象征 + 大宗 niche** |
|----------|--------------------------------------------------------------|
| **趋势** | 小额 **100% 电子**；大额 **voice + dark + upstairs** 承接场内功能 |

| HFT 视角 |
|----------|
| HFT = **自动段的核心参与者** |
| **Block desk** 仍 parallel — 勿以为 **全世界都在 orderbook 里** |
| 设计系统：**lit 自动 + block 人工通道** 双轨 |

## 8. 本章总结

| 要点 | 含义 |
|------|------|
| **自动赢** | 公平（规则化）、审计、扩展、速度、分布式、**低运营成本**、**零售小额** |
| **场内赢** | **大宗信息搜集**、声誉暗示 **真实规模**、评估 **知情风险** |
| **成本** | 电子 **边际成本低**；场内 **劳动力密集** |
| **共存** | **异质客户** — 不是二选一，是 **分段服务** |
| **HFT 位置** | 自动段；理解 floor **为何在机构段仍有类比物** |

> **HFT 读者 takeaway：** `orderbook.go` 代表 **自动段真理** — 公平、可 replay、可扩展。机构 **不把所有 size 打进去** 的原因在本章：**信息交换与信任** 电子 LOB 默认不提供。下一章 [Ch 28](./chapter-28-泡沫崩盘与熔断机制.md) — 电子系统 **超载与极端波动** 时的 **制度应对**。
