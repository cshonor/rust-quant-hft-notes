# Ch 3 §2 交易促成机构 · Trade Facilitators

> **Trading and Exchanges** · Larry Harris · **精读**

## 2. 交易促成机构 (Trade Facilitators)

**交易促成机构** — 提供 **交易场地、规则与技术** 的 **基础设施方**：

| | |
|---|---|
| **做什么** | 把 **做市商买卖价**、**买方委托单** 集中起来，按 **公开规则** 自动 **报价–撮合–成交** |
| **不做什么** | **不** 用自有资金与客户对赌（那是 **做市商/Dealer**，见 [§1](./section-1-1-市场参与者-买方与卖方.md)） |
| **目标** | 让交易 **高效、公平、可预期** 地发生 |

→ 你写的 go-dex **就是** 最小化的「交易促成机构」：**OrderBook + `Match()` + 规则**。

---

## 2.1 交易所 (Exchanges) — 最核心的促成平台

**Nasdaq、NYSE** 等 **上市股票交易所** 是最核心的 **公开「报价–撮合」平台**：

```
做市商 / 散户 / 机构的限价单、报价
              │
              ▼
    ┌─────────────────────┐
    │  集中订单簿 (LOB)    │  ← 透明、按规则排队
    │  价格–时间优先撮合    │
    └─────────────────────┘
              │
              ▼
           成交回报
```

| 特征 | 说明 |
|------|------|
| **公开透明** | 最优买卖价 (BBO)、（专业用户）Level II 深度 |
| **规则驱动** | 撮合优先级、订单类型、涨跌停 — [Ch 4](../chapter-04-orders-and-order-types/) · [Ch 6](../chapter-06-order-driven-markets/) |
| **历史形态** | NYSE **物理大厅 + 专家**；现代 **全电子**（SuperDot / SuperSOES → 今日 matching engine） |

**Harris 故事入口：** [Ch 2 §1 散户 Jennifer](../../chapter-02-trading-stories/notes/section-1-1-散户股票交易.md) — 单进 **公开簿** 的典型路径。

**HFT：** **Colocation** 在 **exchange matching engine** 旁；竞争 **queue position**、**ingress 顺序**。

---

## 2.2 ECN（电子通信网络）

**ECN (Electronic Communication Network)** — **电子化** 的交易促成平台，常与 **传统交易所并列** 或 **嵌入** 路由链：

| 角色（实务表述） | 说明 |
|------------------|------|
| **连接参与者** | 买方、卖方、做市商 **直接** 在电子网络上挂撤单 |
| **聚合 / 接力流动性** | 当 **第一层做市商** 接不住单时，路由 **继续找** ECN 等 **更优报价** — 见 [Ch 2 §1 Level II 多层接力](../../chapter-02-trading-stories/notes/section-1-1-散户股票交易.md) |
| **第四市场** | 与 [§4 市场分布](./section-4-4-交易市场的分布.md) **primary + fourth market** 并列 |

```
订单路由（简化）
  做市商内部化 / 义务报价
       │ 接不住
       ▼
  ECN 更优价
       │ 仍不够
       ▼
  其他 venue / 交易所 / …
```

**与 SOR 的关系：** **ECN** 是 **liquidity venue** 的一种；**SOR (Smart Order Router)** 是 **跨多个 venue（含 ECN、交易所）找最优价** 的 **软件层** — HFT / 机构 **在促成机构之上** 再叠一层。

**HFT：** 同时在 **primary + ECN** 挂单/吃单 → **跨 venue 套利**；**Reg NMS** 下 **best execution** 必须 **扫多个促成平台**。

---

## 2.3 暗池 (Dark Pools)

**暗池** — **机构间** 的 **替代交易促成平台 (ATS)**，核心卖点是 **隐藏**：

| | 公开交易所 | 暗池 |
|---|-----------|------|
| **报价可见性** | BBO / Level II **公开** | **不显示** 挂单（成交前） |
| **典型用户** | 散户 + 机构 **小额** | **大额** 机构块 |
| **主要目的** | 价格发现 + 流动性 | **降低市场冲击 (impact)**、防 **信息泄露** |

**Harris 故事：** [Ch 2 §2 Bob → POSIT](../../chapter-02-trading-stories/notes/section-2-2-机构股票交易.md) — 大单 **先进暗池** 保密撮合，再考虑公开簿。

```
机构 100 万股卖单
  → 若直接扫公开 Asks：所有人看见 → 抢跑、砸盘、冲击成本 ↑
  → 暗池：与对手 **匿名交叉** — 价格常近 **中间价**，量 **不暴露** 在 Level II
```

**注意：** 暗池 **仍是** 受监管的 **促成平台**（ATS）— 有 **撮合规则**，只是 **成交前不公开**。**OTC（§2.4）** 才是 **不经集中簿、一对一议价** 的模式。

---

## 2.4 OTC（场外交易 · Over-the-Counter）

**OTC** = **Over-the-Counter**，直译 **「场外交易」** — 与交易所 **集中挂单、规则自动撮合** **相对**：

```
交易所模式                         OTC 模式
──────────                         ────────
  多买家/卖家 → 公开 LOB              买方 ←→ 经销商/对手
  规则引擎 Match()                    一对一询价、谈价、谈量
  透明 BBO                            成交前不公开（通常）
  中央清算/结算（常）                  成交后 **单独** 确认、结算条款
```

| | **集中交易所** | **OTC** |
|---|----------------|---------|
| **定价** | 簿上 **公开竞价** | **经销商报价 (dealer quote)**、双边 **议价** |
| **透明度** | 高（Level I/II） | **低** — 成交价往往 **事后** 才披露（若有） |
| **灵活性** | 标准合约、固定 tick | **条款可谈** — 量、价、交割、信用条款 |
| **典型产品** | 上市股票、标准期货 | **大额债券**、**FX 远期**、复杂衍生品、**Upstairs 股票块** |
| **主要风险** | 市场冲击、排队 | **对手方违约 (counterparty risk)** — 无 CCP 时尤其 |

**Harris 故事：**

| 案例 | 模式 |
|------|------|
| [Ch 2 §6 Sam 买公司债](../../chapter-02-trading-stories/notes/section-6-6-债券市场交易.md) | 电话找 **所罗门兄弟** 销售，以 **国债收益率 + spread** 议价 — **典型 OTC** |
| [Ch 2 §3 Edna 大宗股](../../chapter-02-trading-stories/notes/section-3-3-超大宗股票抛售.md) | **Upstairs / 场外** 找基金接盘 — 股票也可走 OTC 逻辑 |

**优点（为何大单爱 OTC）：**

- **灵活** — 非标准量、非标准条款
- **不公开冲击** — 不在 LOB 上 walk the book，避免 [§3 抢跑/抬轿](../../chapter-02-trading-stories/notes/section-3-3-超大宗股票抛售.md)
- **适合** 单笔量大、**标准化程度低** 的工具

**缺点：**

- **透明度低** — 难做全市场 price discovery
- **对手方风险** — 依赖 **经销商信用** 或 **ISDA/CSA** 等法律框架（衍生品）

### OTC vs 暗池 vs 交易所（别混）

| | **交易所** | **暗池 (ATS)** | **OTC** |
|---|-----------|----------------|---------|
| **有没有集中撮合** | ✅ 公开 LOB | ✅ 有，但 **隐藏** | ❌ 通常 **无** — 双边谈 |
| **谁报价** | 所有人挂簿 | 参与者提交 **隐藏** 意向 | **经销商** 或 **大宗经纪** 牵线 |
| **股票大块** | 公开扫/挂 | POSIT 等 **匿名交叉** | Edna **Upstairs 折扣** |
| **债券** | 少数上市债 | 少 | **主流** — Sam 电话议价 |

---

## 四类促成/成交模式对照

| 类型 | 代表 | 透明度 | 典型场景 |
|------|------|--------|----------|
| **交易所** | NYSE、Nasdaq | **高** | 散户、做市、价格发现、**HFT 主战场** |
| **ECN** | 历史上 INET 等 | **高**（电子簿） | 电子化撮合、路由接力 |
| **暗池** | POSIT、银行 ATS | **低**（成交前隐藏） | 机构 **大块**、减 impact |
| **OTC** | 债券经销商、Upstairs | **低**（议价） | 大额债券、FX 远期、复杂衍生品、**协商股票块** |

**共同本质（交易所/ECN/暗池）：** **基础设施** — 定规则、跑撮合。**OTC** 更偏 **双边市场结构** — **经销商** 常同时是 [§1 做市商](./section-1-1-市场参与者-买方与卖方.md)，但 **促成方式** 是 **谈价** 而非 **公开 queue**。

---

## HFT：交易所主战场，OTC/暗池作补充

| 路径 | HFT 角色 |
|------|----------|
| **交易所 + ECN** | **主战场** — 高流动性、低延迟、LOB 策略、做市、arb |
| **暗池** | **机构 flow 补充** — 部分 HFT / 机构策略做 **暗池交叉**、**internalization**、**与公开簿联动** 的 hybrid |
| **OTC** | **非 tick 热路径** — 债券/FX **经销商流**、**RFQ 电子平台**；股票 **Upstairs** 与 HFT 分离，但 **影响公开簿**（大单不进 LOB → 少 impact） |

> **读法：** 先把 **公开 `Match()`** 练到熟（go-dex M1–M2）；再理解 **暗池 = 第二本簿**、**OTC = 不进簿的协商成交**（M5+）。

## 与 go-dex 的对照

| 真实世界 | go-dex 现状 |
|----------|-------------|
| 交易所 **公开 LOB** | 单一 `OrderBook` + `Match()` |
| ECN / 多 venue **SOR** | **未实现** — M5+ 路由层 |
| 暗池 **隐藏簿** | **未实现** — M5+ internal cross |
| **OTC 协商成交** | **未实现** — 成交价 **写入账本** 但不 walk 公开 `Bids`/`Asks` |

## 2.5 清算、结算与清算所 (Clearing, Settlement & Clearinghouses)

撮合 **成交** 之后，还有全市场共同的 **「最后一公里」** —— **清算与结算**。  
**不只股票**：上市股、债券、期货、期权、FX 等 **凡成交**，都要走 **清算 → 结算** 链（形式不同）。

```
成交 (Trade)
    │
    ▼
清算 (Clearing)     ← 「算账」：核对明细、净额、算谁付多少
    │
    ▼
结算 (Settlement)   ← 「交货」：证券/现金/仓单 实际划转
```

---

### 清算 vs 结算 — 别混

| 环节 | 白话 | 做什么 |
|------|------|--------|
| **清算 (Clearing)** | **算账** | 核对买卖双方 **成交明细** 一致；**净额 (netting)** — 同一客户多笔买卖轧差；算 **应付/应收** |
| **结算 (Settlement)** | **交货** | **证券 + 资金** 最终过户 — **货银对付 (DVP, Delivery versus Payment)** |
| **清算所 (Clearinghouse / CCP)** | **中央对手方** | 插在买卖双方中间，**担保履约** +（衍生品）**保证金 / 盯市** |

**撮合引擎 (`Match()`)** 只到 **成交回报**；**清算结算** 在 **交易所/经纪后台** — go-dex M1–M2 **刻意省略**，实盘 **必须接**。

---

### 现货证券（股票、债券等）

**典型模式：** **一手交钱、一手交货** 的 **现货清算**。

| 案例 | 链路 |
|------|------|
| [Ch 2 §1 Jennifer 买股票](../../chapter-02-trading-stories/notes/section-1-1-散户股票交易.md) | 成交 → **NSCC** 清算 → **DTC** 过户 → **T+1**（美国现多为 **T+1**）资金交割 |
| [Ch 2 §6 Sam 买公司债](../../chapter-02-trading-stories/notes/section-6-6-债券市场交易.md) | OTC 议价成交 → 同样 **NSCC** 记账 + **DVP** |

| 特点 | 说明 |
|------|------|
| **逐笔/full delivery** | 到期日 **交股票、交钱** — 无「每日浮动盈亏清算」 |
| **净额结算** | 同一经纪/客户当日多笔 **轧差** 后一次交收 — 降流动性需求 |
| **CCP** | 美国上市股经 **NSCC** 中央清算 — 降 **对手方违约** 风险 |

---

### 衍生品（期货、期权等）— 复杂得多

[Ch 2 §4 大豆期货套保](../../chapter-02-trading-stories/notes/section-4-4-期货市场-套期保值.md) · [§5 期权 OCC](../../chapter-02-trading-stories/notes/section-5-5-期权市场交易.md)

衍生品 **不必等平仓** 才算总账 —— 持仓期间就有 **逐日盯市 (Mark-to-Market, MTM)**：

```
每个交易日收盘
    │
    ▼
清算所按「结算价」算当日盈亏
    │
    ├── 盈利 → 划入账户
    └── 亏损 → 从账户扣走（保证金账户）
```

| 机制 | 说明 |
|------|------|
| **CCP** | **CME Clearing**（期货）、**OCC**（美国期权）— 对 **所有买卖双方** 成为交易对手 |
| **初始 / 维持保证金** | 开仓 **预存**；亏损 **每日扣** — 不足则 **追保 (margin call)** / **强平** |
| **盯市清算** | **每天** 按结算价 re-mark — 不是「平仓后再算总账」 |

**到期交割**（若持有至到期）：

| 类型 | 流程 |
|------|------|
| **现金交割 (Cash settlement)** | 按 **最终结算价 vs 开仓价** 算 **差价**，**直接划钱** — 多数 **指数期货/期权** |
| **实物交割 (Physical delivery)** | **仓单划转**、**质检**、物流 — 如 **部分商品期货**（大豆等） |

→ 套保加工商：可能 **平仓** 期货对冲现货，也可能 **实物交割** — 与 [Ch 2 §4](../../chapter-02-trading-stories/notes/section-4-4-期货市场-套期保值.md) 商业节奏相关。

---

### 现货 vs 衍生品清算对照

| | **股票 / 债券（现货）** | **期货 / 期权（衍生品）** |
|---|-------------------------|---------------------------|
| **清算频率** |  mainly **交收日** 净额 | **每日盯市** + 到期交割 |
| **交什么** | 证券 + 现金 (DVP) | 多数时候只 **划盈亏**；到期才 **货/现金** |
| **杠杆** | 现货全额（或融资） | **保证金** 杠杆 |
| **中央对手方** | NSCC 等 | **CME / OCC** 等 **CCP** |
| **HFT 热关注** | 交收失败、借券 | **保证金占用、强平线、结算价跳变** |

---

### HFT：为何必须盯清算？

| 指标 | 为何重要 |
|------|----------|
| **实时 / 日内盯市** | 期货持仓 **每天** 扣盈补亏 → **可用资金** 随结算价变 |
| **保证金占用 (IM/MM)** | 策略容量上限 — 不是「成交完就结束」 |
| **强平 (Liquidation)** | 极端行情 + 杠杆 → **系统自动平仓** — 独立于你的 alpha |
| **Pre-trade risk** | 发单前检查 **购买力 / 保证金** — 热路径外但必须通 |
| **Post-trade** | 成交后 **清算所** 记账 — 对账、审计、监管报送 |

> **Takeaway：** 交易所 **Match()** 决定 **成交价**；**清算所** 决定 **你还剩多少钱能下下一单**。衍生品 HFT 的风控核心之一 = **保证金 + 盯市**，不是只有 LOB 延迟。

## 2.6 存管与托管 (Depositories and Custodians)

代客持有 **现金与证券凭证**，让 **DVP 结算** 可以 **电子化、批量** 完成 —— 现货 **交货** 环节的 **仓库**。

| 机构 | 角色 |
|------|------|
| **DTC** | 美国证券 **中央存管** — 与 **NSCC** 清算衔接 |
| **托管行 (Custodian)** | 机构/基金 **代持** 资产 — 大宗、OTC 成交后 **入账** 到此 |

→ HFT：**tick 路径** 不经 DTC；**日终 / 交收** 必须与 **prime broker 清算文件** 对账。
