## 3. 超大宗股票抛售 (A Very Large Block Stock Trade)

Edna 须抛售 **90 万股** 家族企业股票（数千万美元）缴遗产税：

1. 高盛 **大宗经纪 (block broker)** 接手
2. **不能** 直接在公开市场抛售（会 **价格暴跌**）
3. 查 **13F 公开持仓**，定位对该股有兴趣的大型基金
4. 场外 **Upstairs market** 成交：提供 **价格折扣 (discount)** + 说明抛售原因是 **纳税而非基本面恶化**

| 概念 | HFT 关联 |
|------|----------|
| Block / upstairs | 与 HFT 热路径分离的大宗协商市场 |
| Information leakage | 大单意图暴露 = 最大敌人 |
| 13F | 公开信息用于找 counterparty——HFT 不碰此类但需知 institution 行为 |

---

### OTC 流动性补偿：卖单 **折扣** vs 买单 **溢价**

Edna 若把 **100 万股** 直接在 **公开 LOB** 砸出去：

- 每吃一档深度，价格就被 **往下打一档**（walk-the-book + **市场冲击**）
- 最终 **成交均价** 远低于开始看到的 Level 1 卖价 — **自己砸崩自己**

**买单是镜像** — 冲击方向反过来：

- 公开挂 **巨量买单** / 市价扫货 → 从 **卖一往上吃** Asks，买一到买五全扫光
- 股价被 **一路拉高** → **平均建仓成本** 远高于挂单时的 Level 1 — **「买贵了」**
- 和卖单一样会 **信息泄露**：市场看见你要扫货 → 投机盘 **提前抢低价筹码** → **高价卖给你** — **抬轿套利**

| 方向 | 公开大单后果 | OTC / 暗池里「流动性补偿」长什么样 |
|------|--------------|-----------------------------------|
| **卖（Edna）** | 砸盘、均价偏低 | 卖方给 **折扣**（如比市价低 2%）换快速出清、不暴露 |
| **买（镜像）** | 拉盘、均价偏高、被抬轿 | 买方愿意给 **溢价**（如比市价高 2%）换 **快速拿够筹码**、不暴露扫货意图 |

所以走 **OTC / Upstairs** 或 **暗池**：大宗经纪 / 池子 **直接找对手** — 卖找接盘方，买找有 **出货需求** 的对手 — **一次性谈成**：

| 谁 | 卖单场景（Edna） | 买单场景（镜像） |
|----|------------------|------------------|
| **主动方（机构）** | 比公开砸盘 **更高均价**；快速出清；不暴露 | 比公开扫货 **更低均价**；快速建仓；不被抬轿 |
| **对手盘** | **折扣**接货 — 补偿锁仓、波动风险 | **溢价**出货 — 补偿 **立刻让出筹码、错过继续涨** 的机会 |
| **公开市场** | 避免瀑布式砸盘 | 避免被大单 **一路拉抬** |

**补偿不是白送** — 都是 **流动性 premium** 的两侧：

```
卖：公开损失 ≈ 冲击成本 + 抢跑砸盘     →  OTC 折扣换不暴露
买：公开损失 ≈ 冲击成本 + 抢跑抬轿     →  OTC/暗池 溢价换不暴露、快速成交
机构算账  →  若 溢价/折扣 < 公开冲击+被 front-run 的额外成本，就走场外
```

**例子（数字直觉 · 卖）：**

- Level 1 价 100；公开卖 100 万股可能 **均价跌到 96**
- OTC 谈成 **98** — 卖方比 96 好，买方便宜 2% 换 **立刻全接**

**例子（镜像 · 买）：**

- Level 1 价 100；公开买 100 万股可能 **均价拉到 104**
- 暗池 / OTC 谈成 **102** — 买方比 104 省，卖方多得 2% 换 **立刻让出筹码、不跟散户抢公开深度**

### 公开挂单的信息泄露：抢跑（卖：砸盘再接 / 买：抬轿再卖）

大资金 **怕在公开簿挂超大限价单**，不只因为 walk-the-book 的 **冲击成本** — 还因为 **意图一露，投机盘会抢跑**。

**卖单（Edna · 出货）：**

```
市场看到你挂「超大量限价卖」
  → 投机资金快速做空 / 先砸盘，把价格打下去
  → 你在更低价位被迫成交（或被定点市价买扫走后价格已转弱）
  → 他们反手在低价接货，等价格回升赚差价
```

**买单（镜像 · 扫货 / 建仓）：**

```
市场看到你挂「超大量限价买」/ 即将扫货
  → 投机资金提前在低价档 **抢筹码**
  → 你再往上吃 Asks 时，卖一卖二已被抬价 — **买得更贵**
  → 他们 **高价卖给你** — 「抬轿」套利
```

| 尺度 | 书上哪里讲过 | 卖 | 买（镜像） |
|------|--------------|-----|------------|
| **散户 / 中等挂单** | [Ch 1 §4.2](../../chapter-01-introduction-market-microstructure/notes/section-4-4-贯穿全书的关键主题.md#42-期权与外部性-options-and-externalities) | 市价买扫 Ask → 市价卖砸 Bid → 低价接回 | 市价卖扫 Bid 触发你的买单 → 再拉涨；**利空买** 被吃后继续跌 |
| **机构超大块** | 本节 | 公开 Level II 泄露 → **砸盘再接** | 公开泄露 → **抢筹抬轿** |

这就是典型的 **pick-off 套利** — 限价单给的 **免费期权**（§4.2），在大单场景下变成 **系统性信息泄露**；**买卖方向只是冲击和抢跑的手法镜像**。

**暗池为什么存在（买、卖都需要）：** 把这条信息链 **掐断** —

| 公开 LOB | 暗池 |
|----------|------|
| 挂单进 Level II，**谁都能看见** | 大单 **不出现在公开 Orderbook** |
| 投机盘有时间 **布局、抢跑** | 通常只有 **撮合双方 + 运营方** 知道 |
| 成交价格被 **公开市场踩踏** 牵动 | 买卖在 **双方认可价** 直接交叉，少一层公开竞价 |

**OTC / 暗池 · 流动性补偿（买卖两侧）：**

| | **卖（Edna）** | **买（镜像）** |
|---|----------------|----------------|
| 机构愿意 | **让折扣** | **付溢价** |
| 换来 | 快速出清、不暴露、防砸盘 | 快速拿筹码、不暴露、防抬轿 |
| 对手盘 | 折扣接货 — 扛锁仓与波动 | 溢价出货 — 扛「卖早了」机会成本 |

```
暗池  = 藏单，池内找对手（卖找买盘 / 买找卖盘），抹平信息差
OTC   = 场外议价 + 折扣或溢价
§4.2  = 同一套 pick-off；卖单慢动作见上表，买单见「场景 B」
```

### 暗池 vs OTC vs 公开簿：三种大单出路

| | **公开 LOB**（§1 Jennifer） | **暗池**（§2 Bob · POSIT） | **OTC / Upstairs**（本节 Edna） |
|---|---------------------------|---------------------------|----------------------------------|
| **单去哪** | 挂进 **公开 Orderbook** | **藏单** — 不进公开 Level II | **场外** — 不经 LOB 竞价 |
| **怎么找对手** | 价格–时间优先自动撮合 | 暗池内 **保密交叉**（常近 mid） | 经纪 **点名找** 对手（13F、关系网） |
| **价格** | 市场竞价 | 多在 **价差中间** 附近 | **协商价** — 卖常 **折扣**、买常 **溢价**（流动性补偿） |
| **核心目的** | 散户常规成交 | **藏意图**、减冲击、**防抢跑**（买/卖双向） | **超大块一次清**、避免冲击公开价 |
| **互补关系** | 基础池子 | **藏单** | **直接找对手给优惠价** |

```
机构大单
    要卖（Edna）
    ├─ 能拆小、能等 → 公开簿 + 算法拆单（§2）
    ├─ 要保密 → 暗池 POSIT（§2）
    └─ 超大、必须快 → OTC + 折扣（本节）
    要买（镜像）
    ├─ 能拆小、能等 → 公开簿 + 算法拆单
    ├─ 要保密、悄悄接盘 → 暗池找 **有货要出** 的对手
    └─ 超大、必须快 → OTC + **溢价** 换筹码
```

> **和 go-dex：** 你现在只有 **公开 `Orderbook` + `Match()`**（§1 散户 + MM）。**暗池** = 第二本簿 / internal cross（M5）；**OTC** = **连 LOB 都不进** 的协商成交模块 — 成交价写进系统但不 walk 公开 `Bids`/`Asks`（M5+ 或独立 simulation）。
