# Ch 22 绩效评估与预测 · Performance Evaluation and Prediction

> **Trading and Exchanges** · Larry Harris · **选读** · Part VI

本章探讨如何衡量 **投资组合经理与交易者** 的 **历史表现**，以及为何 **仅凭过去业绩预测未来回报极其不可靠**。

> **HFT 读者：** 策略 **回测 / Sharpe / t-stat** 的 **统计学陷阱** 本章全涵盖；与 [Ch 8 零和](./chapter-08-为什么人们要交易.md)、[Ch 21 TCA](./chapter-21-流动性与交易成本衡量.md)、[Ch 10 edge](./chapter-10-知情交易者与市场效率.md) 衔接 — **「过去好看 ≠ 未来能活」**。

---

## 0. 本章主线

```
衡量历史绩效（绝对 → 相对 → 风险调整）
        ↓
统计检验（t-test）— 数据极度不够
        ↓
样本选择偏差 / 欺诈回报 — 最危险陷阱
        ↓
预测未来：别只看业绩 → 找比较优势 (Comparative Advantage)
```

---

## 1. 核心难题：运气与技能 (Luck vs. Skill)

| 来源 | 说明 |
|------|------|
| **技能 (Skill)** | 管理质量、真实 edge |
| **运气 (Luck)** | 宏观、微观、公司层面 **不可预测** 因素 |

竞争激烈市场中价格变化 **很大程度随机游走** → **区分真技能与碰运气极其困难**。

| HFT 视角 |
|----------|
| **短样本高 Sharpe** — 可能是 **regime luck** 或 **overfit** |
| **Microstructure alpha** 衰减 — 去年有效的 queue 模型今年可能 **crowded out** |
| 交易是 [Ch 8](./chapter-08-为什么人们要交易.md) **零和** — 跑赢指数 ≠ 长期跑赢 **同类 HFT** |

---

## 2. 绩效衡量方法 (Performance Evaluation Methods)

### 2.1 绝对绩效 (Absolute Performance)

特定期间 **总回报率** — 含资本增值 + **股息、利息** 等现金流。

### 2.2 相对绩效 (Relative Performance)

剔除 **市场整体** 影响：

```
市场调整回报 = 组合回报 − 基准指数回报（如 S&P 500）
```

### 2.3 风险调整回报 (Risk-adjusted Returns)

扣除承担 **已知市场风险（Beta）** 应得的回报：

| 目的 | 判断 **选股能力**（**实现 Alpha**）或 **择时能力** |
|------|---------------------------------------------------|

| 常见指标 | CAPM alpha、Sharpe、Information ratio 等 |
|----------|----------------------------------------|

| HFT 视角 |
|----------|
| **Beta-neutral stat arb** — 相对绩效比绝对更 relevant |
| **Risk adj** 须含 **tail risk**（flash crash）— 方差不够 |

---

## 3. 统计评估的局限性与「数据饥渴」

### 3.1 t-检验 (The t-Test)

比较 **平均市场调整回报** 与 **标准误** 的比率 — 判断超额回报是否 **大到无法仅用运气解释**。

### 3.2 极度缺乏数据

投资中 **「噪音」>>「信号」**（经理真实技能）：

| 例子 | 要以 **95% 置信度** 确认经理 **年化 2% 超额** 来自技能 |
|------|--------------------------------------------------------|
| **需要** | 约 **22 年** 收益数据 |

现实中 **难以获得** 足够长期、一致的有效数据 → **无法确信分离技能与运气**。

| HFT 视角 |
|----------|
| **6 个月回测 t=3** — 统计上 **几乎无意义** 于 skill 认定 |
| **Live track** 1–2 年 — 仍可能在 **噪音窗口** 内 |
| **Multiple testing**（试 100 个策略）— 未校正则 **假阳性泛滥** |

---

## 4. 最重要警告：样本选择偏差 (Sample Selection Bias)

**导致投资者亏损的最危险陷阱之一** — 解释「明星基金」后 **表现平庸**。

### 4.1 庞大基数中的极度幸运儿

| 设定 | 成千上万经理，**全无技能**（抛硬币） |
|------|--------------------------------------|
| **结果** | 概率上仍会出现 **连续多年跑赢** 的极端幸运儿 |

### 4.2 生存者偏差与掩盖

| 行为 | 基金公司 **大肆宣传** 优异基金 |
|------|------------------------------|
| **隐藏** | **关闭 / 合并** 差基金 |
| **后果** | 不知 **庞大基数** → 把 **极度幸运** 当成 **高超技能** |

### 4.3 连环信骗局 (My Favorite Fraud)

经典诈骗结构：

```
2 万人：一半信涨、一半信跌
    ↓ 一月后
1 万人「全对」→ 再一半涨、一半跌
    ↓ 重复 10 个月
20 人看到「连续 10 次 100% 准确」→ 奉为神明
```

> **忽视筛选过程、只看最终结果** — 与 **基金星级、策略推销** 同构。

| HFT 视角 |
|----------|
| **Vendor 只展示 survival strategies** |
| **Prop desk 关闭亏损 team** — 外部只见 **winner desk** |
| 问：**这个 Sharpe 是从多少 parallel 实验里 **挑出来的**？**

## 5. 欺诈性回报 (Fraudulent Returns)

高回报有时不仅是运气 — 还有 **造假 / 操纵**：

### 5.1 回报平滑 (Return Smoothing)

| 对象 | 房地产、未上市股权等 **非流动性** 资产 |
|------|----------------------------------------|
| **手法** | **缓慢调整** 估值 |
| **效果** | 人为 **降低波动** → t-检验 **虚高** → 看起来 **更安全** |

### 5.2 庞氏骗局 (Ponzi / Pyramid Schemes)

用 **新投资者资金** 支付 **早期虚假高回报** — 直至 **资金链断裂**。

| HFT 视角 |
|----------|
| **Mark-to-model** 非透明 fund — smoothing 风险 |
| **Crypto yield** 类 — Ponzi 结构警惕 |
| 与 [Ch 12](./chapter-12-虚张声势者与市场操纵.md) 操纵谱系不同但 **投资者后果类似** |

## 6. 预测绩效：寻找比较优势 (Comparative Advantage)

既然 **统计过去业绩不能有效预测未来** — 如何判断谁能成功？

### 6.1 唯一可靠方向

分析经理人是否具备 **真实的比较优势** — 而非 extrapolate 历史曲线。

### 6.2 入场前自问

在以下方面是否拥有 **他人不具备的绝对且相对优势**：

| 维度 | 例子 |
|------|------|
| **智力** | 建模、推理 |
| **经验** | 经历过多种 regime |
| **教育背景** | 领域专长 |
| **组织能力** | 系统、流程、风控 |
| **纪律性** | 执行一致性 |
| **信息获取渠道** | 合法、可持续的数据 edge |

> 交易是 **零和博弈** — 光是表现「好」不够，必须比 **竞争对手更好**。

| HFT 视角 |
|----------|
| **Comparative advantage** = co-lo + feed + 工程 + **合法信息** + **资本** 的组合 |
| 无优势 → 回到 Ch 14 **不知情结构性亏损** |
| **Edge 可持续性** — 技术会被复制；**关系 / 数据独占 / 监管许可** 更 durable |

## 7. 与 Ch 21 的衔接

| Ch 21 | Ch 22 |
|-------|-------|
| **执行绩效** — IS、spread、markout | **投资/策略绩效** — alpha、luck、bias |

---

## 8. 本章总结

| 要点 | 含义 |
|------|------|
| **Luck vs Skill** | 随机游走市场中 **极难区分** |
| **衡量层次** | 绝对 → 市场调整 → 风险调整（Alpha） |
| **t-test** | 有用但 **需数十年数据** 才稳 |
| **样本选择偏差** | 生存者 + 基数忽视 — **最危险** |
| **欺诈** | 回报平滑、庞氏 |
| **预测** | 不看历史曲线 → 问 **比较优势** |

> **HFT 读者 takeaway：** 回测报告须附带 **试验次数、样本外、regime、成本（Ch 21）**。**22 年 rule of thumb** 对 **2% alpha** — HFT 策略周转快，但 **t-stat 逻辑相同**：短窗 **噪音主导**。立项前写清 **comparative advantage 清单**，而非 **去年 PnL**。
