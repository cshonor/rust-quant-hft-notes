## 4. 自旋锁 (Spinlocks)

---

### 一、忙等待 vs 睡眠

| | **自旋锁** | **互斥锁 (mutex)** |
|---|-----------|-------------------|
| 资源被占用时 | **循环查询（自旋）** 等待 | 线程 **睡眠**，唤醒后竞争 |
| 上下文切换 | **无**（等待期间） | 有 — 代价高 |
| 中断上下文 | **可用** | 通常 **不可用**（不可睡眠） |
| 临界区长度 | 必须 **极短** | 可较长 |

DPDK 热路径追求 **低延迟** — 短临界区用自旋锁避免 **睡眠/唤醒** 开销。

---

### 二、DPDK：`rte_spinlock_t`

广泛应用场景：

| 模块 | 典型用途 |
|------|----------|
| **告警 / 日志** | 多核写日志缓冲区 |
| **中断机制** | 与 PMD 中断路径共享状态 |
| **内存共享** | 全局结构短临界区 |
| **link bonding** | 聚合端口状态更新 |

API 模式：`rte_spinlock_lock` / `rte_spinlock_unlock` — 与 Linux 内核 spinlock 语义同构。

---

### 三、风险与调优

| 风险 | 说明 |
|------|------|
| **CPU 空转** | 持锁时间长 → 其他核 **burn cycles** |
| **优先级反转** | 低优先级持锁阻塞高优先级 lcore |
| **Cache 行 bouncing** | 锁变量在多核间 **频繁 invalidate** |

**HFT：** 热路径 **尽量避免 spinlock**；若必须用 — 临界区仅几条指令，且 **per-lcore 数据** 优先。
