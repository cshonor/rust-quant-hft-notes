## 5. 队列长度及阈值设置

---

### 一、收包队列长度 = 描述符个数

| 过长 | 过短 |
|------|------|
| **Mbuf 占用↑**、分配/释放 **时间↑** | 突发流量 **缓冲不足 → 丢包** |
| 内存压力 | 难以吸收 **micro-burst** |

---

### 二、经验值与场景

| 配置 | 适用 |
|------|------|
| **128** | DPDK 多数示例默认 — **一般场景** 经验值 |
| **512 / 1024** | **更高速率**、易丢包、或 **混合中断**（唤醒前需更深缓冲） |

**调优流程：**

1. 从 128 起压测，观察 **imissed / rx_nombuf**（驱动统计）  
2. 有突发丢包 → **加大环深** 或 **加大 mempool**（[Ch6 mbuf](../chapter-06-pcie-packet-io/notes/section-6-Mbuf与Mempool.md)）  
3. 延迟敏感 → 避免 **过长环** 导致 **旧包排队**

---

### 三、与 mempool 的关系

```
RX ring 深度 × 队列数 ≤ 在途 mbuf + mempool 余量
```

- 每描述符占 **1 mbuf** — 环深度直接决定 **最低 pool 规模**  
- **per-lcore cache** 仍须保证全局 pool 不耗尽（Ch6）

---

### 四、HFT 简要建议

| 场景 | 倾向 |
|------|------|
| 低 PPS 组播 tick | **128 或更低** — 减缓存占用与排队 |
| _burst 行情 / 多源_ | 按 **峰值 PPS × 吸收时间** 估算环深 |
| 生产 | 用 **实际 trace 回放** 定参，勿只抄示例 |
