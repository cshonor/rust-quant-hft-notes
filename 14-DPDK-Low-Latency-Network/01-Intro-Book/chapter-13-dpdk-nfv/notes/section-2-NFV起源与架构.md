## 2. 网络功能虚拟化（NFV）起源与架构

---

### 一、诞生背景

传统运营商网络：

| 痛点 | 说明 |
|------|------|
| **设备种类多** | 防火墙、BRAS、移动核心网关… **各用专有 ASIC** |
| **成本高** | 硬件绑定、扩容慢 |
| **运维复杂** | 物理部署、升级周期长 |

**NFV 目标：** 在 **通用服务器（X86）+ 虚拟化** 上运行 **软件化网元（VNF）** — 可创建、迁移、弹性伸缩。

---

### 二、ETSI NFV 基本架构

```
┌─────────────────────────────────────┐
│  NFV MANO（管理与编排）              │
├─────────────────────────────────────┤
│  VNF  ← 虚拟网络功能（软件网元）     │
├─────────────────────────────────────┤
│  NFVI ← 基础设施                     │
│    · 计算 / 存储 / 网络硬件          │
│    · 虚拟化层（KVM、OpenStack…）     │
│    · **DPDK 数据面加速**  ← 本章焦点 │
└─────────────────────────────────────┘
```

| 组件 | DPDK 角色 |
|------|-----------|
| **NFVI** | 高速报文 I/O、多核框架、硬件卸载 API |
| **VNF** | 基于 DPDK 的 vRouter、vFW、vBRAS 等 |
| **MANO** | 编排 **不直接** 跑 DPDK — 决定 VNF 部署拓扑 |

→ 与 [Ch1 认识 DPDK](../chapter-01-dpdk-intro/) 定位一致：**数据面基石**

---

### 三、与 SDN 关系（概念）

- **SDN** — 控制面集中、转发面可编程  
- **NFV** — 网元 **软件化、虚拟化**  
- 常 **结合**：SDN 编排 + NFV VNF + **DPDK 转发**
