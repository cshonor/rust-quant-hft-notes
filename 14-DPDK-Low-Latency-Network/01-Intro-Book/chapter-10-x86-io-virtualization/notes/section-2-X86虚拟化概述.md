## 2. X86 平台虚拟化概述

---

### 一、虚拟化三要素

| 维度 | 问题 |
|------|------|
| **CPU 虚拟化** | 客户机执行敏感指令 → 如何陷出与切换 |
| **内存虚拟化** | GVA → GPA → HPA **两次地址转换** |
| **I/O 虚拟化** | 设备访问与 **DMA** 如何隔离、转发 |

Intel **硬件辅助**：**VT-x**（CPU）、**EPT**（内存）、**VT-d**（I/O DMA，→ §3）。

---

### 二、CPU 虚拟化 (VT-x)

| 概念 | 说明 |
|------|------|
| **VMX 根模式** | 宿主机运行 |
| **VMX 非根模式** | 客户机运行；敏感操作 → **VM-Exit** 陷出 |
| **VMCS** | 虚拟机控制结构 — **加速** 状态切换，缓解传统 x86 虚拟化漏洞 |

→ 客户机内 DPDK **仍跑在非根模式** — 极端敏感路径偶发 VM-Exit 成本（通常 I/O 透传后数据面较少）

---

### 三、内存虚拟化 (EPT)

传统：**一次** 页表转换。虚拟化需：

```
GVA (客户机虚拟) → GPA (客户机物理) → HPA (宿主机物理)
```

**EPT（扩展页表）**：硬件 **直接完成 GPA→HPA**，减软件影子页表开销。

- 结果缓存在 **CPU TLB / Cache** — 与 [Ch2 TLB](../chapter-02-cache-and-memory/notes/section-2-阶梯式Cache系统.md) 同构

---

### 四、I/O 虚拟化三种方式

| 方式 | 机制 | 性能 |
|------|------|------|
| **全虚拟化** | 宿主机 **完全模拟** 设备 | 低 — 透明但慢 |
| **半虚拟化** | **virtio / vhost** 前后端协商 | 中 — 仍非裸金属 |
| **I/O 透传** | 物理设备 **直接分给 VM** | **高** — DPDK in VM 的基础 |

**DPDK 选型：** 数据面追求线速/低延迟 → **透传（+ SR-IOV VF）**；控制面可用 virtio。

→ 半旁路对照 [02-Advanced XDP note](../../02-Advanced-Book/notes/note-XDP与DPDK对照.md)
