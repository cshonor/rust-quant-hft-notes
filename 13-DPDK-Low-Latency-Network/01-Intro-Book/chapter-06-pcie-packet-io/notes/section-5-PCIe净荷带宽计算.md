## 5. PCIe 净荷转发带宽计算

---

### 一、为何要量化

评估包处理 **理论天花板** 不能只看：

- 网口线速（如 10G / 25G / 100G）  
- 帧长 64B 的 **净荷**

必须计入 **每个包在 PCIe 上的全部事务** — 描述符读/写、数据 DMA、Completion 等。

---

### 二、最小包 (64B) 的开销示例

对 **最小以太网帧**（64B 数据净荷），单次转发可能包括：

| 事务类型 | 说明 |
|----------|------|
| RX 描述符读 | 硬件读描述符 |
| RX 数据写 | DMA 写入 64B+ 帧 |
| RX 描述符写回 | 更新 DD 等状态 |
| TX 描述符读/写 | 发送侧对称操作 |
| TX 数据读 | DMA 读待发帧 |
| … | 实现相关额外 TLP |

**合计 PCIe 流量可远大于 64B** — 书中示例可达约 **168B 量级** 总开销（含协议头），视平台与驱动路径而定。

---

### 三、对 PPS 的含义

```
有效 PPS 上限 ≈ PCIe 有效带宽 / 每包 PCIe 总字节
```

| 现象 | 原因 |
|------|------|
| **小包线速难达** | 固定 TLP/描述符开销占比大 |
| **大包更接近线速** | 净荷摊薄协议与描述符成本 |
| **双口双向** | PCIe 带宽 **收发共享** — 更易饱和 |

**HFT 行情：** UDP tick 常 **远小于 64B 最小以太网帧**（垫到 64B）— 仍受 **每包固定 I/O 税** 约束；极高 PPS 需 **多队列多核 + 硬件 offload**（Ch8）。

---

### 四、调优检查清单

- [ ] 网卡 PCIe **Gen × Lane** 是否匹配端口速率  
- [ ] MPS / 描述符 batch 是否凑满 Cache Line  
- [ ] MMIO 频率（§4）  
- [ ] DDIO / NUMA 本地内存（Ch2）
