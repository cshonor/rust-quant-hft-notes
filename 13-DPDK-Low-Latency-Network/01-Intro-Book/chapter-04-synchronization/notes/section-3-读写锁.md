## 3. 读写锁 (Read-Write Locks)

---

### 一、机制

读写锁是 **特殊的自旋锁**，将访问分为 **读 / 写** 两类：

| 模式 | 并发规则 |
|------|----------|
| **读锁** | **多个读者可同时** 持有 — 上限可达逻辑 CPU 数 |
| **写锁** | **排他** — 不能同时存在读者与写者 |

**适用：** **读多写少** — 查找远多于创建/修改/删除。

---

### 二、DPDK API 与场景

典型 API：`rte_rwlock_read_lock` / `rte_rwlock_write_lock` 等。

| 场景 | 保护对象 |
|------|----------|
| **memseg 查找** | 空闲内存段元数据 |
| **LPM 表** | 最长前缀匹配表创建/查找/释放 |
| **Memory ring** | 部分 ring 元数据路径 |
| **ACL 表** | 访问控制表 |
| **Memzone** | 命名内存区注册与查找 |

---

### 三、与自旋锁 / 无锁的选型

```
读 ≫ 写、结构较大、允许短暂阻塞读路径  →  rwlock
极短临界区、不可睡眠                    →  spinlock（§4）
高吞吐队列、MP/MC                       →  rte_ring 无锁（§5）
```

**HFT 注意：** 写锁持有期间 **所有读者阻塞** — 表项 **批量更新** 宜离线或双缓冲，避免 tick 路径撞写锁。
