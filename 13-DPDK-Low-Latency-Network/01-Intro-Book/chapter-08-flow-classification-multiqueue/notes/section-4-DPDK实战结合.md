## 4. DPDK 与流分类的实战结合

---

### 一、分而治之：转发 + 控制

**场景：** 海量 **转发** 流量 + 少量 **控制** 报文（路由协议、管理面、会话建立）

| 路径 | 硬件机制 | 处理核 |
|------|----------|--------|
| **大流量转发** | **RSS** 散列到 **多队列 / 多核** | 多个 lcore 负载均衡 |
| **控制报文** | **Flow Director** → **专属队列** | **单独 lcore** |

**收益：**

- 省 **软件 ACL/filter**  
- 控制面与数据面 **不抢同一核** — 减少 **cache 抖动** 与 **跨核迁移**

**HFT 类比：** 组播 **行情 flood** 走 RSS 多核；**下单 TCP 会话** 或 **固定源** 控制消息走 FD Dedicated queue。

---

### 二、虚拟化与 Cloud Filter

| 组件 | 作用 |
|------|------|
| **SR-IOV VF** | 虚拟机 **直接** 驱动 VF 队列 |
| **Cloud Filter** | 硬件将流 **导向 VF 队列** |

VM 内 **DPDK VF PMD** 轮询 — **绕过 vSwitch** 软件路径 → **VM 内网络密集型** 吞吐 ↑。

共置 **裸金属 HFT** 通常 **不用 VF**；**云/托管** 部署需了解此架构。

---

### 三、与 L3fwd 的关系

[L3fwd](../chapter-01-dpdk-intro/notes/section-6-编程实例入门.md) 在 **软件** 做 Exact Match / LPM — 本章在 **硬件** 做 **队列级** 分流。

**分层：**

```
硬件 RSS/FD → 队列 → lcore
    ↓ 仍可在核内
软件 LPM/ACL（L3fwd 类）
```

能 **硬件做** 的尽量 **硬件做** — 为 SDN/NFV/复杂应用 **省 CPU**。
