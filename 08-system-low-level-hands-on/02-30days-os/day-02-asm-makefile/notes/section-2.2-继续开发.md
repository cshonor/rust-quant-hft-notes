#### 基础汇编指令

| 指令 | 含义 |
|------|------|
| **`ORG`** | 告诉汇编器：程序将被装载到内存的 **起始地址** |
| **`JMP`** | **跳转** — 改 IP，下一条指令从别处执行 |
| **`MOV`** | **赋值** — 寄存器 ↔ 寄存器 / 立即数 / 内存 |

#### CPU 寄存器（16 位实模式）

| 类型 | 示例 | 说明 |
|------|------|------|
| **16 位** | AX、CX、DX、BX、SP、BP、SI、DI | 通用 / 栈指针 / 变址 |
| **8 位** | AL/AH（AX 低/高字节）等 | 同一寄存器的 **高低半字** |

**8 个主寄存器加起来 ≈ 16 字节** — CPU **内部存储极小**，大部分数据必须在 **内存** 里。

```
寄存器  ≈  口袋（快、小）
内存    ≈  超大「住宅区」（慢、大）
```

**HFT 直觉：** L1/L2 cache、寄存器分配、**少访存** — 根源就是 **寄存器快、内存慢**（数量级差）。

#### 内存读写 · 方括号 `[]`

```nasm
MOV AL, [SI]    ; 把 SI 指向的 **内存地址** 上的 1 字节读入 AL
```

- **`[]`** = **内存地址**（不是寄存器里的值本身）
- 访问内存 **比访问寄存器慢得多** — 高效程序 = **少碰内存、多复用寄存器**

#### BIOS 与中断 · `INT`

- **`INT`** = **软件中断**，类似 **函数调用**
- 主板 **出厂固化 BIOS** — 通过指定中断号调用，完成 **显示字符、读键盘** 等基础 I/O
- Day 1 能 **hello, world**，很大程度是 **调 BIOS**，不是 OS 自己驱动显卡

| 概念 | 说明 |
|------|------|
| **BIOS** | 基本输入输出系统，上电后可用 |
| **`INT n`** | 跳到中断向量表第 n 项 — **间接调用** BIOS/固件例程 |

→ 后续 Day 会 **自己写中断处理**；Linux 里对应 **syscall / 驱动** 层。

#### 为什么是 `0x7c00`？

| 问题 | 答案 |
|------|------|
| **`ORG 0x7c00`** 从哪来？ | 早期 **IBM PC + Intel** 约定 |
| 启动时发生什么？ | BIOS 把 **引导扇区 512 字节** 读入内存 **`0x7c00`**，从此处开始执行 |
| 和 Day 1 的关系？ | `helloos.asm` 必须 **ORG 到 0x7c00**，否则跳转、相对地址全错 |

```
磁盘 扇区0 (512B)  ──BIOS 加载──►  物理内存 0x7c00  ──►  CPU 从此执行 IPL
```
