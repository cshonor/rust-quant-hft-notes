## ④ MEMMAN · 基于空闲块列表

**只记录「空闲」**，不记录每一块占用。

#### 核心思想

每条记录：**起始地址 + 连续空闲字节数**

```
空闲列表（示意）
[0x00300000, 65536)  [0x00400000, 131072)  …
```

| 操作 | 行为 |
|------|------|
| **alloc(size)** | 遍历列表，找 **≥ size** 的块 → 切分/标记已用 → **更新列表** |
| **free(addr, size)** | 归还 → **插入列表**；若与 **相邻空闲块** 接壤 → **合并成更大块** |

**合并** 防 **外部碎片** 恶化 — 经典 **coalesce**。

#### `MEMMAN` 结构体

封装 **空闲表 + 块数上限** 等字段，对外 **`memman_alloc` / `memman_free`**（名以原书为准）。

```
程序 malloc 式请求
      │
      ▼
 MEMMAN.alloc → 从某 free 块切出
      │
程序 free
      │
      ▼
 MEMMAN.free → 还回 + 与邻居合并
```

→ 对照 [01-CSAPP Ch9 malloc 与碎片](../../../../01-CSAPP-3rd/chapter-09-virtual-memory/) · Linux **buddy / slab** 是工业级进化 · HFT **对象池 / arena** 常 **预分配、少 free**
