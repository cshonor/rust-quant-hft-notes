> **原书第九章** · 32 位下 **全内存可用** — 容量检测、**CR0 关 Cache**、编译器优化陷阱、**MEMMAN** 空闲块列表。

| 段 | 做什么 | 带走什么 |

## 本日小结

| 问题 | 答案 |
|------|------|
| 怎么测内存？ | **写-反转-读** 探针；**关 Cache（CR0）** |
| C 版为何错？ | **优化删掉测试循环** → 假 3GB |
| 正确实现？ | **汇编 `memtest_sub`** |
| 为何不用位图？ | 大 RAM → **管理表本身太大** |
| MEMMAN 思路？ | **只链表记录空闲块** |
| free 还要啥？ | **相邻块合并** 减碎片 |

**后勤保障：** 多任务、窗口图层、更大数据结构 — 都依赖 Day 9 **可信容量 + alloc/free**。
