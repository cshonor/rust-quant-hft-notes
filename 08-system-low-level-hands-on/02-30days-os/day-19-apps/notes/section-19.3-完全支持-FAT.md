## ③ 完全支持 FAT · 大文件与碎片

#### 小文件方案的局限

| 限制 | 原因 |
|------|------|
| **≤512B** | 只读 **一个簇/扇区** |
| **必须连续** | 未查 **下一簇** |

#### FAT（文件分配表）

软盘 **FAT12** 为省空间：**2 个 12 位簇号压进 3 字节** → 需 **解压展开** 成可用表。

**读文件链：**

```
clust = 起始簇
loop:
    读 clust 对应扇区数据
    clust = FAT[clust]   /* 下一簇 */
until clust == 0xFFF (EOC，文件结束标记)
```

| 能力 | 说明 |
|------|------|
| **任意大小** | 多簇接力 |
| **非连续（碎片）** | FAT **链式** — 物理上不邻接也能读 |

**`file_loadfile`（示意名）** — 完整读入 **MEMMAN 分配** 的缓冲区。

→ 对照 **ext4 inode 间接块** — 思想同为 **「表驱动块链」**
