## ③.3 32 位保护模式与段 / 页

**一句话：让 OS 摆脱 1 MB + 分段 + BIOS 的束缚，用 32 位寄存器跑 C 内核，并打开内存保护与分页基础。**

| 16 位实模式（IPL / nasmhead 前半） | 32 位保护模式（HariMain） |
|-----------------------------------|---------------------------|
| 段×16+偏移，≈ **1 MB** | 平坦或分段，可达 **4 GB** 寻址 |
| **`AX、BX`…** 16 位 | **`EAX、EBX`…** 32 位 |
| 调 **`INT` BIOS** | **不能再调 BIOS** |
| 512 B IPL 只能写汇编 | **C 编译器默认出 32 位代码** |
| 无权限隔离 | **GDT / 特权级 / 后续分页** |

**nasmhead 切模式时要做：**

1. **开 A20** — 解除 1 MB 以上内存访问限制  
2. **建 GDT** — 描述符表  
3. **设 CR0 保护位** — 进保护模式  
4. **跳转到 32 位代码段** — 执行 **`HariMain`**

切模式细节与 C 交接见 [§3.4.3](./section-3.4.3-16切32与call-C完整例子.md)。

### 段 vs 页：两种划分内存的方式（别混）

**16 位实模式没有分页**，不是「段 + 页 + 偏移」，而是：

```text
物理地址 = 段寄存器 × 16 + 偏移（16 位）
例：ES=0x0820, BX=0  →  0x8200
```

| | **段（Segment）** | **页（Page）** |
|---|------------------|----------------|
| **什么时候** | **16 位就有**；32 位仍保留段机制 | **32 位保护模式起** 才广泛用 |
| **按什么切** | **逻辑角色** — CS 代码、DS 数据、SS 栈、ES… | **固定 4 KB 块**，与逻辑功能无关 |
| **谁定边界** | 段基址 + 偏移；保护模式还有 **GDT limit/权限** | **页表** 映射虚拟页 → 物理帧 |
| **典型目的** | 8086 在 **1 MB** 里寻址 | **虚拟内存、进程隔离、换页** |
| **haribote** | **`ES:BX` 读盘** 就是段:偏移 | Day 3 **先 GDT**；完整分页 **后面 Day** |

- **段 ≈ 逻辑划分** — 「代码 / 数据 / 读盘用 ES」  
- **页 ≈ 均匀切格** — 每格 4 KB，方便 **映射与保护**

**x86 实际做法：** 32 位后 **段 + 页叠加** — 很多 OS 用 **平坦模式**：段覆盖 4 GB，**主要靠分页** 管内存。
