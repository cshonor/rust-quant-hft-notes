# Day 13 · 定时器（2）


> **原书第十三章** · **数据结构重构** — 统一 **32 位 FIFO**、性能基准、定时器 **链表 `next`**、**哨兵** 消边界分支。

---

### 本节四段结构

| 段 | 做什么 | 带走什么 |
|----|--------|----------|
| **① 统一 FIFO** | 键/鼠/定时器 **一个队列** | **数值区间** 区分来源 |
| **② 性能测试** | 高速计数器 **3s 后** 开测 | **10s 最大 count** 对比优化 |
| **③ 链表** | 定时器 **`next` 指针** | **不再数组移位** |
| **④ 哨兵** | 末尾 **`0xffffffff`** 假定时器 | 插入逻辑 **少 4 种 if** |

## 本日小结

| 问题 | 答案 |
|------|------|
| 为何统一 FIFO？ | **一个队列 + 数值段** — 主循环 **单 dispatch** |
| 怎么测优化？ | **3s 后清零计数器，10s 比 max** |
| 数组瓶颈？ | 超时 **全体前移 O(n)** |
| 链表解法？ | **`TIMER.next`** — 只 **改指针** |
| 哨兵干嘛？ | **`0xFFFFFFFF` 尾节点** — **消边缘 if** |
| 与 Day 12？ | **12 = PIT + next 比较**；**13 = 结构升级 + 统一事件源** |
