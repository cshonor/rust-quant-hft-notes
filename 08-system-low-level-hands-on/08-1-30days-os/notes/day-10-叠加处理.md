### ① 内存管理（续）· 4KB 对齐与位运算舍入

Day 9 **空闲块列表** 可用；多次 alloc/free 仍可能 **碎片凌乱**。

**规定：** 一律以 **4KB（`0x1000`）** 为单位 **分配与释放** — 与页粒度直觉一致，合并更简单。

| 需求 | 实现 |
|------|------|
| 请求 size **向上取整** 到 4KB 边界 | **位运算 AND**，不用 **除法/模** |
| 地址 **向下对齐** | 如 `(addr + 0xFFF) & ~0xFFF` 一类（示意） |

**为何用 AND：** 当 size 为 **2 的幂** 时，对齐 = **掩码截断** — **几条指令**，比除法 **快几个数量级**。

**HFT：** **64B cache line / 4KB page / hugepage** 对齐 — 同一 **「对齐用位掩码」** 思路；热路径 **避免 div/mod**。

### ② 图层引入 · 解决 Day 8 重影/破坏

Day 8 鼠标 **直接写 VRAM** → 移过 **任务栏** 时 **背景被永久改掉**。

#### 透明胶片模型

| 概念 | 类比 |
|------|------|
| **Sheet（图层）** | 一张 **透明胶片**，上面画一块 bitmap |
| **Z 轴高度** | 胶片 **叠放顺序** |
| **SHTCTL** | **图层控制器** — 管理所有 sheet、高度、刷新 |

典型顺序（自底向上）：

```
Z 低 ──► 高
  桌面背景  →  窗口/任务条  →  鼠标指针（最顶）
```

**刷新：** 自 **底到顶** 依次 **合成** 到屏幕 — 顶层 **自然盖住** 下层，鼠标 **悬浮** 而不 **破坏** 底下像素（底图保存在 **各自 sheet 缓冲** 里）。

#### 结构体（示意）

| 结构 | 职责 |
|------|------|
| **`SHEET`** | 单图层：buf、位置、大小、高度、是否可见… |
| **`SHTCTL`** | 全系统 sheet 表、排序、refresh 入口 |

**Day 8 伏笔在此闭合** — 为 **多窗口时代** 打 GUI 合成基础。

### ③ 提高速度（1）· 局部刷新

初版 refresh：**鼠标动 1 像素 → 重画全屏 320×200 = 64 000 像素** → **慢 + 闪**。

**改 `sheet_refreshsub`：** 只重绘 **发生变化的矩形区域（damage rect）**。

| 例 | 量级 |
|----|------|
| 16×16 鼠标 **旧位置 + 新位置** 并集 | 约 **512 像素** 量级 |
| 占全屏 | **~0.8%** |

```
全屏 refresh（❌）          局部 refresh（✅）
████████████████          ████············
████████████████    →    ····████········
（64000 px）              （仅 dirty rect）
```

**HFT 对照：** **damage region / 脏矩形** — 游戏、UI、行情面板 **只重算变化区**；全量重绘 = 热路径灾难。

---

### ④ 提高速度（2）· 倒推循环边界

局部刷新后仍 **for 全图层宽高 + if 跳过** → **空转 CPU**。

**倒推：** 由 dirty rect 与 sheet 交集， **直接算** 要写的 **`x0..x1`, `y0..y1`（原书如 `bx0..bx1`）** — **`for` 只跑这个范围**。

| 之前 | 之后 |
|------|------|
| 遍历 64000 次，大多 `if` 跳过 | 循环次数 ≈ **脏区像素数** |
| 分支预测 miss、浪费 | **窗口/鼠标「嗖嗖」动** |

```c
/* 示意：循环边界 = 交集，而非 sheet 全尺寸 */
for (y = by0; y < by1; y++)
    for (x = bx0; x < bx1; x++)
        /* 合成像素 */
```

**两层优化叠加：**  
① **只 refresh 脏矩形**  
② **脏矩形内只 loop 必要坐标**

→ [02 SysPerf 少做无用功](../../02-Systems-Performance-2nd/chapter-02-methodologies/) · 量化 **减少遍历域** = 降延迟

### Day 10 小结

| 问题 | 答案 |
|------|------|
| MEMMAN 新规则？ | **4KB 对齐** alloc/free · **AND 舍入** |

| 里程碑？ | **高性能多图层引擎** → 多窗口 |
