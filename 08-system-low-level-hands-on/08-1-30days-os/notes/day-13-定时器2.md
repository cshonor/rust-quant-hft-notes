### ① 统一 FIFO 缓冲区 · 化繁为简

Day 7–12：**键盘 FIFO、鼠标 FIFO、多个定时器 FIFO…** — 各自 init，**HariMain 里一堆 if** 问「哪个设备有数据」。

#### 合并为一个 **32 位 FIFO**

| 数据值范围（原书示意） | 含义 |
|------------------------|------|
| **0~1** | 光标定时器等 |
| **3、10** 等 | 不同 **秒数** 的超时事件 |
| **256~511** | **键盘** |
| **512~767** | **鼠标** |

**ISR / 定时器逻辑：** 事件发生时 **往同一 FIFO Put(编码)**  
**主循环：** **只读一个 FIFO** → 看数值落在哪段 → **dispatch**

```
键盘 ISR ──► Put(256+scancode)
鼠标 ISR ──► Put(512+byte)
timer    ──► Put(3) / Put(10) / …
                │
                ▼
           单一 FIFO32
                │
HariMain ◄──────┘  一次 Get → 区间判断
```

| 收益 | 说明 |
|------|------|
| **初始化一次** | 少 N 套 buffer |
| **主循环单入口** | 类似 **event loop / epoll 统一 fd** |
| **扩展设备** | 新设备 = **新数值段** |

**HFT：** **单生产者多消费者 ring**、**multiplexed event id** — 同一 **「一条队列 + 类型字段」** 模式。

### ② 性能测试机制 · 量化优化

Day 11 **窗口内高速计数器** 复用为 **benchmark**。

| 设定 | 原因 |
|------|------|
| 启动 **3 秒后** 再 **清零** 计数器 | 避开 **init / 首屏 refresh** 干扰 |
| 跑 **10 秒** 看 **count 最大值** | 优化前后 **同一 workload** 比吞吐 |

```
优化前 10s → count = A
优化后 10s → count = B   (B > A 即 ISR/主循环更「空」)
```

**原则：** 底层改算法 **必须可测** — 与 [02 SysPerf 基准](../../02-Systems-Performance-2nd/chapter-12-benchmarking/) 同源。

### ③ 链表结构 · 告别数组移位

Day 12 **数组 + `next` 全局最近超时** 仍可能在 **超时删除** 时：

```c
/* 朴素：最前定时器到期 */
for (i = 0; i < n-1; i++)
    timer[i] = timer[i+1];   /* 全体前移 — O(n) */
```

定时器 **500 个** × **100Hz** 检查 → **移位极贵**。

#### 链表版 `TIMER`

在 **`TIMER` 结构体加 `next` 指针** — 按 **超时先后顺序** 串成 **有序链表**：

| 操作 | 数组移位 | 链表 |
|------|----------|------|
| 超时弹出 | O(n) memmove | **`prev->next = cur->next`** — O(1) |
| 插入新定时器 | 可能 O(n) | **找位置改指针** — 无 bulk 移动 |

```
timerctl.head ──► T1 ──► T3 ──► T7 ──► …
                  next   next   next
```

**内存中的 TIMER 对象不搬家** — 只 **改链接** — ISR **时间骤降**。

→ [01-CSAPP Ch12 链表 vs 数组](../../01-CSAPP-3rd/chapter-12-concurrent-programming/) · Linux **`hrtimer` 红黑树/堆** 是更高级版

### ④ 哨兵（Sentinel）· 简化插入

链表快了，但 **插入** 要分：

- 插 **最前**
- 插 **最后**
- 只有 **1 个** 节点
- 插 **中间**

→ **`if` 爆炸**。

#### 哨兵技巧

初始化时在链表 **末尾挂一个特殊节点**：

| 属性 | 值 |
|------|-----|
| **超时时刻** | **`0xFFFFFFFF`**（极大） |
| **角色** | **永不真正超时** 的 **留守节点** |

**效果：**

- 真实链表 **永远至少 2 个节点**（含哨兵）
- **「插到最后」** = 插在哨兵 **前面** — 与普通插入 **同一逻辑**
- **「仅 1 个定时器」** 不再单独分支

```
… ──► T_last ──► [哨兵 timeout=∞] ──► NULL
```

**经典模式：** 哑头/哑尾 **sentinel node** — 内核链表、`std::list` 技巧同源。
