### ① 性能测试总结 · 验证 Day 13 优化

Day 13 引入 **定时器链表 + 哨兵** 后，用 Day 11–13 的 **高速计数器** 再测一轮。

| 指标 | 含义 |
|------|------|
| **中断禁止时间** | `CLI` 到 `STI`（或等价临界区）**持续更短** |
| **并发响应** | 键/鼠/定时器 **同时来时** 更少 **丢事件 / 卡顿** |

**结论：** 算法优化 **不只 count 变大** — **ISR 临界区变短** = **实时性** 提升。

**HFT：** **IRQ 里关中断窗口、lock hold time** — 同一 **「临界区越短，并发越稳」**。

### ② VBE · 提高画面分辨率

Day 3–11 长期 **320×200×8**（模式 13h 一类）— 画面 **偏「复古」**。

#### VBE（VESA BIOS Extension）

| 项 | 说明 |
|----|------|
| **标准** | 显卡 **统一 BIOS 扩展接口** |
| **调用** | 实模式 **`INT 0x10`**，如 **`AX = 0x4F02`** 设模式 |
| **目标** | **640×480** 等 **高分辨率** |
| **VRAM** | 模式不同 → **基址/步长** 可能变 — 需 **`BOOTINFO` 更新** |

#### 兼容性 · 自动降级

```
尝试 VBE 640×480
    │
    ├─ 成功 → 高分辨率启动
    └─ 不支持 / 失败 → 回退 320×200（旧路径仍可用）
```

**在 `asmhead.nas` 完成** — 仍须在 **进 32 位前** 调 BIOS（Day 3/8 约束）。

→ [Day 8 asmhead](./day-08-鼠标控制与32位模式切换.md) · [Day 5 BOOTINFO](./day-05-结构体文字显示与GDT-IDT.md)

### ③ 真正实现键盘输入 · keytable

Day 6–7 屏幕显示 **原始扫描码**（如 **A = 0x1E**）— 用户要 **字母**。

#### 查表法

定义 **`keytable[]`**（及 **shift 表** 等，以原书为准）：

```c
/* 示意：scancode → ASCII */
char c = keytable[scancode];
putchar(c, ...);
```

| 对比 | 数十个 `if (code==0x1E)` | **`keytable[code]`** |
|------|--------------------------|----------------------|
| 维护 | 难扩展 | **改表即可** |
| 速度 | 分支链 | **O(1) 数组读** |

FIFO 统一路径（Day 13）：键盘中断 → **Put(256+scancode)** → 主循环 **查表输出**。

→ [Day 7 0x0060 扫描码](./day-07-FIFO与鼠标控制.md)

#### 拖拽窗口

复用 Day 10–11 **图层**：

| 已有 | 本日组合 |
|------|----------|
| **`sheet_slide(sheet, dx, dy)`** | 移动 sheet **原点** |

