### ② tek 解压缩 · file_loadfile2

#### 动机

**`nihongo.fnt` + 众多 .hrb** — 软盘 **容量紧** · 启动 **读盘慢**。

#### tek（主要 tek5）

作者自研格式 — **压缩率 / 解压速度 / 解压器体积** 三角 **平衡**。

| 组件 | 作用 |
|------|------|
| **`tek.c`** | 解压算法移植进内核 |
| **`file_loadfile2`** | 载入时 **判 tek 魔数/头** → **内存中解压** → 交给调用方 |

```
磁盘 .tek / tek 包装文件 → loadfile2 → 明文 buffer
```

| 成效（原书） | |
|--------------|--|
| **`nihongo.fnt`** | **142KB → 56.6KB** |
| 各 **.hrb** | 体积 **显著下降** |

**透明解压** — app **不必知道** 文件是否压缩 — 类似 **内核 zlib 读 ext4** 的迷你版。

**HFT：** **zstd/lz4 行情字典、快照压缩** — 同一 **「CPU 换 I/O 与空间」**。

→ [Day 19/28 file_load](./day-19-应用程序.md)

### ③ 封装 C 语言标准函数

在 **INT 0x40 API** 之上再包一层 **C 标准库风格**：

| 函数 | 底层 |
|------|------|
| **`putchar` / `printf`** | EDX 1/2/3… 显示 API |
| **`exit`** | 结束 app / RETF 链 |
| **`malloc` / `free`** | EDX 9/10 |

**app 写「像 POSIX 一点」的 C** — 不直接 **`_api_*` + EDX`** — **可移植感↑**。

→ [Day 27 apilib.lib](./day-27-LDT与库.md) · [Day 20 API](./day-20-API.md)
