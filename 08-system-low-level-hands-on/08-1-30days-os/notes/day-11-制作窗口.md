### ① 解决鼠标移出画面时的异常

Day 10 **局部 refresh** 未严格 **裁到屏幕内** → 指针拖到 **右/下边界外** 时，dirty rect 仍含 **屏外坐标** → **VRAM 写错区 / 合成错乱**。

**改 `sheet_refreshsub`：**

| 步骤 | 行为 |
|------|------|
| 算 dirty 与 sheet 交集 | 与 Day 10 相同 |
| **再与屏幕 `[0,w)×[0,h)` 求交** | **屏外部分直接跳过** |

→ **画面外坐标安全** — 边界 clamp 是 GUI 引擎 **必做** 项。

### ③ 高速计数器 · 初步消闪烁

#### 动态压测

- 窗口内 **计数器** 不断 **+1 并重绘数字**
- **去掉 HLT** — CPU **满速跑**（测 **refresh 性能**）

#### 闪烁原因（初版）

每次 **局部 refresh** 都从 **最底背景层** 开始 **整栈合成** → 计数器区域 **背景也被反复重画** → **闪**。

#### 初步优化

**只 refresh：**

1. **内容发生变化的 sheet**（计数器所在层）
2. **以及 Z 轴在其之上的所有 sheet**（如鼠标）

**不再从背景层重画整个 dirty 区** → **背景闪烁消失**。

```
优化前：背景 → … → 计数器 → 鼠标（全栈）
优化后：     … → 计数器 → 鼠标（仅变化层及以上）
```

---

### ④ map 机制 · 彻底消除闪烁

#### 新问题

鼠标 **悬停在狂刷的计数器上** → **鼠标本身闪**。

| 原因 | 时序 |
|------|------|
| 重绘计数器数字 | **先写入** 计数器层像素 → **瞬间盖住** 鼠标 |
| 接着 refresh 上层 | **再画鼠标** |
| 人眼 | **覆盖→恢复** 时间差 = **闪烁/重影** |

#### map（图层地图）

开辟 **与屏幕同尺寸** 的 **map[]** — **每个像素记录「当前最可见层编号 / 色号归属」**。

**刷新下层 sheet 写 VRAM 前：**

```
若 map[vram_offset] 指向 **比当前 sheet 更高** 的层
    → **跳过写入**（该像素归鼠标等上层所有）
否则
    → 正常合成写入
```

| 无 map | 有 map |
|--------|--------|
| 下层 refresh **盲目覆盖** 上层占用的像素 | **对照 map** — **只写仍归本层可见的像素** |
| 鼠标闪 | **计数器 + 鼠标 同时高速** 仍 **无闪** |

```
屏幕像素 P
    map[P] = 最高可见 sheet id
refresh 计数器层写 P 前：
    if map[P] > 计数器层高度 → skip
```

**核心技术总结：** **合成器知道每个像素的「所有权」** — 类似现代 Compositor 的 **damage + z-order +  occlusion 信息**。

**HFT：** 无 GUI，但 **order book 多源写同一 slot** 也要 **版本/归属** 避免 **stale overwrite** — 同一 **「写前先查谁当家」** 逻辑。

### Day 11 小结

| 问题 | 答案 |
|------|------|
| 出屏错乱？ | **`sheet_refreshsub` 与屏幕矩形求交** |
