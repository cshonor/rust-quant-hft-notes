### ① 初始化 PIT · 使用定时器

#### 为何需要硬件定时器？

键鼠 **事件驱动** — 没输入就没事；但 OS 还需要 **流逝时间**：超时、动画、**时间片**。

| 硬件 | 说明 |
|------|------|
| **PIT** | Programmable Interval Timer（如 **8254**） |
| **连线** | **IRQ0** → 经 PIC → CPU |
| **编程** | 设定 **分频** → 每隔固定间隔 **脉冲一次中断** |

原书示例：**约每秒 100 次** → tick 间隔 **10ms**。

```
PIT 计数减到 0 ──► IRQ0 ──► PIC ──► IDT[0x20] ──► inthandler20
```

**ISR 仍要短**（Day 7 原则）— 这里 mostly **count++**、检查 **next**。

### ② 计量时间 · 操作系统的秒表

#### `TIMERCTL` 结构体

| 字段（示意） | 作用 |
|--------------|------|
| **`count`** | 每来一次定时器中断 **+1** |
| 其他 | 后续 timeout / `next` 等 |

**频率固定** → 时间换算：

```
秒数 ≈ count / 100        （100Hz 时）
3 分钟泡面 = 180 秒 = 18000 tick（示意）
```

**里程碑：** OS 从「能画能点」→ **能回答 now 是启动后第几秒** — 调度、超时、动画的 **时钟源**。

**HFT：** **steady_clock / TSC / chrono** — 工业级 **单调时钟**；PIT tick 是 **最简全局 heartbeat**。

---

### ③ 超时通知 · 管理多个定时器

#### Timeout 语义

**「过了 XX tick / 秒 → 通知程序」** — 光标闪、应用闹钟、网络重传（概念上同源）。

#### 与 FIFO 结合

倒计时 **归零** 时 → ISR 或 timer 逻辑向 **主程序 FIFO** **写入约定字节/事件码** → **HariMain** 里 **读 FIFO 处理**（与 Day 7 键鼠 **同一套路**）。

#### 多路定时器

| 机制 | 说明 |
|------|------|
| **定时器数组** | 如 **500 个 `TIMER` 槽** |
| **`TIMER_FLAGS_USING`** | 槽位 **是否占用** |
| 每槽 | **剩余 tick**、超时回调数据、flags… |

**并发需求：** 光标 0.5s 闪 + 窗口 3s 关闭 + 应用 N 个 alarm — **不能只有一个全局 countdown**。

---

### ④ 加快中断处理 · `next` 优化

#### 朴素做法的问题

**每 tick（100 次/秒）** → **for 循环扫 500 个定时器** 看是否到期 → **绝大多数空转** — CPU **浪费在 ISR 里**（HFT 大忌）。

#### 优化：维护「最近超时时刻」`next`

| 变量 | 含义 |
|------|------|
| **`next`** | 所有 **使用中** 定时器里，**最早到期** 的全局 tick 值 |

**每次定时器中断：**

```
if (now != next)   // 多数情况
    count++; return;     // 只 tick 一次比较
else
    扫描/处理到期定时器; 更新 next;
```

| 对比 | 每 tick 成本 |
|------|----------------|
| 扫 500 个 | O(500) × 100Hz |
| **`next` 比较** | **O(1)** 常见；到期时才 O(活跃数) |

**思路：** **事件驱动 + 最小堆/最近事件** 的雏形 — Linux **`timer wheel` / hrtimer`** 是同一问题的工业答案。
