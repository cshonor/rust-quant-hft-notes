## 2. 文件系统与 FAT 选型

---

### 一、文件系统是什么

| 问题 | 文件系统回答 |
|------|--------------|
| 磁盘是一串 **字节** | 如何 **切块、命名、索引**？ |
| 用户要 **`hello.txt`** | 字节偏移在哪 · 多长 · 是否目录 |

**本质：** 在 **块设备** 上维护 **元数据 + 数据区** 的约定。

---

### 二、为何选 FAT

| 理由 | 说明 |
|------|------|
| **UEFI 生态** | 启动分区常见 **FAT32** — Ch1 **BOOTX64.EFI** 就在 FAT 上 |
| **标准公开** | **BPB / Directory Entry** 文档齐全 |
| **易于测试** | QEMU **虚拟 FAT 盘** · 真机 U 盘同格式 |
| **实现量可控** | 本章只需 **读目录** — 不必先写完整 **分配表更新** |

→ [Ch1 FAT U 盘](../chapter-01-hello-world/notes/section-2-二进制编辑器与BOOTX64.md)

---

### 三、FAT 粗粒度布局（读 `ls` 所需）

```
[ 引导扇区 + BPB ]
[ FAT 表区 × N ]
[ 根目录 / 数据区 — 按簇组织 ]
```

| 概念 | 作用 |
|------|------|
| **扇区 (sector)** | 最小读写单位（通常 512B） |
| **簇 (cluster)** | **若干扇区** — 分配与目录指向的单位 |
| **FAT 表** | 记录 **下一簇** 链（读文件内容时用；列目录可先读 **目录簇**） |

---

### 四、与 Ch2 UEFI 文件访问对照

**Ch2：** **Simple File System Protocol** — UEFI **高层** `Open/Read` 文件。

**Ch17：** 内核 **自己解析 FAT 字节布局** — 不依赖 UEFI 运行时 — 为 **独立 OS** 铺路。
