## 3. 迁移栈与关键数据结构

> **完全接管内存前** — 必须把 UEFI 时代落在 **临时区域** 的结构 **搬到安全处**。

---

### 一、为何要迁移

| 结构 | UEFI 阶段位置风险 |
|------|-------------------|
| **栈（Stack）** | UEFI 堆/固件区 — 后续 **Allocate 可能覆盖** |
| **GDT** | 临时缓冲 — 被覆盖 → **段错误/三重 fault** |
| **页表** | 同上 — **CR3 指向失效表** → 立即崩溃 |

**目标：** 全部位于 **OS 明确保留、不会交给位图自由分配** 的物理页。

---

### 二、重置栈 — `kernel_main_stack`

```cpp
alignas(16) char kernel_main_stack[8192];  // 示意大小
// 汇编切换 RSP：
//   mov rsp, kernel_main_stack + sizeof(stack)
```

| 要点 | 说明 |
|------|------|
| **`kernel_main_stack`** | 内核 **新栈数组** — 在 .bss 或静态分配区 |
| **`RSP` 16 字节对齐** | x86-64 **ABI 要求** — `call` 前对齐 |
| **迁移时机** | **Early kernel init** — 在大量 C++ 调用前完成 |

**之后：** 局部变量、`printk`、中断嵌套 **不再依赖 UEFI 栈**。

---

### 三、迁移 GDT 与页表

| 步骤 | 说明 |
|------|------|
| **memcpy** | 旧 GDT/页表内容 → 新物理/虚拟地址 |
| **更新指针** | 代码中 **GDT 基址、CR3** 指向新位置 |
| **旧区释放策略** | 确认 UEFI 不再使用后 **可纳入分配池**（按书顺序） |

**顺序敏感：** 通常 **新页表 identity 映射新栈/GDT 所在物理页** 后，再 **切 RSP / lgdt / mov cr3**。
