## 3. IDT 与 lidt

---

### 一、中断向量（0–255）

不同事件需要 **不同处理函数** — CPU 用 **向量号** 区分：

| 向量（示例） | 来源 |
|--------------|------|
| **0–31** | CPU 异常（除零、页故障…） |
| **32+** | 可编程 **IRQ/MSI** — 鼠标、键盘、定时器… |

**鼠标 / xHC MSI** 映射到某一 **分配好的向量号** — 与 IDT 表项对应。

---

### 二、中断描述符表（IDT）

**IDT** — 将 **向量号 → 处理程序入口地址** 的表：

```
向量 i  →  IDT[i]  →  { 段选择子, 偏移, 属性… }
                          ↓
                    IntHandler_xxx
```

| 字段（概念） | 作用 |
|--------------|------|
| **Handler 地址** | ISR 入口 |
| **段选择子** | 长模式下仍通过 **GDT** 描述代码段 |
| **类型/特权** | 中断门、陷阱门等 |

**构建流程：**

1. 分配 **对齐的 IDT 数组**
2. 为每个使用的向量 **SetDescriptor(vector, handler)**
3. 汇编 **`lidt`** — 把 **IDTR（基址+限长）** 载入 CPU

```asm
lidt [idtr]    ; 正式注册给 CPU
```

### 三、MikanOS（64 位）与 01（32 位）差异

| | **02 川合** | **MikanOS** |
|---|-------------|-------------|
| 模式 | 保护模式 32-bit | **长模式 64-bit** |
| 门描述符细节 | 16/32 位 IDT 项 | **64 位 IDT 项**（IST 等扩展） |
| 中断控制器 | **PIC** 为主 | **APIC + MSI**（xHC） |

**思想相同：** **向量 → 函数**；实现细节跟手册。
