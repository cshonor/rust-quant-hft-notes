## 5. ACPI PM 定时器校准

---

### 一、问题：APIC 频率不固定

**Local APIC Timer** 递减速度 **因 CPU/芯片组而异** — 不能直接：

```
tick_count / 某常数 = 秒   ❌（换机器就错）
```

**需要：** 用 **已知频率** 的硬件 **反测 APIC Hz**。

---

### 二、ACPI PM Timer

**ACPI 电源管理定时器** — 规范频率 **3.579545 MHz**（固定参照）：

| 特性 | 说明 |
|------|------|
| **频率稳定** | 跨机型 **同一标准** |
| **访问** | **I/O 端口** 或 **MMIO** — 地址来自 **ACPI 表**（§6 RSDP 链） |
| **用法** | 测「APIC 减 N 次」同时 PM 增多少 → 换算 **APIC ticks/sec** |

**校准流程（概念）：**

```
1. 读 PM Timer 起始值 T0，读 APIC 起始值 A0
2. 等待 PM 增加 ΔP（或固定时间）
3. 读 APIC A1
4. apic_hz = (A1-A0) * 3579545 / ΔP
```

校准后：**tick → 毫秒/微秒** 换算可靠。

---

### 三、与 Ch9 benchmark 区别

| Ch9 | Ch11 |
|-----|------|
| APIC 测 **DrawAll 耗时** | APIC **作系统时钟** |
| 相对值 | **绝对时间**（经 PM 校准） |
